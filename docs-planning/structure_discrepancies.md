## How the comparison works

Docusaurus derives a doc's **route id** from its path relative to `docs_v2/`, stripping the leading numeric `N.` ordering prefix from each path segment and the `.md`/`.mdx` extension. So:

- `docs_v2/home/0.index.md` → id `home/index`
- `docs_v2/operating-dial/configuration/3.chat-configuration.md` → id `operating-dial/configuration/chat-configuration`

The sidebar references files by that route id (in `type: doc`, `type: ref`, and category `link.id` entries). The `reference/changelog` category is `type: autogenerated`, so every file under that folder is pulled in automatically and is **not** expected to be listed individually.

These are cosmetic for an explicitly-listed sidebar (Docusaurus renders in menu order regardless of filename), but they make the file tree misleading and should be reconciled with the menu.

## Appendix — suggested order of operations

Re-run `npm run build` (links are set to `throw`) after renaming to catch breakage.
4. Reconcile the **numeric gaps** in §D if the skipped numbers indicate stale removals.
