# Admin configuration

The DIAL Admin Panel consists of two components: Admin Backend ([`epam/ai-dial-admin-backend`](https://github.com/epam/ai-dial-admin-backend)) and Admin Frontend ([`epam/ai-dial-admin-frontend`](https://github.com/epam/ai-dial-admin-frontend)). Admin Backend manages DIAL Core's dynamic configuration (`config.json`), handles config import/export, and provides an API for the frontend. Admin Frontend is a Next.js web application that provides the admin UI.

Both components are configured exclusively through environment variables.

## Admin Backend

### Docker Compose example

From [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
admin-backend:
  image: epam/ai-dial-admin-backend:0.13.0
  environment:
    ENABLE_CORE_CONFIG_VERSION_AUTO_DETECT: "true"
    CONFIG_EXPORT_OUTPUTFILE_PATH: /app/config/config.json
    CONFIG_REST_SECURITY_MODE: "oidc"
    CORE_CLIENT_URL: "http://core:8080"
    DIAL_ADMIN_CLIENT_ID: "dial-admin-console"
    ENABLE_CONFIG_AUTO_IMPORT_ON_BOOTSTRAP: "true"
    H2_DATASOURCE_ENCRYPTED_FILE_KEY: ${H2_DATASOURCE_ENCRYPTED_FILE_KEY}
    H2_DATASOURCE_MASTER_KEY: ${H2_DATASOURCE_MASTER_KEY}
    H2_DATASOURCE_PASSWORD: ${H2_DATASOURCE_PASSWORD}
    H2_FILE: ./data/testdb
    METRICS_ENABLED: "false"
    'providers.keycloak.allowed-roles': 'ConfigAdmin,admin'
    'providers.keycloak.audiences': 'dial-admin-console'
    'providers.keycloak.issuer': "http://localhost:8180/realms/dial"
    'providers.keycloak.jwk-set-uri': "http://keycloak:8180/realms/dial/protocol/openid-connect/certs"
    'providers.keycloak.role-claims': roles
  volumes:
    - ./admin/data:/app/data
    - ./core:/app/config
```

### DIAL Core client

| Variable | Default | Description |
|---|---|---|
| `CORE_CLIENT_URL` | `localhost:8081` | DIAL Core service URL. |
| `DIAL_ADMIN_CLIENT_ID` | — | Keycloak client ID for the admin console. |
| `CORE_PROMPTS_METADATA_DEFAULT_LIMIT` | `256` | Default limit for prompts metadata queries. |
| `CORE_APPLICATIONS_METADATA_DEFAULT_LIMIT` | `256` | Default limit for applications metadata queries. |
| `CORE_TOOLSETS_METADATA_DEFAULT_LIMIT` | `256` | Default limit for toolsets metadata queries. |

### Config file export

Admin Backend periodically exports the managed configuration to a file that DIAL Core reads as its `config.json`.

| Variable | Default | Description |
|---|---|---|
| `CONFIG_EXPORT_ENABLED` | `true` | Enable or disable config file export. |
| `CORE_CONFIG_VERSION` | — | DIAL Core config version for export. Required if auto-detect is disabled. |
| `ENABLE_CORE_CONFIG_VERSION_AUTO_DETECT` | `false` | Auto-detect DIAL Core version from the running instance. |
| `CORE_VERSION_CACHE_EXPIRATION_MS` | `300000` | Cache expiration for auto-detected version (ms). |
| `CONFIG_EXPORT_SYNCPERIOD` | `15000` | Export interval (ms). |
| `ENABLE_CONFIG_RELOAD` | `true` | Write config to destination and call DIAL Core's `/reload_config` endpoint. |
| `DELAY_CONFIG_RELOAD_MILLISECONDS` | `5000` | Delay before calling `/reload_config` after writing the file. |
| `ENABLE_CONFIG_AUTO_RELOAD` | `false` | Enable automatic scheduled `/reload_config` calls. |
| `CONFIG_AUTO_RELOAD_SCHEDULE_DELAY_MILLISECONDS` | `120000` | Interval between auto-reload calls (ms). |
| `CONFIG_EXPORT_STORAGETYPE` | `LOCAL_FILE` | Storage type: `LOCAL_FILE`, `KUBE_SECRET`, or `CONFIG_MAP`. |
| `CONFIG_EXPORT_OUTPUTFILE_PATH` | `data/export/out.json` | Output file path (`LOCAL_FILE` mode). |
| `CONFIG_EXPORT_CONFIGMAP_NAMES` | — | ConfigMap names, comma-separated (`CONFIG_MAP` mode). |
| `CONFIG_EXPORT_CONFIGMAP_KEY` | `env.config.json` | Key in the ConfigMap. |
| `CONFIG_EXPORT_KUBESECRET_NAMES` | — | Kubernetes Secret names, comma-separated (`KUBE_SECRET` mode). |
| `CONFIG_EXPORT_KUBESECRET_KEY` | `kube-secret-key` | Key in the Kubernetes Secret. |
| `CONFIG_EXPORT_CREATE_RESOURCES` | `false` | Create export destination resources if they do not exist. |
| `CONFIG_EXPORT_SYNC_DURATION_THRESHOLD_MS` | `120000` | Threshold before sync status changes to `IN_PROGRESS_TOO_LONG` (ms). |

### Config file import

| Variable | Default | Description |
|---|---|---|
| `IMPORT_CONFIGS_MAX_COUNT` | `64` | Maximum files per import operation. |
| `ENABLE_CONFIG_AUTO_IMPORT_ON_BOOTSTRAP` | `false` | Auto-import configuration on first startup (runs once if database is empty). |
| `IMPORT_AUTO_BOOTSTRAP_FILE_PATHS` | — | Comma-separated file paths for bootstrap import. |
| `IMPORT_AUTO_BOOTSTRAP_STRATEGY` | `MERGE_JSON` | Import strategy: `MERGE_JSON` or `SEQUENTIAL`. |
| `IMPORT_AUTO_BOOTSTRAP_CONFLICT_RESOLUTION` | `OVERRIDE` | Conflict resolution: `OVERRIDE` or `SKIP`. |

### Key vault integration

Admin Backend can fetch secrets from cloud key vaults for config export.

| Variable | Default | Description |
|---|---|---|
| `CONFIG_EXPORT_KEYVAULT_TYPE` | `none` | Key vault type: `none`, `azure`, `vault`, `aws`, or `gcp`. |
| `CONFIG_EXPORT_KEYVAULT_SECRETNAMES` | — | Secret names in key vault (`azure`, `aws`, `gcp`). |
| `CONFIG_EXPORT_KEYVAULT_SECRETPATH` | — | Path to secrets in Hashicorp Vault. |
| `CONFIG_EXPORT_KEYVAULT_EXPIRATION_PERIOD` | `3` | Key vault value expiration period. |
| `CONFIG_EXPORT_KEYVAULT_EXPIRATION_UNIT` | `MONTHS` | Time unit for key vault expiration. |

Cloud provider authentication for key vaults:

| Variable | Description |
|---|---|
| `AUTH_AZURE_TYPE` | Azure auth type: `credential`, `cli`, or `managed`. |
| `AUTH_AZURE_CLIENT_ID` | Azure service principal client ID (`credential` mode). |
| `AUTH_AZURE_TENANT_ID` | Azure tenant ID (`cli` or `credential` mode). |
| `AUTH_AZURE_CLIENT_SECRET` | Azure service principal client secret (`credential` mode). |
| `AZURE_KEY_VAULT_URL` | Azure Key Vault URL. |
| `GCP_KEY_VAULT_PROJECT_ID` | GCP project ID for Secret Manager. |
| `VAULT_URI` | Hashicorp Vault server URL. |
| `VAULT_TOKEN` | Hashicorp Vault access token. |

### Identity providers

Admin Backend supports multiple identity providers using the pattern `providers.{name}.{field}`:

| Field | Description |
|---|---|
| `providers.*.issuer` | Accepted JWT issuers. |
| `providers.*.jwk-set-uri` | JWK Set URI for token verification. |
| `providers.*.user-info-endpoint` | User info URI (alternative to `jwk-set-uri`). |
| `providers.*.audiences` | Accepted JWT audiences. |
| `providers.*.role-claims` | JWT claim paths for roles. |
| `providers.*.allowed-roles` | Roles with access (deprecated — use `roles-mapping`). |
| `providers.*.roles-mapping` | JSON role mapping to `FULL_ADMIN` or `READ_ONLY_ADMIN`. |
| `providers.*.aliases` | Issuer aliases (Azure only). |
| `providers.*.email-claims` | JWT claim paths for email. |
| `providers.*.principal-claim` | Claim used as principal. |

### Datasource

Admin Backend stores its configuration state in a database. H2 is the default for development; PostgreSQL and MS SQL Server are supported for production.

| Variable | Default | Description |
|---|---|---|
| `DATASOURCE_VENDOR` | `H2` | Database vendor: `H2`, `POSTGRES`, or `MS_SQL_SERVER`. |
| `DATASOURCE_AUTH_TYPE` | `basic` | Authentication type: `basic`, `azure`, or `gcp`. |

**H2 (default):**

| Variable | Default | Description |
|---|---|---|
| `H2_FILE` | `./data/testdb` | H2 database file path. |
| `H2_OPS` | `CIPHER=AES;DB_CLOSE_DELAY=-1;DB_CLOSE_ON_EXIT=FALSE` | H2 connection options. |
| `H2_DATASOURCE_MASTERKEY` | — | H2 encryption master key. |
| `H2_DATASOURCE_ENCRYPTEDFILEKEY` | — | H2 encrypted file key. |
| `H2_DATASOURCE_PASSWORD` | — | H2 password. |

**PostgreSQL:**

## Admin Frontend

### Docker Compose example

From [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
admin-frontend:
  image: epam/ai-dial-admin-frontend:0.13.0
  environment:
    DIAL_ADMIN_API_URL: "http://admin-backend:8080/"
    NEXTAUTH_URL: "http://localhost:3100"
    NEXTAUTH_SECRET: "${NEXTAUTH_SECRET}"
    NEXTAUTH_COOKIE_PREFIX: "admin"
    AUTH_KEYCLOAK_HOST: "http://keycloak:8180/realms/dial/"
    AUTH_KEYCLOAK_CLIENT_ID: "dial-admin-console"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
    ADMIN_ROLE_NAMES: 'ConfigAdmin,admin'
  ports:
    - "3100:3000"
```

### Core variables

| Variable | Default | Description |
|---|---|---|
| `DIAL_ADMIN_API_URL` | — | Admin Backend API URL. Required. |
| `APP_NAME` | `AI Dial Admin` | Application name. |

### Feature toggles

| Variable | Default | Description |
|---|---|---|
| `DEPLOYMENTS_ENABLED` | `false` | Enable the deployment manager. |
| `MCP_REGISTRY_ENABLED` | `false` | Enable MCP Registry integration (preview). |
| `NIM_ENABLED` | `false` | Enable NIM Model Serving. |
| `HF_ENABLED` | `false` | Enable HF Model Serving. |
| `ENABLE_EXPORT_CONFIG_MAP` | `false` | Enable config export from DIAL Core config source. |
| `DEPLOYMENTS_DISABLE_JSON_EDITOR` | `false` | Disable JSON editor in deployments. |
| `DEPLOYMENTS_RESOURCES_DEFAULTS` | See below | Default resource limits for deployments. |

Default value for `DEPLOYMENTS_RESOURCES_DEFAULTS` (JSON environment variable):

```json
{
  "CPU_REQUEST": "0.5",
  "CPU_LIMIT": "0.5",
  "MEMORY_REQUEST": "1024",
  "MEMORY_LIMIT": "1024",
  "GPU_REQUEST": "1",
  "GPU_LIMIT": "1"
}
```

### Authentication

| Variable | Default | Description |
|---|---|---|
| `NEXTAUTH_URL` | — | NextAuth URL. Required in production. |
| `NEXTAUTH_SECRET` | — | NextAuth secret. Required. |
| `NEXTAUTH_COOKIE_PREFIX` | — | Custom prefix for NextAuth cookie names. Use when running Admin Frontend alongside DIAL Chat to avoid cookie conflicts. |

Admin Frontend supports the same authentication providers as DIAL Chat (Auth0, Azure AD, Azure AD B2C, GitLab, Google, Keycloak, PingID, Cognito, Okta) with the same variable naming pattern. See [DIAL Chat configuration](chat-configuration) for the full per-provider variable list.
