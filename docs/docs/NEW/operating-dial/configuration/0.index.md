# Configuration overview

This section provides configuration references for all DIAL components. It is intended for DevOps and platform engineers who deploy and operate DIAL.

## Configuration files

DIAL uses JSON, XML, and YAML configuration files alongside environment variables. Each file has a dedicated reference page:

| File | Component | Format | Reload | Reference |
|------|-----------|--------|--------|-----------|
| `settings.json` | DIAL Core | JSON | At startup (static) | [settings.json](core/settings-json/index) |
| `config.json` | DIAL Core | JSON | Every 60 seconds (dynamic) | [config.json](core/config-json/index) |
| `gflog.xml` | DIAL Core | XML | At startup | [gflog.xml](core/gflog-xml) |
| `realm-config.json` | Keycloak | JSON | At Keycloak startup | [realm-config.json](realm-config-json) |
| `application.yaml` | Auth Helper | YAML | At startup | [Auth Helper](auth-helper-configuration) |
| `docker-compose.yml` | Docker Compose | YAML | At `docker compose up` | [Docker Compose](docker-compose/index) |
| `.env` | Docker Compose | Key=Value | At `docker compose up` | [.env reference](docker-compose/env-reference) |
