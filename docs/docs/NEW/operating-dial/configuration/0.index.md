# Configuration overview

This section provides configuration references for all DIAL components. It is intended for DevOps and platform engineers who deploy and operate DIAL.

## How DIAL is configured

DIAL uses a combination of JSON configuration files, environment variables, and (for Kubernetes) Helm chart values. The two most important files are:

- **`settings.json`** — DIAL Core static settings. Loaded once at startup. Configures identity providers, blob storage, Redis, and encryption.
- **`config.json`** — DIAL Core dynamic settings. Hot-reloaded every 60 seconds without restart. Defines models, applications, roles, API keys, and routes.

All other DIAL components (Chat, adapters, Auth Helper, Admin) are configured exclusively through environment variables.

See [Configuration files](configuration-files) for a complete inventory of all files, their formats, mount points, and working examples.

## Docker Compose examples

Working Docker Compose configurations are available in the [AI DIAL repository](https://github.com/epam/ai-dial):

| Setup | Directory | Description |
|---|---|---|
| Echo application | [`dial-docker-compose/application/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/application) | Minimal setup with a custom echo app |
| Azure models | [`dial-docker-compose/model/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/model) | Two Azure OpenAI models via the OpenAI adapter |
| Ollama (self-hosted) | [`dial-docker-compose/ollama/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/ollama) | Self-hosted chat, vision, and embedding models |
| vLLM (self-hosted) | [`dial-docker-compose/vllm/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/vllm) | Self-hosted model via vLLM |
| Production (advanced) | [`dial-docker-compose-advanced/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced) | Keycloak auth, Admin Panel, DALL-E, RAG, multiple models |

Each setup includes a complete `config.json`, `settings.json`, and `docker-compose.yml` that can be used as a starting point.
