# DIAL Core configuration

DIAL Core ([`epam/ai-dial-core`](https://github.com/epam/ai-dial-core)) is configured through two files:

- **`settings.json`** — static settings loaded at startup. Configures identity providers, blob storage, Redis, encryption, and server options. Requires a restart to change.
- **`config.json`** — dynamic settings hot-reloaded every 60 seconds. Defines models, applications, interceptors, roles, API keys, routes, and tool sets.

Static settings can also be passed as environment variables with the `aidial.` prefix (e.g., `aidial.server.port`). See [Configuration files](configuration-files) for file format details, mount points, and complete working examples.

## Static settings

### Configuration precedence

When the same setting is defined in multiple places:

1. **Environment variables** with the `aidial.` prefix (highest priority)
2. **Settings file** specified by the `AIDIAL_SETTINGS` environment variable
3. **Default resource file** bundled with DIAL Core (`aidial.settings.json`)

### Server

| Setting | Default | Description |
|---|---|---|
| `server.port` | `8080` | HTTP server port. |
| `server.acceptBacklog` | `4096` | TCP accept backlog. |
| `server.idleTimeout` | `300000` | Idle timeout in milliseconds. |
| `server.compressionSupported` | `true` | Enable HTTP response compression. |

Additional Vert.x `HttpServerOptions` can be passed under `server.*`.

### Client

| Setting | Default | Description |
|---|---|---|
| `client.idleTimeout` | `300000` | Idle timeout for outbound HTTP connections (ms). |
| `client.connectTimeout` | `10000` | Connection timeout for outbound HTTP connections (ms). |
| `client.keepAlive` | `true` | Enable HTTP keep-alive for outbound connections. |
| `client.maxPoolSize` | `4096` | Maximum connection pool size for outbound requests. |

Additional Vert.x `HttpClientOptions` can be passed under `client.*`. WebSocket client settings are available under `webSocketClient.*`.

### Identity providers

At least one identity provider must be configured. Each provider is defined under `identityProviders.<name>`.

| Setting | Default | Required | Description |
|---|---|---|---|
| `identityProviders.<name>.jwksUrl` | — | Conditional | URL to JWKS provider. Required if `disableJwtVerification` is `false`. Either `jwksUrl` or `userInfoEndpoint` must be provided. |
| `identityProviders.<name>.userInfoEndpoint` | — | Conditional | URL to user info endpoint. Either `jwksUrl` or `userInfoEndpoint` must be provided. |
| `identityProviders.<name>.rolePath` | — | Yes | Path to user roles claim in JWT or user info response. Can be a single string or array of strings. |
| `identityProviders.<name>.projectPath` | — | No | Path to the claim from which the project name is taken (e.g., `azp`, `aud`). |
| `identityProviders.<name>.rolesDelimiter` | — | No | Delimiter to split roles when they come as a single string. |
| `identityProviders.<name>.loggingKey` | — | No | Claim name for user identification in logs (e.g., `email`, `sub`). |
| `identityProviders.<name>.loggingSalt` | — | No | Salt to hash user information for logging. |
| `identityProviders.<name>.issuerPattern` | — | No | Regular expression to match the `iss` claim to this identity provider. |
| `identityProviders.<name>.disableJwtVerification` | `false` | No | Disable JWT verification. `userInfoEndpoint` must be unset when `true`. For development only. |
| `identityProviders.<name>.audience` | — | No | When set, validated against the `aud` claim in the JWT. |
| `identityProviders.<name>.userDisplayName` | — | No | Path to claim for user display name. |
| `identityProviders.<name>.userIdPath` | `sub` | No | Path to claim for user ID. Microsoft Entra ID uses `oid`. |
| `identityProviders.<name>.positiveCacheExpirationMs` | `600000` | No | JWKS cache TTL on success (ms). |
| `identityProviders.<name>.negativeCacheExpirationMs` | `10000` | No | JWKS cache TTL on failure (ms). |

**Development example** (JWT verification disabled):

```json
{
  "identityProviders": {
    "test": {
      "rolePath": "roles",
      "disableJwtVerification": true
    }
  }
}
```

**Production example** (Keycloak, via environment variables):

```yaml
core:
  environment:
    'aidial.identityProviders.keycloak.jwksUrl': 'http://keycloak:8180/realms/dial/protocol/openid-connect/certs'
    'aidial.identityProviders.keycloak.rolePath': 'roles'
    'aidial.identityProviders.keycloak.loggingKey': 'email'
    'aidial.identityProviders.keycloak.loggingSalt': 'loggingSalt'
```

### Storage

DIAL Core stores files (conversation history, uploaded attachments, application data) in blob storage.

| Setting | Default | Required | Description |
|---|---|---|---|
| `storage.provider` | `filesystem` | Yes | Blob storage provider: `s3`, `aws-s3`, `azureblob`, `google-cloud-storage`, or `filesystem`. |
| `storage.endpoint` | — | Conditional | Endpoint URL for S3-compatible storage. |
| `storage.identity` | — | Conditional | Storage access key. Optional for `filesystem`, `aws-s3`, `google-cloud-storage`. |
| `storage.credential` | — | Conditional | Storage secret key. Optional for `filesystem`, `aws-s3`, `google-cloud-storage`. |
| `storage.bucket` | — | No | Bucket name. |
| `storage.createBucket` | `false` | No | Create the bucket on startup if it does not exist. |
| `storage.prefix` | — | No | Base prefix for all stored resources. Enables multiple environments to share a bucket. Must not contain path separators. |
| `storage.maxUploadedFileSize` | `536870912` (512 MB) | No | Maximum uploaded file size in bytes. Returns HTTP 413 if exceeded. |
| `storage.overrides.*` | — | No | Key-value pairs to override storage settings (e.g., jclouds properties). |

**Filesystem example** (from [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml)):

```yaml
'aidial.storage.provider': 'filesystem'
'aidial.storage.prefix': 'core'
'aidial.storage.bucket': 'dial'
'aidial.storage.createBucket': 'true'
'aidial.storage.overrides': '{"jclouds.filesystem.basedir": "data"}'
```

### Redis

DIAL Core requires Redis for in-memory caching. Either single server or cluster configuration must be provided.

| Setting | Default | Description |
|---|---|---|
| `redis.singleServerConfig.address` | — | Redis single server address (e.g., `redis://host:6379`). |
| `redis.clusterServersConfig.nodeAddresses` | — | JSON array of Redis cluster node addresses. |

For cloud-managed Redis with IAM authentication:

| Setting | Description |
|---|---|
| `redis.provider.name` | Provider: `aws-elasti-cache`, `gcp-memory-store`, or `azure-redis-cache`. |
| `redis.provider.userId` | IAM-enabled user ID (AWS ElastiCache). |
| `redis.provider.accountName` | Service account resource name (GCP). |
| `redis.provider.region` | Geo region (AWS ElastiCache). |
| `redis.provider.clusterName` | Redis cluster name (AWS ElastiCache). |
| `redis.provider.serverless` | Whether the cache is serverless (AWS ElastiCache). |

See [Supporting services](supporting-services) for Redis server configuration.

### Minimal example

From [`dial-docker-compose/settings/settings.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/settings/settings.json) — a minimal development setup with JWT verification disabled:

```json
{
  "server": {
    "port": 8080
  },
  "identityProviders": {
    "test": {
      "rolePath": "roles",
      "disableJwtVerification": true
    }
  },
  "encryption": {
    "secret": "salt",
    "key": "password"
  }
}
```

This file is mounted into the container via Docker Compose:

```yaml
core:
  environment:
    AIDIAL_SETTINGS: '/opt/settings/settings.json'
  volumes:
    - ./settings:/opt/settings
```

### Alternative: inline environment variables

Instead of a settings.json file, static settings can be passed as environment variables with the `aidial.` prefix. The advanced Docker Compose setup uses this pattern for identity provider configuration:

```yaml
core:
  environment:
    'aidial.identityProviders.keycloak.jwksUrl': 'http://keycloak:8180/realms/dial/protocol/openid-connect/certs'
    'aidial.identityProviders.keycloak.rolePath': 'roles'
    'aidial.identityProviders.keycloak.loggingKey': 'email'
    'aidial.identityProviders.keycloak.loggingSalt': 'loggingSalt'
    'aidial.encryption.secret': '${DIAL_CORE_ENCRYPTION_SECRET}'
    'aidial.encryption.key': '${DIAL_CORE_ENCRYPTION_KEY}'
    'aidial.redis.singleServerConfig.address': 'redis://redis:6379'
    'aidial.storage.provider': 'filesystem'
    'aidial.storage.prefix': 'core'
    'aidial.storage.bucket': 'dial'
    'aidial.storage.createBucket': 'true'
    'aidial.storage.overrides': '{"jclouds.filesystem.basedir": "data"}'
    'aidial.applications.includeCustomApps': 'true'
```

Environment variables take precedence over settings.json values. See [Configuration precedence](#configuration-precedence) for the full priority order.

## Dynamic settings

Dynamic settings are defined in `config.json` and hot-reloaded at the interval specified by `config.reload` (default: 60 seconds). They can also be managed through [DIAL Admin](/tutorials/admin/home).

### Models

Each model is defined under `models.<deployment_name>`.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `type` | string | Yes | Model type: `chat`, `embedding`. |
| `displayName` | string | No | Display name in the UI. |
| `displayVersion` | string | No | Display version. |
| `iconUrl` | string | No | Icon URL. |
| `description` | string | No | Model description. |
| `endpoint` | string | Yes | Adapter endpoint URL for this model. |
| `upstreams` | array | No | List of upstream provider endpoints. See [Upstream configuration](#upstream-configuration). |
| `features` | object | No | Feature flags for the model. |
| `features.rateEndpoint` | string | No | Endpoint for rate requests. |
| `features.tokenizeEndpoint` | string | No | Endpoint for tokenization requests. |
| `features.truncatePromptEndpoint` | string | No | Endpoint for prompt truncation. |
| `features.systemPromptSupported` | boolean | No | Whether the model supports system prompts. |
| `features.toolsSupported` | boolean | No | Whether the model supports tool calling. |
| `features.seedSupported` | boolean | No | Whether the model supports seed for reproducibility. |
| `features.urlAttachmentsSupported` | boolean | No | Whether the model supports URL attachments. |
| `features.folderAttachmentsSupported` | boolean | No | Whether the model supports folder attachments. |
| `inputAttachmentTypes` | array | No | MIME types the model accepts as input (e.g., `["image/png","image/jpeg"]` or `["*/*"]`). |
| `maxInputAttachments` | integer | No | Maximum number of input attachments. |
| `pricing` | object | No | Pricing configuration. |
| `pricing.unit` | string | No | Pricing unit: `token`, `char`, `request`. |
| `pricing.prompt` | string | No | Cost per unit for prompt tokens. |
| `pricing.completion` | string | No | Cost per unit for completion tokens. |
| `limits` | object | No | Default deployment-level limits. |
| `userRoles` | array | No | Roles that can access this model. |
| `forwardAuthToken` | boolean | No | Forward the user's auth token to the model endpoint. |
| `overrideName` | string | No | Override the deployment name sent to the upstream. |
| `interceptors` | array | No | Local interceptors applied to this model. |
| `defaults` | object | No | Default parameter values (e.g., `max_tokens`). |

### Upstream configuration

Each entry in a model's `upstreams` array:

| Parameter | Type | Required | Description |
|---|---|---|---|
| `endpoint` | string | Yes | Provider endpoint URL. |
| `key` | string | No | API key for the provider endpoint. |
| `weight` | integer | No | Load balancing weight across multiple upstreams. |
| `maxRetryAttempts` | integer | No | Maximum retry attempts for this upstream. |

### Applications

Each application is defined under `applications.<application_name>`.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `endpoint` | string | Yes | Application endpoint URL. |
| `displayName` | string | No | Display name. |
| `displayVersion` | string | No | Display version. |
| `iconUrl` | string | No | Icon URL. |
| `description` | string | No | Description. |
| `inputAttachmentTypes` | array | No | Accepted input MIME types. |
| `maxInputAttachments` | integer | No | Maximum input attachments. |
| `features` | object | No | Feature flags (same schema as models). |
| `defaults` | object | No | Default parameter values. |
| `interceptors` | array | No | Local interceptors. |
| `userRoles` | array | No | Roles that can access this application. |
| `forwardAuthToken` | boolean | No | Forward the user's auth token to the application endpoint. |
| `reference` | object | No | Reference to an application type schema. |

### Interceptors

Interceptors transform requests before they reach a model or application and responses before they reach the client. They are defined under `interceptors.<interceptor_name>`.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `endpoint` | string | Yes | Interceptor endpoint URL. |
| `displayName` | string | No | Display name. |
| `description` | string | No | Description. |

Interceptors can be applied at three levels:

- **Global** (`globalInterceptors`): Applied to all deployments. Defined as a top-level array.
- **Application type** (`dial:applicationTypeInterceptors` in application type schemas): Applied to all instances of a schema-rich app type.
- **Local** (`interceptors` on a model or application): Applied to a specific deployment.

Execution order — request: global, then application type, then local. Response: local, then application type, then global.

### Roles

Each role is defined under `roles.<role_name>`.

| Parameter | Type | Description |
|---|---|---|
| `limits.<deployment>` | object | Rate limits for a specific deployment. |
| `limits.<deployment>.requestHour` | integer | Maximum requests per hour. |
| `limits.<deployment>.requestDay` | integer | Maximum requests per day. |
| `limits.<deployment>.minute` | integer | Maximum tokens per minute (floating window). |
| `limits.<deployment>.day` | integer | Maximum tokens per day (floating window). |
| `limits.<deployment>.week` | integer | Maximum tokens per week (floating window). |
| `limits.<deployment>.month` | integer | Maximum tokens per month (floating window). |
| `costLimit.minute` | string | Cost limit per minute in USD across all models. |
| `costLimit.day` | string | Cost limit per day in USD. |
| `costLimit.week` | string | Cost limit per week in USD. |
| `costLimit.month` | string | Cost limit per month in USD. |
| `share.<RESOURCE_TYPE>.invitation_ttl` | integer | TTL of invitation link in hours. Default: 72. |
| `share.<RESOURCE_TYPE>.max_accepted_users` | integer | Maximum users who can accept an invite. Default: 10 for `APPLICATION`, unlimited for others. |

If limits are not defined for a role, limits from the `default` role apply. If the `default` role has no limits, values are unlimited.

### Keys

Each API key is defined under `keys.<key_name>`.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `project` | string | Yes | Project name assigned to this key. |
| `role` | string | Conditional | A role assigned to the key. At least one of `role` or `roles` is required. |
| `roles` | array | Conditional | List of roles. At least one of `role` or `roles` is required. |
| `secured` | boolean | No | When `true`, request/response is not saved to prompt log storage. |
| `allowedIpAddressRanges` | array | No | IP ranges for access control. `null`/undefined = any IP; `[]` = no access. Supports IPv4 and IPv6 CIDR. |

### Routes

Routes proxy requests to upstream services. Each route is defined under `routes.<route_name>`.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `paths` | array | — | List of path patterns (plain string or regex). |
| `methods` | array | — | HTTP methods: `GET`, `POST`, `PUT`, `DELETE`, `HEAD`, `PATCH`. |
| `rewritePath` | boolean | — | Replace upstream path with original request path. |
| `userRoles` | array | all users | Roles that can access this route. |
| `response` | object | — | Pre-configured response with `status` and `body`. Mutually exclusive with `upstreams`. |
| `maxRetryAttempts` | integer | — | Maximum retries on upstream failure. |
| `order` | integer | `2147483647` | Priority (lower = higher). Cannot be negative. |
| `upstreams` | array | — | Upstream servers. Same schema as model upstreams; also supports WebSocket endpoints (`ws://`, `wss://`). |

### Toolsets

Toolsets provide MCP (Model Context Protocol) tool integrations. Each toolset is defined under `toolsets.<toolset_name>`.

| Parameter | Type | Required | Description |
|---|---|---|---|
| `endpoint` | string | Yes | Toolset API endpoint. |
| `transport` | string | No | Transport protocol: `HTTP` or `SSE`. |
| `displayName` | string | No | Display name. |
| `description` | string | No | Description. |
| `iconUrl` | string | No | Icon URL. |
| `userRoles` | array | No | Roles that can access this toolset. |
| `descriptionKeywords` | array | No | Keywords for discovery. |
| `maxRetryAttempts` | integer | No | Maximum retry attempts. |
| `allowedTools` | array | No | Available tools in the MCP server. |
| `forwardPerRequestKey` | boolean | No | Forward per-request API key to the toolset endpoint. Cannot be `true` when `authType` is `API_KEY`. |

Toolsets support `OAUTH`, `API_KEY`, and `NONE` authentication types with credential management via the `/v1/ops/toolset/signin` and `/v1/ops/toolset/signout` API endpoints.

### Application type schemas

Application type schemas define custom application types with their own UI editors and endpoints. They are defined as a top-level `applicationTypeSchemas` array containing JSON schemas conforming to the meta-schema `https://dial.epam.com/application_type_schemas/schema#`.

Key DIAL-specific properties in the schema:

| Property | Type | Description |
|---|---|---|
| `dial:applicationTypeDisplayName` | string | Display name for the application type. Required. |
| `dial:applicationTypeCompletionEndpoint` | string (URI) | Chat completion endpoint URL. |
| `dial:applicationTypeResponsesEndpoint` | string (URI) | Responses API endpoint URL. |
| `dial:applicationTypeSchemaEndpoint` | string (URI) | JSON schema endpoint URL. |
| `dial:applicationTypeEditorUrl` | string (URI) | Editor UI URL. |
| `dial:applicationTypeViewerUrl` | string (URI) | Viewer UI URL. |
| `dial:applicationTypeIconUrl` | string | Icon URL. |
| `dial:applicationTypeInterceptors` | array | Interceptors applied to all instances of this type. |

### Other top-level dynamic settings

| Setting | Type | Description |
|---|---|---|
| `globalInterceptors` | array | Interceptor names applied to all deployments. |
| `retriableErrorCodes` | array | HTTP error codes that trigger upstream retries. |

The file path is set by the `aidial.config.files` static setting:

```yaml
core:
  environment:
    'aidial.config.files': '["/opt/config/config.json"]'
  volumes:
    - ./core:/opt/config
```
