# Configuration documentation coverage

This page tracks the current state of DIAL configuration documentation — what is fully covered, what has known gaps, and what is planned. It exists to prevent readers from searching for documentation that does not yet exist.

These configuration areas have dedicated reference pages with verified settings, working examples, and Docker Compose context:

| Topic | Page | Source verified against |

## Partially documented

These areas are covered but have known gaps:

| Topic | What exists | What is missing |
|---|---|---|
| **Helm chart `values.yaml`** | Overview and examples in the configuration files page. Component pages reference Helm `env` and `secrets` patterns. | Full parameter reference for the Helm chart itself. The definitive source is the [DIAL Helm repository](https://github.com/epam/ai-dial-helm). A dedicated Helm configuration page is planned. |
| **DIAL RAG configuration** | Basic `DIAL_URL` documented in adapter and Docker Compose examples. | The nested `DIAL_RAG__*` environment variable convention (e.g., `DIAL_RAG__INDEX_STORAGE__USE_DIAL_FILE_STORAGE`, `DIAL_RAG__REQUEST__QA_CHAIN__CHAT_CHAIN__LLM__DEPLOYMENT_NAME`). Requires its own reference page. |
| **Interceptors configuration** | Interceptor *registration* documented in DIAL Core dynamic settings (endpoint, display name, global interceptors). | SDK-side configuration for building interceptors is a developer topic, covered under Building with DIAL rather than here. |
| **OpenAI Adapter advanced features** | Env vars and basic upstream examples documented. | Detailed operational guides for: configurable models (DALL-E parameters, Responses API reasoning configuration), tokenization algorithms, and API versioning strategies. These are documented in the adapter's README but not yet fully migrated. |

## Not yet documented

These configuration areas are planned but do not have pages yet:

| Topic | Planned location | Blocking on |
|---|---|---|
| **Load balancer configuration** | Operating DIAL > Configuration Reference | Research into [`epam/ai-dial-core`](https://github.com/epam/ai-dial-core) load balancer module |
| **Observability configuration** | Operating DIAL > Observability | OpenTelemetry exporter setup, metrics endpoints, tracing configuration, collector → backend pipeline |
| **Code Interpreter configuration** | DIAL Core configuration (expansion) | Docker image settings, session TTL, pod deployment, proxy URL patterns |
| **DIAL-to-DIAL Adapter** | Operating DIAL > Local Setup | [`epam/ai-dial-adapter-dial`](https://github.com/epam/ai-dial-adapter-dial) configuration for local dev against remote DIAL Core |
| **Secrets management patterns** | Operating DIAL > Production Readiness | Integration with HashiCorp Vault, AWS Secrets Manager, Azure Key Vault for production secret injection |
| **Multi-environment strategies** | Operating DIAL > Production Readiness | Patterns for managing dev/staging/prod configuration drift, config.json per environment, Helm values overlays |
| **Analytics Realtime advanced configuration** | Operating DIAL > Observability | Beyond the existing how-to — InfluxDB setup, retention policies, custom dashboards |
| **Themes configuration** | Building with DIAL > Chat Customization | Custom themes, image assets, Docker image customization |

## How this page is maintained

This page is updated each time configuration documentation is added or revised. The `last_verified` date in each reference page's frontmatter indicates when its content was last cross-checked against the source repository.
