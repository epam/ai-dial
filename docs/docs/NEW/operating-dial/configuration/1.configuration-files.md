# Configuration files

A DIAL deployment uses several configuration files across its components. This page provides a quick-reference inventory of every file, its format, and where it is loaded. All examples are taken from working Docker Compose setups in the [AI DIAL repository](https://github.com/epam/ai-dial).

## File inventory

| File | Component | Format | Mount point | Loaded |
|---|---|---|---|---|
| `settings.json` | DIAL Core | JSON | `/opt/settings/settings.json` | At startup (static — requires restart to change) |
| `config.json` | DIAL Core | JSON | `/opt/config/config.json` | Every 60 seconds (dynamic — hot-reloaded, no restart needed) |
| `gflog.xml` | DIAL Core | XML | `/opt/settings/gflog.xml` | At startup via `JAVA_OPTS` |
| `.env` | Docker Compose | Key=Value | N/A (compose substitution) | At `docker compose up` |
| `realm-config.json` | Keycloak | JSON | `/opt/keycloak/data/import/` | At Keycloak startup (`--import-realm`) |
| `application.yaml` | Auth Helper | YAML | Bundled in container | At startup |
| `values.yaml` | Helm chart | YAML | N/A (Helm input) | At `helm install` / `helm upgrade` |

## settings.json — DIAL Core static settings

DIAL Core reads static settings from a JSON file at startup. The file path is set by the `AIDIAL_SETTINGS` environment variable. If not set, DIAL Core falls back to the bundled `aidial.settings.json` resource file. Static settings can also be passed as environment variables with the `aidial.` prefix.

See [DIAL Core configuration](core-configuration) for the complete settings reference, working examples, and configuration precedence rules.

## config.json — DIAL Core dynamic settings

Dynamic settings define models, applications, interceptors, roles, API keys, routes, and tool sets. DIAL Core reloads these files at a configurable interval (default: 60 seconds), so changes take effect without restarting the service. The file path is set by the `aidial.config.files` static setting. Dynamic settings can also be managed through [DIAL Admin](/tutorials/admin/home).

See [DIAL Core configuration](core-configuration) for the complete dynamic settings reference and working examples for echo apps, Azure models, Ollama, and production setups.

## gflog.xml — DIAL Core logging

DIAL Core uses the [gflog](https://github.com/epam/gflog) logging framework. The configuration file is passed via the `JAVA_OPTS` environment variable. See [Supporting services](supporting-services#logging-gflogxml) for the full logging configuration reference and examples.

## .env files — Docker Compose variable substitution

Docker Compose setups use `.env` files for environment variable substitution. Variables defined in `.env` are available as `${VAR}` in `docker-compose.yml`.

### Variable categories

| Category | Variables | Description |
|---|---|---|
| Ports | `DIAL_CHAT_HOST_PORT`, `DIAL_CORE_HOST_PORT`, `DIAL_ADMIN_FRONTEND_HOST_PORT`, `KC_PORT` | Host ports for each service. Allows running multiple DIAL instances on different ports. |
| Networking | `COMMON_HOST` | Hostname used in URLs for inter-service communication and Keycloak issuer configuration. |
| Auth secrets | `NEXTAUTH_SECRET`, `AUTH_KEYCLOAK_SECRET` | NextAuth session encryption key and Keycloak client secret. Both are required for authenticated deployments. |
| Encryption | `DIAL_CORE_ENCRYPTION_SECRET`, `DIAL_CORE_ENCRYPTION_KEY` | AES encryption parameters for DIAL Core blob storage prefix hashing. |
| Keycloak | `KC_ADMIN_PASSWORD` | Initial Keycloak admin password. |
| Database | `H2_DATASOURCE_ENCRYPTED_FILE_KEY`, `H2_DATASOURCE_MASTER_KEY`, `H2_DATASOURCE_PASSWORD` | Admin Backend H2 database encryption and access credentials. |

The advanced Docker Compose setup uses `.env` for secrets and host-specific values:

```ini
# Ports
DIAL_CHAT_HOST_PORT=3000
DIAL_CORE_HOST_PORT=8080
DIAL_ADMIN_FRONTEND_HOST_PORT=3100
KC_PORT=8180
COMMON_HOST=localhost

# Secrets
NEXTAUTH_SECRET=<generate-with-openssl-rand-base64-32>
AUTH_KEYCLOAK_SECRET=<keycloak-client-secret>
DIAL_CORE_ENCRYPTION_SECRET=<random-string>
DIAL_CORE_ENCRYPTION_KEY=<random-string>
KC_ADMIN_PASSWORD=<keycloak-admin-password>
H2_DATASOURCE_ENCRYPTED_FILE_KEY=<random-string>
H2_DATASOURCE_MASTER_KEY=<random-string>
H2_DATASOURCE_PASSWORD=<random-string>
```

## realm-config.json — Keycloak identity realm

When DIAL uses Keycloak for authentication, the realm configuration defines clients, roles, and users. Keycloak imports this file at startup via the `--import-realm` flag. See [Supporting services](supporting-services#keycloak) for the full Keycloak configuration, realm structure, and examples.
