# Server & Client

## Server

Server settings configure the DIAL Core HTTP listener.

### All server settings

```json
{
  "server": {
    "port": 8080,
    "acceptBacklog": 4096,
    "idleTimeout": 300000,
    "compressionSupported": true
  }
}
```

:::note
Additional Vert.x `HttpServerOptions` can be passed under `server.*`.
:::

### `server.port`

HTTP port the server listens on.

**Default:** `8080`

### `server.acceptBacklog`

TCP accept backlog queue size. Increase for high-connection-rate deployments.

**Default:** `4096`

### `server.idleTimeout`

Maximum idle connection time in milliseconds. Applies to HTTP/1.1 keep-alive and HTTP/2 connections.

**Default:** `300000` (5 minutes)

### `server.compressionSupported`

Enable HTTP response compression.

**Default:** `true`

## Client

Client settings configure outbound HTTP connections to adapters and upstreams.

### All client settings

```json
{
  "client": {
    "idleTimeout": 300000,
    "connectTimeout": 10000,
    "keepAlive": true,
    "maxPoolSize": 4096
  }
}
```

:::note
Additional Vert.x `HttpClientOptions` can be passed under `client.*`. WebSocket client settings are available under `webSocketClient.*`.
:::

### `client.idleTimeout`

Idle timeout for outbound connections in milliseconds.

**Default:** `300000` (5 minutes)

### `client.connectTimeout`

Connection timeout for outbound connections in milliseconds.

**Default:** `10000` (10 seconds)

### `client.keepAlive`

Enable HTTP keep-alive for connection reuse.

**Default:** `true`

### `client.maxPoolSize`

Maximum connection pool size for outbound requests.

**Default:** `4096`
