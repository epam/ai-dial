# Redis

DIAL Core requires Redis for in-memory caching of resources, sessions, and rate-limit counters. Redis is configured on two levels: the DIAL Core connection settings in `settings.json` and the Redis server itself.

## Connection settings

Either single server or cluster configuration must be provided in `settings.json`.

### `redis.singleServerConfig.address`

Redis single server address.

```json
{
  "redis": {
    "singleServerConfig": {
      "address": "redis://redis:6379"
    }
  }
}
```

### `redis.clusterServersConfig.nodeAddresses`

Array of Redis cluster node addresses.

```json
{
  "redis": {
    "clusterServersConfig": {
      "nodeAddresses": [
        "redis://host1:6379",
        "redis://host2:6379"
      ]
    }
  }
}
```

### Cloud-managed Redis

For cloud-managed Redis with IAM authentication:

#### `redis.provider.name`

Cloud provider: `aws-elasti-cache`, `gcp-memory-store`, or `azure-redis-cache`.

#### `redis.provider.userId`

IAM-enabled user ID (AWS ElastiCache).

#### `redis.provider.accountName`

Service account resource name in `projects/-/serviceAccounts/{ACCOUNT}` format (GCP).

#### `redis.provider.region`

Geo region (AWS ElastiCache).

#### `redis.provider.clusterName`

Redis cluster name (AWS ElastiCache).

#### `redis.provider.serverless`

Whether the cache is serverless (AWS ElastiCache).

## Redis server configuration

The Redis server itself is configured via command-line arguments, typically in a Docker Compose file.

From [`dial-docker-compose/common.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/common.yml):

```yaml
redis:
  image: redis:7.2.4-alpine3.19
  restart: always
  command: >
    redis-server
    --maxmemory 2000mb
    --maxmemory-policy volatile-lfu
    --save ""
    --appendonly no
    --loglevel warning
  mem_limit: 2200M
```

| Argument | Value | Description |
|---|---|---|
| `--maxmemory` | `2000mb` | Maximum memory Redis is allowed to use. Set slightly below the container memory limit. |
| `--maxmemory-policy` | `volatile-lfu` | Eviction policy. `volatile-lfu` evicts the least-frequently-used keys among those with an expiration set. |
| `--save ""` | (empty) | Disables RDB persistence. DIAL uses Redis as a cache, not a primary datastore â€” persistence is not required. |
| `--appendonly no` | â€” | Disables AOF persistence. |
| `--loglevel warning` | â€” | Suppresses INFO and DEBUG log output. |
