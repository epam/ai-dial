# Storage

DIAL Core stores files (conversation history, uploaded attachments, application data) in blob storage. Storage settings are defined under `storage.*` in `settings.json`.

### `storage.provider`

Blob storage provider.

**Required**

| Value | Description |
|-------|-------------|
| `filesystem` | Local filesystem (development) |
| `s3` | S3-compatible storage (MinIO, Ceph) |
| `aws-s3` | AWS S3 with IAM authentication |
| `azureblob` | Azure Blob Storage |
| `google-cloud-storage` | Google Cloud Storage |

```json
{
  "storage": {
    "provider": "filesystem"
  }
}
```

### `storage.endpoint`

Endpoint URL for S3-compatible storage.

**Required if** using `s3` provider

```json
{
  "storage": {
    "endpoint": "http://minio:9000"
  }
}
```

### `storage.identity`

Storage access key.

**Required if** using `s3` or `azureblob` with key-based auth

### `storage.credential`

Storage secret key.

**Required if** using `s3` or `azureblob` with key-based auth

S3 authentication example:

```json
{
  "storage": {
    "provider": "s3",
    "endpoint": "http://minio:9000",
    "identity": "minioadmin",
    "credential": "minioadmin"
  }
}
```

### `storage.bucket`

Bucket name for storing resources.

```json
{
  "storage": {
    "bucket": "dial"
  }
}
```

### `storage.createBucket`

Create the bucket on startup if it does not exist.

**Default:** `false`

### `storage.prefix`

Base prefix for all stored resources. Enables multiple DIAL environments to share a bucket.

```json
{
  "storage": {
    "prefix": "production"
  }
}
```

> **Note:** Must not contain path separators.

### `storage.maxUploadedFileSize`

Maximum uploaded file size in bytes. Returns HTTP 413 if exceeded.

**Default:** `536870912` (512 MB)

### `storage.overrides`

Key-value pairs to override storage settings (e.g., jclouds properties).

```json
{
  "storage": {
    "overrides": {
      "jclouds.filesystem.basedir": "data"
    }
  }
}
```

## Filesystem example

A complete filesystem storage configuration:

```json
{
  "storage": {
    "provider": "filesystem",
    "prefix": "core",
    "bucket": "dial",
    "createBucket": true,
    "overrides": {
      "jclouds.filesystem.basedir": "data"
    }
  }
}
```
