# Access Control

Access control settings in `settings.json` define admin rules and application permissions.

## Admin access

### `access.admin.rules`

Rules to match JWT claims for admin access. Each rule has:

- `function`: `TRUE`, `FALSE`, `EQUAL`, `CONTAIN`, `REGEX`
- `source`: claim path
- `targets`: array of values

```json
{
  "access": {
    "admin": {
      "rules": [
        {
          "source": "roles",
          "function": "EQUAL",
          "targets": ["admin"]
        }
      ]
    }
  }
}
```

### `access.createCodeAppRoles`

Roles allowed to create custom code applications or run the code interpreter.

```json
{
  "access": {
    "createCodeAppRoles": ["developer", "admin"]
  }
}
```

---

## Applications

### `applications.includeCustomApps`

Include custom applications in the OpenAI listing endpoint. Required for Code Apps, Quick Apps, and custom applications.

**Default:** `false`

```json
{
  "applications": {
    "includeCustomApps": true
  }
}
```
