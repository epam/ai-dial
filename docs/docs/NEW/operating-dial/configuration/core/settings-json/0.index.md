# settings.json

`settings.json` is DIAL Core's static configuration file. It is loaded once at startup — changes require a restart.

## File location

The file path is set by the `AIDIAL_SETTINGS` environment variable. If not set, DIAL Core falls back to the bundled `aidial.settings.json` resource file.

In Docker Compose deployments, the file is mounted as a volume:

```yaml
core:
  environment:
    AIDIAL_SETTINGS: '/opt/settings/settings.json'
  volumes:
    - ./settings:/opt/settings
```

## Configuration precedence

When the same setting is defined in multiple places:

1. **Environment variables** with the `aidial.` prefix (highest priority)
2. **Settings file** specified by `AIDIAL_SETTINGS`
3. **Default resource file** bundled with DIAL Core (`aidial.settings.json`)

For example, `aidial.server.port` as an environment variable overrides `server.port` in the JSON file.

## Minimal example

From [`dial-docker-compose/settings/settings.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/settings/settings.json):

```json
{
  "server": {
    "port": 8080
  },
  "identityProviders": {
    "test": {
      "rolePath": "roles",
      "disableJwtVerification": true
    }
  },
  "encryption": {
    "secret": "salt",
    "key": "password"
  }
}
```
