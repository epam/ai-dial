# Identity Providers

At least one identity provider must be configured for DIAL Core to authenticate requests. Providers are defined in `settings.json` under `identityProviders.<name>` where `<name>` is your chosen identifier (e.g., `keycloak`, `azure`, `okta`).

## Provider attributes

### `identityProviders.<name>.jwksUrl`

URL to the JSON Web Key Set (JWKS) endpoint for validating JWT signatures. DIAL Core fetches public keys from this endpoint and caches them.

**Required if** `disableJwtVerification` is `false` (the default)

```json
{
  "identityProviders": {
    "keycloak": {
      "jwksUrl": "http://keycloak:8180/realms/dial/protocol/openid-connect/certs"
    }
  }
}
```

> **Note:** Either `jwksUrl` or `userInfoEndpoint` must be provided.

### `identityProviders.<name>.userInfoEndpoint`

URL to user info endpoint. Alternative to `jwksUrl` for providers that don't expose a JWKS endpoint.

**Required if** `jwksUrl` is not provided

### `identityProviders.<name>.rolePath`

Path to the claim containing user roles in the JWT payload or user info response. Supports dot notation for nested claims.

**Required**

Simple claim at root level:

```json
{
  "identityProviders": {
    "keycloak": {
      "rolePath": "roles"
    }
  }
}
```

Nested claim (Azure AD example):

```json
{
  "identityProviders": {
    "azure": {
      "rolePath": "resource_access.dial.roles"
    }
  }
}
```

> **Tip:** Can be a single string or array of strings for multiple possible paths.

### `identityProviders.<name>.projectPath`

Path to the claim from which the project name is taken (e.g., `azp`, `aud`).

```json
{
  "identityProviders": {
    "keycloak": {
      "projectPath": "azp"
    }
  }
}
```

### `identityProviders.<name>.rolesDelimiter`

Delimiter to split roles when they come as a single string instead of an array.

```json
{
  "identityProviders": {
    "custom": {
      "rolesDelimiter": ","
    }
  }
}
```

### `identityProviders.<name>.loggingKey`

Claim name for user identification in logs. Common values: `email`, `sub`, `preferred_username`.

```json
{
  "identityProviders": {
    "keycloak": {
      "loggingKey": "email"
    }
  }
}
```

### `identityProviders.<name>.loggingSalt`

Salt to hash user information for logging. When set, the `loggingKey` value is hashed before writing to logs.

```json
{
  "identityProviders": {
    "keycloak": {
      "loggingSalt": "random-salt-string"
    }
  }
}
```

> **Privacy:** Use this to avoid storing PII in logs while maintaining user traceability.

### `identityProviders.<name>.issuerPattern`

Regular expression to match the `iss` claim to this identity provider. Useful when multiple providers share similar issuers.

```json
{
  "identityProviders": {
    "azure": {
      "issuerPattern": "https://login.microsoftonline.com/.*"
    }
  }
}
```

### `identityProviders.<name>.disableJwtVerification`

Disable JWT signature verification. `userInfoEndpoint` must be unset when `true`.

**Default:** `false`

> **Warning:** For development only. Never disable in production.

```json
{
  "identityProviders": {
    "test": {
      "disableJwtVerification": true
    }
  }
}
```

### `identityProviders.<name>.audience`

When set, validated against the `aud` claim in the JWT. Rejects tokens not intended for DIAL.

```json
{
  "identityProviders": {
    "keycloak": {
      "audience": "dial-core"
    }
  }
}
```

### `identityProviders.<name>.userDisplayName`

Path to claim for user display name shown in conversations and audit logs.

```json
{
  "identityProviders": {
    "keycloak": {
      "userDisplayName": "name"
    }
  }
}
```

### `identityProviders.<name>.userIdPath`

Path to claim for user ID. Used for identifying users across sessions.

**Default:** `sub`

```json
{
  "identityProviders": {
    "azure": {
      "userIdPath": "oid"
    }
  }
}
```

### `identityProviders.<name>.positiveCacheExpirationMs`

JWKS cache TTL on successful fetch in milliseconds.

**Default:** `600000` (10 minutes)

### `identityProviders.<name>.negativeCacheExpirationMs`

JWKS cache TTL on failed fetch in milliseconds.

**Default:** `10000` (10 seconds)

## Examples

### Development (JWT verification disabled)

From [`dial-docker-compose/settings/settings.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/settings/settings.json):

```json
{
  "identityProviders": {
    "test": {
      "rolePath": "roles",
      "disableJwtVerification": true
    }
  }
}
```

This setup allows any JWT-formatted token without signature verification. Use only for local development.

### Production (Keycloak)

```json
{
  "identityProviders": {
    "keycloak": {
      "jwksUrl": "http://keycloak:8180/realms/dial/protocol/openid-connect/certs",
      "rolePath": "roles",
      "loggingKey": "email",
      "loggingSalt": "loggingSalt"
    }
  }
}
```

### Multiple providers

Configure multiple providers when users authenticate through different identity systems:

```json
{
  "identityProviders": {
    "internal": {
      "jwksUrl": "http://keycloak:8180/realms/dial/protocol/openid-connect/certs",
      "rolePath": "roles",
      "issuerPattern": "http://keycloak:8180/realms/dial"
    },
    "external": {
      "jwksUrl": "https://login.microsoftonline.com/tenant-id/discovery/v2.0/keys",
      "rolePath": "roles",
      "issuerPattern": "https://login.microsoftonline.com/.*",
      "userIdPath": "oid"
    }
  }
}
```

DIAL Core matches the `iss` claim in incoming tokens against `issuerPattern` to determine which provider configuration to use.
