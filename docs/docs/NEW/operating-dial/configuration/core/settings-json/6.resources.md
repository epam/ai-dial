# Resources

Resource settings control caching and sync behavior for stored objects in `settings.json`.

### All resource settings

```json
{
  "resources": {
    "maxSize": 67108864,
    "maxSizeToCache": 1048576,
    "syncPeriod": 60000,
    "syncDelay": 120000,
    "syncBatch": 4096,
    "cacheExpiration": 300000,
    "compressionMinSize": 256,
    "heartbeatPeriod": 60000
  }
}
```

### `resources.maxSize`

Maximum resource size in bytes. Returns HTTP 413 if exceeded.

**Default:** `67108864` (64 MB)

### `resources.maxSizeToCache`

Maximum resource size to cache in Redis.

**Default:** `1048576` (1 MB)

### `resources.syncPeriod`

Period in milliseconds for checking resources to sync to object storage.

**Default:** `60000` (1 minute)

### `resources.syncDelay`

Delay in milliseconds before writing back to object storage after last modification.

**Default:** `120000` (2 minutes)

### `resources.syncBatch`

Number of resources to sync in one batch.

**Default:** `4096`

### `resources.cacheExpiration`

Expiration in milliseconds for synced resources in Redis.

**Default:** `300000` (5 minutes)

### `resources.compressionMinSize`

Compress resources with gzip if size is at or above this value (bytes).

**Default:** `256`

### `resources.heartbeatPeriod`

Heartbeat period in milliseconds.

**Default:** `60000`

### Per-resource-type expiration

| Setting | Default |
|---------|---------|
| `resources.resourceTypesExpiration.FILE` | `300000` |
| `resources.resourceTypesExpiration.CONVERSATION` | `300000` |
| `resources.resourceTypesExpiration.PROMPT` | `300000` |
| `resources.resourceTypesExpiration.APPLICATION` | `Infinity` |
| `resources.resourceTypesExpiration.TOOL_SET` | `Infinity` |

```json
{
  "resources": {
    "resourceTypesExpiration": {
      "FILE": 300000,
      "CONVERSATION": 300000,
      "PROMPT": 300000,
      "APPLICATION": "Infinity",
      "TOOL_SET": "Infinity"
    }
  }
}
```
