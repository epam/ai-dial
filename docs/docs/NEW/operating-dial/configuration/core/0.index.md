# DIAL Core configuration

DIAL Core ([`epam/ai-dial-core`](https://github.com/epam/ai-dial-core)) is the central component of DIAL. It uses three configuration files:

| File | Format | Purpose | Reload behavior |
|------|--------|---------|-----------------|
| [`settings.json`](settings-json/index) | JSON | Server, storage, Redis, identity providers, encryption, access control | Static — loaded at startup, requires restart |
| [`config.json`](config-json/index) | JSON | Models, applications, interceptors, roles, API keys, routes, tool sets | Dynamic — hot-reloaded every 60 seconds |
| [`gflog.xml`](gflog-xml) | XML | Logging framework configuration (appenders, levels, rotation) | Static — loaded at startup via `JAVA_OPTS` |

Static settings can also be passed as environment variables with the `aidial.` prefix (e.g., `aidial.server.port`). Environment variables take precedence over the settings file.
