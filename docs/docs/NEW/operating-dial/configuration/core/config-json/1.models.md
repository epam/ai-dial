# Models configuration

Models are the primary deployments in DIAL Core. Each model entry defines how DIAL routes requests to a specific LLM or embedding service.

Models are defined in `config.json` under `models.<deployment_name>` where `<deployment_name>` is the identifier used in API requests (e.g., `gpt-4o`, `claude-3-sonnet`).

## Model attributes

### `type`

The model category. Determines which API endpoints are exposed.

**Required**

| Value | Description |
|-------|-------------|
| `chat` | Chat completion model (`/chat/completions` endpoint) |
| `embedding` | Embedding model (`/embeddings` endpoint) |

```json
{
  "models": {
    "gpt-4o": {
      "type": "chat"
    }
  }
}
```

### `endpoint`

The adapter URL that handles requests for this model.

**Required**

```json
{
  "models": {
    "gpt-4o": {
      "type": "chat",
      "endpoint": "http://adapter-openai:5000/openai/deployments/gpt-4o/chat/completions"
    }
  }
}
```

> **Self-hosted models:** For OpenAI-compatible endpoints (Ollama, vLLM), you can point directly to the model server without an adapter.

### `displayName`

Human-readable name shown in DIAL Chat and Admin Panel.

```json
{
  "displayName": "GPT-4o"
}
```

### `displayVersion`

Version string shown in the UI alongside the display name.

```json
{
  "displayName": "GPT-4o",
  "displayVersion": "2024-05-13"
}
```

### `iconUrl`

URL to the model's icon image.

```json
{
  "iconUrl": "https://cdn.example.com/icons/gpt4o.png"
}
```

### `description`

Model description shown in the UI.

```json
{
  "description": "OpenAI's most capable multimodal model"
}
```

### `upstreams`

List of upstream provider endpoints. Required when the adapter needs to connect to external APIs.

See [Upstream configuration](#upstream-configuration) below.

### `features`

Feature flags for the model. See [Features](#features) below.

### `inputAttachmentTypes`

MIME types the model accepts as input attachments.

```json
{
  "inputAttachmentTypes": ["image/png", "image/jpeg"]
}
```

Use `["*/*"]` to accept all attachment types:

```json
{
  "inputAttachmentTypes": ["*/*"]
}
```

### `maxInputAttachments`

Maximum number of input attachments allowed per request.

```json
{
  "maxInputAttachments": 10
}
```

### `pricing`

Pricing configuration for cost tracking.

#### `pricing.unit`

Pricing unit: `token`, `char`, or `request`.

#### `pricing.prompt`

Cost per unit for prompt/input tokens.

#### `pricing.completion`

Cost per unit for completion/output tokens.

```json
{
  "pricing": {
    "unit": "token",
    "prompt": "0.005",
    "completion": "0.015"
  }
}
```

### `limits`

Default deployment-level limits. Overridden by role-specific limits.

```json
{
  "limits": {
    "requestHour": 100
  }
}
```

### `userRoles`

Roles that can access this model. When omitted, the model is available to all authenticated users.

```json
{
  "userRoles": ["premium", "admin"]
}
```

### `forwardAuthToken`

Forward the user's auth token to the model endpoint.

**Default:** `false`

```json
{
  "forwardAuthToken": true
}
```

### `overrideName`

Override the deployment name sent to the upstream. Useful when the upstream expects a different model name.

```json
{
  "overrideName": "gpt-4-turbo-preview"
}
```

### `interceptors`

Local interceptors applied to this model. See [Interceptors](interceptors).

```json
{
  "interceptors": ["content-filter", "pii-redactor"]
}
```

### `defaults`

Default parameter values sent with every request.

```json
{
  "defaults": {
    "max_tokens": 4096,
    "temperature": 0.7
  }
}
```

## Upstream configuration

Each entry in a model's `upstreams` array configures a connection to a provider endpoint.

### `upstreams[].endpoint`

The provider API URL.

**Required**

```json
{
  "upstreams": [
    {
      "endpoint": "https://my-resource.openai.azure.com/openai/deployments/gpt-4o/chat/completions"
    }
  ]
}
```

### `upstreams[].key`

API key for authenticating with the provider.

```json
{
  "upstreams": [
    {
      "endpoint": "https://...",
      "key": "sk-..."
    }
  ]
}
```

> **Secrets:** In production, inject keys via environment variables or a secrets manager rather than storing them in `config.json`.

### `upstreams[].weight`

Load balancing weight across multiple upstreams. Higher values receive proportionally more traffic.

**Default:** `1`

```json
{
  "upstreams": [
    { "endpoint": "https://region1.azure.com/...", "weight": 2 },
    { "endpoint": "https://region2.azure.com/...", "weight": 1 }
  ]
}
```

### `upstreams[].maxRetryAttempts`

Maximum retry attempts for this upstream before failing over to the next.

```json
{
  "upstreams": [
    {
      "endpoint": "https://...",
      "maxRetryAttempts": 3
    }
  ]
}
```

## Features

Feature flags customize model behavior. Defined under `features` within a model entry.

### `features.rateEndpoint`

Endpoint for rate/usage requests.

### `features.tokenizeEndpoint`

Endpoint for tokenization requests.

### `features.truncatePromptEndpoint`

Endpoint for prompt truncation.

### `features.systemPromptSupported`

Whether the model supports system prompts.

**Default:** `true`

```json
{
  "features": {
    "systemPromptSupported": false
  }
}
```

> **Note:** Set to `false` for models like DALL-E that ignore system messages.

### `features.toolsSupported`

Whether the model supports tool/function calling.

**Default:** `false`

### `features.seedSupported`

Whether the model supports seed for reproducibility.

### `features.urlAttachmentsSupported`

Whether the model can fetch and process URL attachments.

### `features.folderAttachmentsSupported`

Whether the model can process folder attachments.

## Examples

### Azure OpenAI model

From [`dial-docker-compose/model/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/model/core/config.json):

```json
{
  "models": {
    "gpt-4o": {
      "type": "chat",
      "displayName": "GPT-4o",
      "endpoint": "http://adapter-openai:5000/openai/deployments/gpt-4o/chat/completions",
      "upstreams": [
        {
          "endpoint": "https://my-resource.openai.azure.com/openai/deployments/gpt-4o/chat/completions",
          "key": "AZURE_API_KEY"
        }
      ],
      "inputAttachmentTypes": ["*/*"]
    }
  }
}
```

### Self-hosted Ollama model

From [`dial-docker-compose/ollama/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/ollama/core/config.json):

```json
{
  "models": {
    "chat-model": {
      "type": "chat",
      "displayName": "Self-hosted chat model",
      "endpoint": "http://ollama:11434/v1/chat/completions"
    },
    "vision-model": {
      "type": "chat",
      "displayName": "Self-hosted vision model",
      "endpoint": "http://adapter-openai:5000/openai/deployments/vision-model/chat/completions",
      "inputAttachmentTypes": ["image/png", "image/jpeg"],
      "upstreams": [
        {
          "endpoint": "http://ollama:11434/v1/chat/completions",
          "key": "dummy-key"
        }
      ]
    },
    "embedding-model": {
      "type": "embedding",
      "endpoint": "http://ollama:11434/v1/embeddings"
    }
  }
}
```

> **No adapter needed:** Ollama's OpenAI-compatible endpoint can be used directly for simple models. Use the adapter when you need attachment processing or tokenization.

### DALL-E with pricing

From [`dial-docker-compose-advanced/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/core/config.json):

```json
{
  "models": {
    "dall-e-3": {
      "type": "chat",
      "displayName": "DALL-E 3",
      "features": {
        "systemPromptSupported": false
      },
      "pricing": {
        "unit": "token",
        "prompt": "0",
        "completion": "0.04"
      },
      "endpoint": "http://adapter-openai:5000/openai/deployments/dall-e-3/chat/completions",
      "upstreams": [
        {
          "endpoint": "https://my-resource.openai.azure.com/openai/deployments/dall-e-3/images/generations",
          "key": "AZURE_API_KEY"
        }
      ]
    }
  }
}
```

### Load balancing across regions

```json
{
  "models": {
    "gpt-4o": {
      "type": "chat",
      "displayName": "GPT-4o (Load Balanced)",
      "endpoint": "http://adapter-openai:5000/openai/deployments/gpt-4o/chat/completions",
      "upstreams": [
        {
          "endpoint": "https://eastus.openai.azure.com/openai/deployments/gpt-4o/chat/completions",
          "key": "${AZURE_EASTUS_KEY}",
          "weight": 2
        },
        {
          "endpoint": "https://westus.openai.azure.com/openai/deployments/gpt-4o/chat/completions",
          "key": "${AZURE_WESTUS_KEY}",
          "weight": 1
        }
      ]
    }
  }
}
```
