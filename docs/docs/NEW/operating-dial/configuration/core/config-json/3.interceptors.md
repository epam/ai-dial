# Interceptors configuration

Interceptors transform requests before they reach a model or application, and responses before they reach the client. Common use cases include content filtering, PII redaction, prompt injection, and audit logging.

Interceptors are defined in `config.json` under `interceptors.<interceptor_name>`.

## Interceptor attributes

### `endpoint`

The interceptor endpoint URL.

**Required**

```json
{
  "interceptors": {
    "content-filter": {
      "endpoint": "http://content-filter:5000/openai/deployments/filter/chat/completions"
    }
  }
}
```

### `displayName`

Human-readable name shown in Admin Panel.

```json
{
  "interceptors": {
    "content-filter": {
      "endpoint": "http://content-filter:5000/openai/deployments/filter/chat/completions",
      "displayName": "Content Safety Filter"
    }
  }
}
```

### `description`

Interceptor description.

```json
{
  "interceptors": {
    "content-filter": {
      "endpoint": "http://content-filter:5000/openai/deployments/filter/chat/completions",
      "description": "Filters harmful content from requests and responses"
    }
  }
}
```

## Application levels

Interceptors can be applied at three levels:

### Global interceptors

Applied to all deployments. Defined as a top-level `globalInterceptors` array.

```json
{
  "globalInterceptors": ["audit-logger", "rate-limiter"],
  "interceptors": {
    "audit-logger": {
      "endpoint": "http://audit:5000/intercept"
    },
    "rate-limiter": {
      "endpoint": "http://limiter:5000/intercept"
    }
  }
}
```

### Application type interceptors

Applied to all instances of a schema-rich app type. Defined in the application type schema under `dial:applicationTypeInterceptors`.

```json
{
  "applicationTypeSchemas": [
    {
      "$id": "https://dial.epam.com/application_types/quick_app/v1",
      "dial:applicationTypeDisplayName": "Quick App",
      "dial:applicationTypeInterceptors": ["quick-app-validator"]
    }
  ]
}
```

### Local interceptors

Applied to a specific model or application. Defined in the deployment's `interceptors` array.

```json
{
  "models": {
    "gpt-4o": {
      "endpoint": "http://adapter-openai:5000/...",
      "interceptors": ["pii-redactor"]
    }
  }
}
```

---

## Execution order

**Request flow:** Global → Application type → Local → Deployment

**Response flow:** Deployment → Local → Application type → Global

```
Request:  Client → [Global] → [Type] → [Local] → Model/App
Response: Model/App → [Local] → [Type] → [Global] → Client
```

## Examples

### Content filter interceptor

```json
{
  "interceptors": {
    "content-filter": {
      "endpoint": "http://content-filter:5000/openai/deployments/filter/chat/completions",
      "displayName": "Content Safety",
      "description": "Blocks harmful or inappropriate content"
    }
  },
  "globalInterceptors": ["content-filter"]
}
```

### PII redaction on specific models

```json
{
  "interceptors": {
    "pii-redactor": {
      "endpoint": "http://pii-service:5000/intercept",
      "displayName": "PII Redaction"
    }
  },
  "models": {
    "gpt-4o": {
      "endpoint": "http://adapter-openai:5000/...",
      "interceptors": ["pii-redactor"]
    },
    "claude-3": {
      "endpoint": "http://adapter-bedrock:5000/...",
      "interceptors": ["pii-redactor"]
    }
  }
}
```

### Audit logging for all deployments

```json
{
  "interceptors": {
    "audit-logger": {
      "endpoint": "http://audit-service:5000/log",
      "displayName": "Audit Logger",
      "description": "Logs all requests and responses for compliance"
    }
  },
  "globalInterceptors": ["audit-logger"]
}
```
