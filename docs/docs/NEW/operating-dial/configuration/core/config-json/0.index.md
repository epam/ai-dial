# config.json

`config.json` is DIAL Core's dynamic configuration file. It defines models, applications, interceptors, roles, API keys, routes, and tool sets. DIAL Core reloads this file at a configurable interval (default: 60 seconds), so changes take effect without restarting the service.

Dynamic settings can also be managed through [DIAL Admin](/tutorials/admin/home).

## File location

The file path is set by the `config.files` static setting in [`settings.json`](../settings-json/config-reload). Multiple files can be specified — they are merged in order.

In Docker Compose deployments:

```yaml
core:
  volumes:
    - ./core:/opt/config
```

With the static setting:

```json
{
  "config": {
    "files": ["/opt/config/config.json"]
  }
}
```
