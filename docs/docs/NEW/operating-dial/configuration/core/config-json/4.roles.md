# Roles configuration

Roles control access and usage limits for DIAL deployments. Each role can define rate limits, cost limits, and sharing permissions. Users are assigned roles through their identity provider's JWT claims.

Roles are defined in `config.json` under `roles.<role_name>`.

## Role attributes

### `limits.<deployment>`

Rate limits for a specific deployment (model or application).

```json
{
  "roles": {
    "default": {
      "limits": {
        "gpt-4o": {
          "requestHour": 100,
          "day": 50000
        }
      }
    }
  }
}
```

#### `limits.<deployment>.requestHour`

Maximum requests per hour.

```json
{
  "roles": {
    "default": {
      "limits": {
        "gpt-4o": {
          "requestHour": 100
        }
      }
    }
  }
}
```

#### `limits.<deployment>.requestDay`

Maximum requests per day.

```json
{
  "roles": {
    "default": {
      "limits": {
        "gpt-4o": {
          "requestDay": 1000
        }
      }
    }
  }
}
```

#### `limits.<deployment>.minute`

Maximum tokens per minute (floating window).

#### `limits.<deployment>.day`

Maximum tokens per day (floating window).

#### `limits.<deployment>.week`

Maximum tokens per week (floating window).

#### `limits.<deployment>.month`

Maximum tokens per month (floating window).

```json
{
  "roles": {
    "premium": {
      "limits": {
        "gpt-4o": {
          "minute": 10000,
          "day": 100000,
          "month": 1000000
        }
      }
    }
  }
}
```

### `costLimit`

Cost limits in USD across all models. Applies to roles with access to priced models.

#### `costLimit.minute`

Cost limit per minute in USD.

#### `costLimit.day`

Cost limit per day in USD.

#### `costLimit.week`

Cost limit per week in USD.

#### `costLimit.month`

Cost limit per month in USD.

```json
{
  "roles": {
    "standard": {
      "costLimit": {
        "day": "10.00",
        "month": "100.00"
      }
    }
  }
}
```

### `share.<RESOURCE_TYPE>`

Sharing permissions for different resource types.

#### `share.<RESOURCE_TYPE>.invitation_ttl`

TTL of invitation link in hours.

**Default:** `72`

#### `share.<RESOURCE_TYPE>.max_accepted_users`

Maximum users who can accept an invite.

**Default:** `10` for `APPLICATION`, unlimited for others

```json
{
  "roles": {
    "default": {
      "share": {
        "CONVERSATION": {
          "invitation_ttl": 168,
          "max_accepted_users": 50
        },
        "APPLICATION": {
          "invitation_ttl": 720,
          "max_accepted_users": 100
        }
      }
    }
  }
}
```

Resource types: `CONVERSATION`, `PROMPT`, `APPLICATION`, `FILE`

## Default role behavior

If limits are not defined for a role, limits from the `default` role apply. If the `default` role has no limits for a deployment, access is unlimited.

```json
{
  "roles": {
    "default": {
      "limits": {
        "gpt-4o": { "day": 50000 },
        "gpt-4o-mini": { "day": 200000 }
      }
    },
    "premium": {
      "limits": {
        "gpt-4o": { "day": 500000 }
      }
    }
  }
}
```

In this example:
- `default` users: 50k tokens/day for gpt-4o, 200k for gpt-4o-mini
- `premium` users: 500k tokens/day for gpt-4o, 200k for gpt-4o-mini (inherits from default)

## Examples

### Basic role structure

From [`dial-docker-compose/application/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/application/core/config.json):

```json
{
  "roles": {
    "default": {
      "limits": {
        "echo": {}
      }
    }
  }
}
```

Empty limits object `{}` grants unlimited access to the deployment.

### Tiered access

```json
{
  "roles": {
    "default": {
      "limits": {
        "gpt-4o-mini": { "day": 100000 }
      }
    },
    "standard": {
      "limits": {
        "gpt-4o-mini": { "day": 500000 },
        "gpt-4o": { "day": 50000 }
      }
    },
    "premium": {
      "limits": {
        "gpt-4o-mini": {},
        "gpt-4o": { "day": 500000 },
        "claude-3-opus": { "day": 100000 }
      },
      "costLimit": {
        "month": "500.00"
      }
    }
  }
}
```

### Request-based limits

```json
{
  "roles": {
    "api-consumer": {
      "limits": {
        "gpt-4o": {
          "requestHour": 60,
          "requestDay": 500
        }
      }
    }
  }
}
```

### Sharing permissions

```json
{
  "roles": {
    "collaborator": {
      "limits": {
        "gpt-4o": {}
      },
      "share": {
        "CONVERSATION": {
          "invitation_ttl": 168,
          "max_accepted_users": 25
        },
        "PROMPT": {
          "invitation_ttl": 720,
          "max_accepted_users": 100
        }
      }
    }
  }
}
```
