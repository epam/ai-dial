# Applications configuration

Applications are custom deployments that extend DIAL beyond standard LLM chat. Examples include RAG systems, code interpreters, image generators, and domain-specific assistants.

Applications are defined in `config.json` under `applications.<application_name>`.

## Application attributes

### `endpoint`

The application endpoint URL.

**Required**

```json
{
  "applications": {
    "my-rag": {
      "endpoint": "http://dial-rag:5000/openai/deployments/dial-rag/chat/completions"
    }
  }
}
```

### `displayName`

Human-readable name shown in DIAL Chat and Admin Panel.

```json
{
  "displayName": "Document Q&A"
}
```

### `displayVersion`

Version string shown in the UI.

```json
{
  "displayVersion": "1.0.0"
}
```

### `iconUrl`

URL to the application's icon image.

```json
{
  "iconUrl": "https://cdn.example.com/icons/rag.png"
}
```

### `description`

Application description shown in the UI.

```json
{
  "description": "Ask questions about your uploaded documents"
}
```

### `inputAttachmentTypes`

MIME types the application accepts as input attachments.

```json
{
  "inputAttachmentTypes": ["application/pdf", "text/plain", "text/markdown"]
}
```

Use `["*/*"]` to accept all attachment types:

```json
{
  "inputAttachmentTypes": ["*/*"]
}
```

### `maxInputAttachments`

Maximum number of input attachments allowed per request.

```json
{
  "maxInputAttachments": 20
}
```

### `features`

Feature flags for the application. Same schema as [model features](models#features).

```json
{
  "features": {
    "systemPromptSupported": true,
    "toolsSupported": true
  }
}
```

### `defaults`

Default parameter values sent with every request.

```json
{
  "defaults": {
    "max_tokens": 2048
  }
}
```

### `interceptors`

Local interceptors applied to this application. See [Interceptors](interceptors).

```json
{
  "interceptors": ["audit-logger"]
}
```

### `userRoles`

Roles that can access this application. When omitted, the application is available to all authenticated users.

```json
{
  "userRoles": ["analyst", "admin"]
}
```

### `forwardAuthToken`

Forward the user's auth token to the application endpoint.

**Default:** `false`

```json
{
  "forwardAuthToken": true
}
```

### `reference`

Reference to an application type schema. Used for schema-rich applications like Quick Apps.

```json
{
  "reference": {
    "applicationTypeSchemaId": "https://dial.epam.com/application_types/quick_app/v1"
  }
}
```

## Examples

### Echo application

From [`dial-docker-compose/application/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/application/core/config.json):

```json
{
  "applications": {
    "echo": {
      "displayName": "My Echo App",
      "description": "Simple application that repeats user's message",
      "endpoint": "http://echo:5000/openai/deployments/echo/chat/completions"
    }
  },
  "roles": {
    "default": {
      "limits": {
        "echo": {}
      }
    }
  }
}
```

### RAG application

From [`dial-docker-compose-advanced/core/config.json`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/core/config.json):

```json
{
  "applications": {
    "dial-rag": {
      "displayName": "Dial RAG",
      "description": "RAG (Retrieval Augmented Generation)",
      "endpoint": "http://dial-rag:5000/openai/deployments/dial-rag/chat/completions",
      "inputAttachmentTypes": ["*/*"]
    }
  }
}
```

### Application with role restriction

```json
{
  "applications": {
    "internal-kb": {
      "displayName": "Internal Knowledge Base",
      "description": "Search internal documentation",
      "endpoint": "http://kb-app:5000/chat/completions",
      "inputAttachmentTypes": ["*/*"],
      "userRoles": ["employee", "admin"],
      "forwardAuthToken": true
    }
  }
}
```

> **Note:** When `forwardAuthToken` is `true`, the application receives the user's JWT, enabling user-specific access control within the application.
