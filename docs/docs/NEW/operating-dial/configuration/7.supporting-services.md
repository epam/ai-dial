# Supporting services configuration

DIAL depends on several supporting services: Redis for in-memory caching, an identity provider (typically Keycloak) for authentication, and a logging framework. This page documents their configuration as used in DIAL deployments.

## Redis

DIAL Core requires Redis for in-memory caching of resources, sessions, and rate-limit counters. Redis is configured on two levels: the Redis server itself (command-line arguments) and the DIAL Core connection to it (static settings).

### Redis server configuration

From [`dial-docker-compose/common.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/common.yml):

```yaml
redis:
  image: redis:7.2.4-alpine3.19
  restart: always
  command: >
    redis-server
    --maxmemory 2000mb
    --maxmemory-policy volatile-lfu
    --save ""
    --appendonly no
    --loglevel warning
  mem_limit: 2200M
```

| Argument | Value | Description |
|---|---|---|
| `--maxmemory` | `2000mb` | Maximum memory Redis is allowed to use. Set slightly below the container memory limit. |
| `--maxmemory-policy` | `volatile-lfu` | Eviction policy. `volatile-lfu` evicts the least-frequently-used keys among those with an expiration set. |
| `--save ""` | (empty) | Disables RDB persistence. DIAL uses Redis as a cache, not a primary datastore ā€” persistence is not required. |
| `--appendonly no` | ā€” | Disables AOF persistence. |
| `--loglevel warning` | ā€” | Suppresses INFO and DEBUG log output. |

### DIAL Core Redis connection

DIAL Core connects to Redis via static settings. Either `singleServerConfig` or `clusterServersConfig` must be provided.

**Single server** (from Docker Compose):

```yaml
core:
  environment:
    'aidial.redis.singleServerConfig.address': 'redis://redis:6379'
```

**Cluster** (Kubernetes example):

```yaml
core:
  env:
    aidial.redis.clusterServersConfig.nodeAddresses: '["redis://host1:6379","redis://host2:6379"]'
```

For cloud-managed Redis, a provider configuration enables IAM-based authentication:

| Setting | Description |
|---|---|
| `redis.provider.name` | Cloud provider: `aws-elasti-cache`, `gcp-memory-store`, or `azure-redis-cache`. |
| `redis.provider.userId` | IAM-enabled user ID (AWS ElastiCache). |
| `redis.provider.accountName` | Service account resource name in `projects/-/serviceAccounts/{ACCOUNT}` format (GCP). |
| `redis.provider.region` | Geo region (AWS ElastiCache). |
| `redis.provider.clusterName` | Redis cluster name (AWS ElastiCache). |
| `redis.provider.serverless` | Whether the cache is serverless (AWS ElastiCache). |

See [DIAL Core configuration](core-configuration) for the full Redis settings reference.

## Keycloak

Keycloak is the recommended identity provider for DIAL. It handles user authentication, role assignment, and token issuance for DIAL Chat, Admin Panel, and Core.

### Keycloak container configuration

From [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
keycloak:
  image: keycloak/keycloak:26.3
  user: root
  command: ["start-dev", "--import-realm", "--http-port=${KC_PORT}", "--hostname-backchannel-dynamic=true"]
  environment:
    KC_BOOTSTRAP_ADMIN_USERNAME: admin
    KC_BOOTSTRAP_ADMIN_PASSWORD: ${KC_ADMIN_PASSWORD}
    KC_HOSTNAME: http://localhost:${KC_PORT}
    KC_HOSTNAME_STRICT: false
    KC_HOSTNAME_STRICT_HTTPS: false
    KC_HTTP_ENABLED: true
    KC_PROXY: edge
    QUARKUS_HTTP_LIMITS_MAX_HEADER_SIZE: 64k
  volumes:
    - ./keycloak/data:/opt/keycloak/data
    - ./keycloak/import/realm-config.json:/opt/keycloak/data/import/realm-config.json
```

| Variable | Description |
|---|---|
| `KC_BOOTSTRAP_ADMIN_USERNAME` | Initial admin username created on first startup. |
| `KC_BOOTSTRAP_ADMIN_PASSWORD` | Initial admin password. Store as a secret in production. |
| `KC_HOSTNAME` | Public URL of Keycloak. Clients use this URL for authentication redirects. |
| `KC_HOSTNAME_STRICT` | When `false`, Keycloak accepts requests on any hostname. Set to `true` in production. |
| `KC_HOSTNAME_STRICT_HTTPS` | When `false`, allows HTTP connections. Set to `true` in production with TLS termination. |
| `KC_HTTP_ENABLED` | Enable HTTP listener. Required when TLS is terminated at a load balancer. |
| `KC_PROXY` | Proxy mode: `edge` (TLS terminated at proxy), `reencrypt`, or `passthrough`. |
| `QUARKUS_HTTP_LIMITS_MAX_HEADER_SIZE` | Maximum HTTP header size. Set to `64k` to accommodate large JWTs with many role claims. |

The `--import-realm` flag causes Keycloak to import `realm-config.json` at startup. See [Configuration files](configuration-files) for the realm file structure and example.

### DIAL components that connect to Keycloak

**DIAL Core** ā€” validates JWTs using the Keycloak JWKS endpoint:

```yaml
core:
  environment:
    'aidial.identityProviders.keycloak.jwksUrl': 'http://keycloak:8180/realms/dial/protocol/openid-connect/certs'
    'aidial.identityProviders.keycloak.rolePath': 'roles'
    'aidial.identityProviders.keycloak.loggingKey': 'email'
```

**DIAL Chat** ā€” uses NextAuth.js with the Keycloak provider:

```yaml
chat:
  environment:
    AUTH_KEYCLOAK_CLIENT_ID: "dial-chat"
    AUTH_KEYCLOAK_DIAL_ROLES_FIELD: "resource_access.dial-chat.roles"
    AUTH_KEYCLOAK_HOST: "http://keycloak:8180/realms/dial/"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
```

**Admin Frontend** ā€” uses its own Keycloak client:

```yaml
admin-frontend:
  environment:
    AUTH_KEYCLOAK_HOST: "http://keycloak:8180/realms/dial/"
    AUTH_KEYCLOAK_CLIENT_ID: "dial-admin-console"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
```

**Admin Backend** ā€” validates tokens from the Keycloak provider:

```yaml
admin-backend:
  environment:
    'providers.keycloak.allowed-roles': 'ConfigAdmin,admin'
    'providers.keycloak.audiences': 'dial-admin-console'
    'providers.keycloak.issuer': "http://localhost:8180/realms/dial"
    'providers.keycloak.jwk-set-uri': "http://keycloak:8180/realms/dial/protocol/openid-connect/certs"
    'providers.keycloak.role-claims': roles
```

## Logging (gflog.xml)

DIAL Core uses the [gflog](https://github.com/epam/gflog) logging framework. The configuration file is passed via the `JAVA_OPTS` environment variable and mounted as a volume:

```yaml
core:
  environment:
    'JAVA_OPTS': '-Dgflog.config=/opt/settings/gflog.xml'
    'LOG_DIR': '/app/log'
  volumes:
    - ./settings:/opt/settings
    - ./core-logs:/app/log
```

### Default configuration

From [`dial-docker-compose/settings/gflog.xml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/settings/gflog.xml):

```xml
<config>
    <appender name="console" factory="com.epam.deltix.gflog.core.appender.ConsoleAppenderFactory"/>
    <appender name="file" factory="com.epam.deltix.gflog.core.appender.DailyRollingFileAppenderFactory"
              bufferCapacity="32m"
              file="/app/log/aidial.log"
              maxFiles="10"
              maxFileSize="1g">
        <layout template="%m%n"/>
    </appender>

    <logger level="INFO">
        <appender-ref ref="console"/>
    </logger>

    <logger level="INFO" name="aidial.log">
        <appender-ref ref="file"/>
    </logger>

    <service entryEncoding="UTF-8" entryMaxCapacity="30m" bufferCapacity="128m"/>
</config>
```

### Appender reference

| Appender | Type | Description |
|---|---|---|
| `console` | `ConsoleAppenderFactory` | Writes log entries to standard output. |
| `file` | `DailyRollingFileAppenderFactory` | Writes to a daily rolling file. Rotates daily, retaining up to `maxFiles` files of up to `maxFileSize` each. |

### File appender settings

| Attribute | Default | Description |
|---|---|---|
| `file` | ā€” | Log file path. |
| `maxFiles` | `10` | Number of rotated log files to retain. |
| `maxFileSize` | `1g` | Maximum size per log file before rotation. |
| `bufferCapacity` | `32m` | In-memory buffer size for the file appender. |

### Service settings

| Attribute | Default | Description |
|---|---|---|
| `entryEncoding` | `UTF-8` | Character encoding for log entries. |
| `entryMaxCapacity` | `30m` | Maximum size of a single log entry. |
| `bufferCapacity` | `128m` | Global logging service buffer size. |

### Customization

To increase log verbosity for debugging, change the logger level:

```xml
<logger level="DEBUG">
    <appender-ref ref="console"/>
</logger>
```

To disable file logging entirely, remove the `file` appender and its logger reference.

## Themes

DIAL Chat Themes is a static file server that hosts theme CSS, images, and configuration for DIAL Chat and other DIAL applications. It requires minimal configuration:

```yaml
themes:
  image: epam/ai-dial-chat-themes:0.10.0
  ports:
    - "3101:8080"
```

DIAL Chat points to the themes server via the `THEMES_CONFIG_HOST` environment variable:

```yaml
chat:
  environment:
    THEMES_CONFIG_HOST: "http://themes:8080"
```

Custom themes require building a custom Docker image based on [`epam/ai-dial-chat-themes`](https://github.com/epam/ai-dial-chat-themes). Theme customization is a developer topic ā€” see the Chat customization documentation for details.
