# DIAL Chat configuration

DIAL Chat ([`epam/ai-dial-chat`](https://github.com/epam/ai-dial-chat)) is the default web UI for DIAL. It is a Next.js application configured exclusively through environment variables.

## Docker Compose examples

**Minimal setup** (from [`dial-docker-compose/common.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/common.yml)) — API key authentication, no identity provider:

```yaml
chat:
  image: epam/ai-dial-chat:0.26.0
  platform: linux/amd64
  environment:
    NEXTAUTH_SECRET: "secret"
    THEMES_CONFIG_HOST: "http://themes:8080"
    DIAL_API_HOST: "http://core:8080"
    DIAL_API_KEY: "dial_api_key"
    ENABLED_FEATURES: "conversations-section,prompts-section,top-settings,top-clear-conversation,top-chat-info,top-chat-model-settings,empty-chat-settings,header,footer,request-api-key,report-an-issue,likes,conversations-sharing,input-files,attachments-manager,prompts-sharing,prompts-publishing,conversations-publishing,custom-logo,input-links,custom-applications,message-templates,marketplace,quick-apps,code-apps,applications-sharing,marketplace-table-view"
    KEEP_ALIVE_TIMEOUT: ${CHAT_KEEP_ALIVE_TIMEOUT:-20000}
```

**Production setup** (from [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml)) — Keycloak authentication:

```yaml
chat:
  image: epam/ai-dial-chat:0.41.7
  platform: linux/amd64
  environment:
    AUTH_KEYCLOAK_CLIENT_ID: "dial-chat"
    AUTH_KEYCLOAK_DIAL_ROLES_FIELD: "resource_access.dial-chat.roles"
    AUTH_KEYCLOAK_HOST: "http://keycloak:8180/realms/dial/"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
    DIAL_API_HOST: "http://core:8080"
    ENABLED_FEATURES: "conversations-section,prompts-section,top-settings,..."
    KEEP_ALIVE_TIMEOUT: ${CHAT_KEEP_ALIVE_TIMEOUT:-20000}
    NEXTAUTH_SECRET: "${NEXTAUTH_SECRET}"
    THEMES_CONFIG_HOST: "http://themes:8080"
    NEXTAUTH_URL: "http://localhost:3000"
    NEXTAUTH_URL_INTERNAL: "http://chat:3000"
```

In the minimal setup, `DIAL_API_KEY` provides a static API key for Core authentication. In the production setup, Keycloak issues JWTs and `DIAL_API_KEY` is not needed.

## Core connection

| Variable | Default | Required | Description |
|---|---|---|---|
| `DIAL_API_HOST` | — | Yes | DIAL Core API host URL. |
| `DIAL_CORE_EXTERNAL_URL` | — | No | DIAL Core external URL. Not used for API calls — used for constructing links visible to end users. |
| `DIAL_API_KEY` | — | Conditional | DIAL Core API key. Required when JWT authentication is not configured. |
| `DIAL_API_VERSION` | `2025-01-01-preview` | No | DIAL API version string. |

| `NEXT_PUBLIC_DEFAULT_SYSTEM_PROMPT` | — | No | Default system prompt for new conversations. |

| `RECENT_MODELS_IDS` | — | No | Comma-separated list of recently-used model IDs. |
| `STORAGE_TYPE` | `api` | No | Storage type: `api` (DIAL Core) or `browserStorage` (local browser storage). |
| `KEEP_ALIVE_TIMEOUT` | `61000` | No | Maximum idle connection time in milliseconds. |
| `MAX_PROMPT_TOKENS_DEFAULT_PERCENT` | `75` | No | Percentage of `MAX_PROMPT_TOKENS_DEFAULT_VALUE` used as the token limit. |

| `SHOW_TOKEN_SUB` | `false` | No | Show token subject in refresh login error logs. |

## Feature flags

The `ENABLED_FEATURES` variable controls which UI features are active. It accepts a comma-separated list of feature names:

| Feature | Description |
|---|---|
| `conversations-section` | Show the conversations sidebar section. |
| `prompts-section` | Show the prompts sidebar section. |
| `top-settings` | Show settings in the top bar. |
| `top-clear-conversation` | Show "Clear conversation" in the top bar. |
| `top-chat-info` | Show chat info in the top bar. |
| `top-chat-model-settings` | Show model settings in the top bar. |
| `empty-chat-settings` | Show settings on the empty chat screen. |
| `header` | Show the application header. |
| `footer` | Show the application footer. |
| `request-api-key` | Enable the "Request API Key" feature. |
| `report-an-issue` | Enable issue reporting. |
| `likes` | Enable message rating (likes/dislikes). |
| `conversations-sharing` | Enable sharing conversations with other users. |
| `input-files` | Enable file input in the chat. |
| `attachments-manager` | Enable the attachments manager. |
| `prompts-sharing` | Enable sharing prompts. |
| `prompts-publishing` | Enable publishing prompts. |
| `conversations-publishing` | Enable publishing conversations. |
| `custom-logo` | Enable custom logo support. |
| `input-links` | Enable link input in the chat. |
| `custom-applications` | Enable custom applications. |
| `message-templates` | Enable message templates. |
| `marketplace` | Enable the marketplace. |
| `quick-apps` | Enable Quick Apps. |
| `code-apps` | Enable Code Apps. |
| `applications-sharing` | Enable sharing applications. |
| `marketplace-table-view` | Enable table view in the marketplace. |

Role-based feature access is controlled with `ENABLED_FEATURES_ROLES` (stringified JSON mapping features to roles) and `ENABLED_FEATURES_DATA` (stringified JSON providing additional feature data).

## Content Security Policy

| Variable | Default | Description |
|---|---|---|
| `ALLOWED_IFRAME_ORIGINS` | `none` | URLs allowed to embed DIAL Chat in an iframe (`frame-ancestors` CSP directive). |
| `ALLOWED_IFRAME_SOURCES` | `none` | iframe sources allowed to load content (`frame-src` CSP directive). |
| `ALLOWED_SCRIPT_SOURCES` | `'self'` + iframe origins/sources | Script sources (`script-src` CSP directive). |
| `IS_IFRAME` | `false` | Enable DIAL Overlay mode (embedded in an iframe). |
| `ALLOW_OPEN_SIGNIN_PAGE_IN_IFRAME` | `false` | Allow the sign-in page to render within an iframe (for Overlay). |

### Multiple instances of the same provider

To configure multiple instances of the same provider type, use the pattern `AUTH_{PROVIDER_TYPE}_{INDEX}_{FIELD_NAME}` where `INDEX` is 1, 2, 3, etc. For example: `AUTH_KEYCLOAK_1_CLIENT_ID`, `AUTH_KEYCLOAK_2_CLIENT_ID`.
