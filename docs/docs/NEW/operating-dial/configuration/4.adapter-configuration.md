# Adapter configuration

DIAL model adapters translate the DIAL Unified API into provider-specific APIs. Each adapter runs as a separate container configured through environment variables. Model routing — which requests go to which adapter — is defined in DIAL Core's `config.json` via upstream configurations.

This page documents environment variables for three officially supported adapters: OpenAI, Bedrock, and Vertex AI.

## Two-part configuration pattern

Adapter configuration has two parts:

1. **Adapter environment variables** — set on the adapter container. Control logging, deployment categories, storage integration, and provider-specific behavior.
2. **Upstream configuration in DIAL Core** — defined in `config.json` under each model's `upstreams` array. Specifies the provider endpoint and API key that the adapter uses for each model.

Both parts are required. The adapter container handles protocol translation; DIAL Core handles routing and credentials.

## OpenAI Adapter

The OpenAI Adapter ([`epam/ai-dial-adapter-openai`](https://github.com/epam/ai-dial-adapter-openai)) supports Azure OpenAI, OpenAI, Mistral, Databricks, vLLM, and other OpenAI-compatible APIs.

### Docker Compose example

From [`dial-docker-compose-advanced/docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
adapter-openai:
  image: epam/ai-dial-adapter-openai:0.36.1
  environment:
    DALLE3_DEPLOYMENTS: "dall-e-3"
    DIAL_USE_FILE_STORAGE: "true"
    DIAL_URL: "http://core:8080"
    GPT4O_DEPLOYMENTS: "gpt-4o"
    GPT4O_MINI_DEPLOYMENTS: "gpt-4o-mini"
    WEB_CONCURRENCY: "3"
```

### Deployment category variables

Deployment categories tell the adapter which protocol to use for each model deployment. A deployment listed in `DALLE3_DEPLOYMENTS` uses the DALL-E 3 image generation API; one in `GPT4O_DEPLOYMENTS` uses the GPT-4o chat completion API with appropriate defaults.

| Variable | Default | Description |
|---|---|---|
| `DALLE3_DEPLOYMENTS` | `""` | Comma-separated deployments using the DALL-E 3 API. Example: `dall-e-3,dalle3` |
| `DALLE3_AZURE_API_VERSION` | `2024-02-01` | Azure API version for DALL-E 3 requests. |
| `GPT_IMAGE_1_DEPLOYMENTS` | `""` | Comma-separated deployments using the GPT-Image 1 API. Example: `gpt-image-1` |
| `GPT_IMAGE_1_AZURE_API_VERSION` | `2024-02-01` | Azure API version for GPT-Image 1 requests. |
| `MISTRAL_DEPLOYMENTS` | `""` | Comma-separated deployments using the Mistral Large Azure API. Example: `mistral-large-azure` |
| `DATABRICKS_DEPLOYMENTS` | `""` | Comma-separated Databricks chat completion deployments. Example: `databricks-dbrx-instruct` |
| `GPT4O_DEPLOYMENTS` | `""` | Comma-separated GPT-4o chat completion deployments. Example: `gpt-4o-2024-05-13` |
| `GPT4O_MINI_DEPLOYMENTS` | `""` | Comma-separated GPT-4o mini chat completion deployments. Example: `gpt-4o-mini-2024-07-18` |
| `VLLM_DEPLOYMENTS` | `""` | Comma-separated deployments using a vLLM OpenAI-compatible upstream. Example: `vllm-llama3` |
| `QWEN3_ASR_VLLM_DEPLOYMENTS` | `""` | Comma-separated Qwen3-ASR deployments served via vLLM. Example: `qwen3-asr` |
| `AZURE_AI_VISION_DEPLOYMENTS` | `""` | Comma-separated Azure AI Vision embedding deployments. Endpoint should be `https://<service>.cognitiveservices.azure.com/`. |
| `AUDIO_AZURE_API_VERSION` | `2025-03-01-preview` | Azure API version for Audio API endpoints. |

### Storage and DIAL Core integration

| Variable | Default | Description |
|---|---|---|
| `DIAL_USE_FILE_STORAGE` | `False` | When `True`, image model artifacts (DALL-E images) are uploaded to DIAL file storage and base64 encodings are replaced with storage links. |
| `DIAL_URL` | — | URL of DIAL Core. Required when `DIAL_USE_FILE_STORAGE` is `True`. |

### Performance and operational

| Variable | Default | Description |
|---|---|---|
| `LOG_LEVEL` | `INFO` | Log level. |
| `WEB_CONCURRENCY` | `1` | Number of Uvicorn worker processes. Set to the number of logical CPUs for full utilization under tokenization load. |
| `THREAD_POOL_SIZE` | `min(32, CPUs + 4)` | Thread pool size for CPU-heavy tasks (tokenization, image analysis). |
| `NON_STREAMING_DEPLOYMENTS` | `""` | Comma-separated deployments that do not support streaming. The adapter emulates streaming by converting the response into a single-chunk stream. Example: `o1-mini,o1-preview` |
| `ELIMINATE_EMPTY_CHOICES` | `False` | When `True`, response streams exclude chunks with empty choices lists. Useful for clients that do not handle Azure OpenAI content filter chunks. |
| `SSE_HEARTBEAT_INTERVAL` | — (disabled) | Seconds of idle time before the adapter inserts a ping comment into streaming responses. Prevents read timeouts on long-running requests. |

### Tokenization and API versioning

| Variable | Default | Description |
|---|---|---|
| `TIKTOKEN_MODEL_MAPPING` | `{}` | JSON dictionary mapping deployment IDs to tiktoken model names for adapter-side tokenization. Default tokenizer is `gpt-4o` (`o200k_base`). Example: `{"my-gpt":"gpt-3.5-turbo"}` |
| `API_VERSIONS_MAPPING` | `{}` | JSON mapping of API versions for Azure OpenAI. An empty key sets the default version. Example: `{"2023-03-15-preview":"2023-05-15","":"2024-02-15-preview"}` |

### Azure-specific

| Variable | Default | Description |
|---|---|---|
| `ACCESS_TOKEN_EXPIRATION_WINDOW` | `10` | Seconds before actual expiration when the Azure access token is renewed. |
| `AZURE_OPEN_AI_SCOPE` | `https://cognitiveservices.azure.com/.default` | Scope of access token to Azure OpenAI services. |

### Other

| Variable | Default | Description |
|---|---|---|
| `CLAUDE_DEFAULT_MAX_TOKENS` | `1536` | Default `max_tokens` when not provided in the request. Discouraged — configure the default in DIAL Core's `config.json` instead. |
| `USE_SYSTEM_CA_CERTS` | — (disabled) | Set to any non-empty value to trust private CAs and self-signed certificates. Requires `*.crt` files mounted at `/certificates`. |

### Upstream configuration in config.json

Each model using the OpenAI Adapter needs an upstream entry in DIAL Core's `config.json`:

**Azure OpenAI upstream:**

```json
{
  "models": {
    "gpt-4o": {
      "type": "chat",
      "displayName": "GPT-4o",
      "endpoint": "http://adapter-openai:5000/openai/deployments/gpt-4o/chat/completions",
      "upstreams": [
        {
          "endpoint": "http://azure_deployment_host/openai/deployments/gpt-4o/chat/completions",
          "key": "AZURE_API_KEY"
        }
      ]
    }
  }
}
```

**Self-hosted (Ollama/vLLM) upstream:**

```json
{
  "models": {
    "vision-model": {
      "type": "chat",
      "endpoint": "http://adapter-openai:5000/openai/deployments/vision-model/chat/completions",
      "upstreams": [
        {
          "endpoint": "http://ollama:11434/v1/chat/completions",
          "key": "dummy-key"
        }
      ]
    }
  }
}
```

Self-hosted models that are fully OpenAI-compatible can bypass the adapter entirely and point `endpoint` directly at the model server. The adapter is needed when the model requires deployment category handling (image generation, tokenization, or API version mapping).

## Bedrock Adapter

The Bedrock Adapter ([`epam/ai-dial-adapter-bedrock`](https://github.com/epam/ai-dial-adapter-bedrock)) supports AWS Bedrock models including Anthropic Claude, Amazon Titan, Stability, Cohere, and Meta Llama.

### Environment variables

| Variable | Default | Description |
|---|---|---|
| `AWS_ACCESS_KEY_ID` | — | AWS credentials with Bedrock access. |
| `AWS_SECRET_ACCESS_KEY` | — | AWS credentials with Bedrock access. |
| `AWS_SESSION_TOKEN` | — | AWS session token (optional). |
| `AWS_DEFAULT_REGION` | — | AWS region. Example: `us-east-1` |
| `AWS_ASSUME_ROLE_ARN` | — | AWS assume role ARN. Example: `arn:aws:iam::123456789012:role/RoleName` |
| `LOG_LEVEL` | `INFO` | Log level. |
| `AIDIAL_LOG_LEVEL` | `WARNING` | DIAL SDK log level. |
| `DIAL_URL` | — | URL of DIAL Core. When set, generated images are uploaded to DIAL file storage. |
| `WEB_CONCURRENCY` | `1` | Number of server workers. |
| `CLAUDE_DEFAULT_MAX_TOKENS` | `1536` | Default `max_tokens` when not provided. Discouraged — configure in DIAL Core instead. |
| `BOTOCORE_MAX_RETRY_ATTEMPTS` | `0` | Retry attempts for Bedrock API/Converse API errors. |
| `ANTHROPIC_MAX_RETRY_ATTEMPTS` | `0` | Retry attempts for Anthropic model errors. |
| `ANTHROPIC_MAX_CONNECTIONS` | `1000` | Maximum concurrent requests via Anthropic SDK. |
| `ANTHROPIC_MAX_KEEPALIVE_CONNECTIONS` | `100` | Maximum idle connections in the connection pool. |
| `BOTOCORE_CLIENT_MAX_POOL_CONNECTIONS` | `1000` | Maximum connections in the Bedrock API connection pool. |
| `COMPATIBILITY_MAPPING` | `{}` | **Deprecated** since 0.37.0. JSON mapping of unsupported Bedrock deployments to supported ones. Use DIAL Core config instead. |

### Upstream configuration in config.json

Bedrock models use the adapter endpoint with the model's Bedrock deployment name:

```json
{
  "models": {
    "anthropic.claude-3-5-sonnet-20240620-v1:0": {
      "type": "chat",
      "displayName": "Claude 3.5 Sonnet",
      "endpoint": "http://adapter-bedrock:5000/openai/deployments/anthropic.claude-3-5-sonnet-20240620-v1:0/chat/completions"
    }
  }
}
```

Bedrock adapter credentials are set on the adapter container, not in config.json upstreams. The adapter uses the AWS credentials from its environment to authenticate with the Bedrock service.

## Vertex AI Adapter

The Vertex AI Adapter ([`epam/ai-dial-adapter-vertexai`](https://github.com/epam/ai-dial-adapter-vertexai)) supports Google Vertex AI models including Gemini, PaLM, Imagen, and Anthropic Claude on Vertex.

### Environment variables

| Variable | Default | Description |
|---|---|---|
| `GOOGLE_APPLICATION_CREDENTIALS` | — | Path to GCP Application Default Credentials JSON file. When not set, the adapter uses ADC (Application Default Credentials) from the environment. |
| `DEFAULT_REGION` | — | Default Vertex AI region. Example: `us-central1` |
| `GCP_PROJECT_ID` | — | GCP project ID. |
| `LOG_LEVEL` | `INFO` | Log level. |
| `AIDIAL_LOG_LEVEL` | `WARNING` | DIAL SDK log level. |
| `WEB_CONCURRENCY` | `1` | Number of server workers. |
| `DIAL_URL` | — | URL of DIAL Core. Used to access images stored in DIAL file storage. |
| `CLAUDE_DEFAULT_MAX_TOKENS` | `1536` | Default `max_tokens` when not provided. Discouraged — configure in DIAL Core instead. |
| `GOOGLE_GENAI_MAX_RETRY_ATTEMPTS` | `0` | Retry attempts for Google GenAI model errors. |
| `ANTHROPIC_MAX_RETRY_ATTEMPTS` | `0` | Retry attempts for Anthropic model errors (Claude on Vertex). |
| `COMPATIBILITY_MAPPING` | `{}` | **Deprecated** since 0.32.0. JSON mapping of unsupported Vertex AI deployments to supported ones. Use DIAL Core config instead. |

### Upstream configuration in config.json

Vertex AI models use the adapter endpoint with a region/project-specific upstream:

```json
{
  "models": {
    "gemini-1.5-pro": {
      "type": "chat",
      "displayName": "Gemini 1.5 Pro",
      "endpoint": "http://adapter-vertexai:5000/openai/deployments/gemini-1.5-pro/chat/completions"
    }
  }
}
```

Like the Bedrock Adapter, Vertex AI credentials are set on the adapter container via `GOOGLE_APPLICATION_CREDENTIALS` or GCP ADC, not in config.json upstreams.

## Cross-adapter variables

Several variables appear in multiple adapters with identical behavior:

| Variable | OpenAI | Bedrock | Vertex AI | Description |
|---|:---:|:---:|:---:|---|
| `LOG_LEVEL` | `INFO` | `INFO` | `INFO` | Application log level. |
| `AIDIAL_LOG_LEVEL` | — | `WARNING` | `WARNING` | DIAL SDK log level (not available in OpenAI Adapter). |
| `DIAL_URL` | Yes | Yes | Yes | DIAL Core URL for file storage integration. |
| `WEB_CONCURRENCY` | `1` | `1` | `1` | Number of server worker processes. |
| `CLAUDE_DEFAULT_MAX_TOKENS` | `1536` | `1536` | `1536` | Default max tokens. Discouraged in all three. |
