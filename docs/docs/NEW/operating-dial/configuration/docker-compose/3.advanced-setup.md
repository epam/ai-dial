# Advanced setup

The advanced Docker Compose setup in [`dial-docker-compose-advanced/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced) provides a production-like deployment with Keycloak authentication, Admin Panel, and multiple model adapters.

## File structure

| File | Purpose |
|------|---------|
| `docker-compose-base.yml` | All core services: Chat, Core, Redis, Themes, Keycloak, Admin Backend, Admin Frontend, OpenAI Adapter |
| `docker-compose-full.yml` | Includes `docker-compose-base.yml` and adds DIAL RAG |
| `.env` | Port assignments, secrets, and host configuration |

## docker-compose-base.yml

[`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml) defines 8 services:

### Services overview

| Service | Image | Description |
|---------|-------|-------------|
| `themes` | `epam/ai-dial-chat-themes` | Static theme server |
| `redis` | `redis:7.2.4-alpine3.19` | In-memory cache |
| `chat` | `epam/ai-dial-chat` | Web UI with Keycloak auth |
| `core` | `epam/ai-dial-core` | API server with identity providers |
| `adapter-openai` | `epam/ai-dial-adapter-openai` | OpenAI/Azure adapter |
| `keycloak` | `keycloak/keycloak` | Identity provider |
| `admin-backend` | `epam/ai-dial-admin-backend` | Admin API |
| `admin-frontend` | `epam/ai-dial-admin-frontend` | Admin web UI |

### Core service configuration

Unlike quick start setups, the advanced Core service uses environment variable overrides instead of `AIDIAL_SETTINGS`:

```yaml
core:
  image: epam/ai-dial-core:0.39.2
  environment:
    'JAVA_OPTS': '-Dgflog.config=/opt/settings/gflog.xml'
    'proxy.config.files': '["/app/config/config.json"]'
    'aidial.identityProviders.keycloak.jwksUrl': 'http://keycloak:${KC_PORT}/realms/dial/protocol/openid-connect/certs'
    'aidial.identityProviders.keycloak.rolePath': 'roles'
    'aidial.identityProviders.keycloak.loggingKey': 'email'
    'aidial.identityProviders.keycloak.loggingSalt': 'loggingSalt'
    'aidial.encryption.secret': '${DIAL_CORE_ENCRYPTION_SECRET}'
    'aidial.encryption.key': '${DIAL_CORE_ENCRYPTION_KEY}'
    'aidial.redis.singleServerConfig.address': 'redis://redis:6379'
    'aidial.storage.provider': 'filesystem'
    'aidial.storage.prefix': 'core'
    'aidial.storage.bucket': 'dial'
    'aidial.storage.createBucket': 'true'
    'aidial.storage.overrides': '{"jclouds.filesystem.basedir": "data"}'
    'aidial.applications.includeCustomApps': 'true'
  volumes:
    - ./core:/app/config
    - ./core-logs:/app/log
    - ./core-data:/app/data
    - ./settings:/opt/settings
```

All `aidial.*` environment variables override their corresponding `settings.json` keys. See [settings.json](../core/settings-json/index) for the precedence rules.

### Keycloak service

Keycloak imports the realm configuration at startup:

```yaml
keycloak:
  image: keycloak/keycloak:26.3
  command: ["start-dev", "--import-realm", "--http-port=${KC_PORT}", "--hostname-backchannel-dynamic=true"]
  environment:
    KC_BOOTSTRAP_ADMIN_USERNAME: admin
    KC_BOOTSTRAP_ADMIN_PASSWORD: ${KC_ADMIN_PASSWORD}
    KC_HOSTNAME: http://localhost:${KC_PORT}
    KC_HOSTNAME_STRICT: false
    KC_HTTP_ENABLED: true
    KC_PROXY: edge
    QUARKUS_HTTP_LIMITS_MAX_HEADER_SIZE: 64k
  volumes:
    - ./keycloak/import/realm-config.json:/opt/keycloak/data/import/realm-config.json
```

See [realm-config.json](../realm-config-json) for the realm file structure.

## docker-compose-full.yml

[`docker-compose-full.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-full.yml) extends the base setup with DIAL RAG:

```yaml
include:
  - path: ./docker-compose-base.yml

services:
  dial-rag:
    image: ghcr.io/epam/ai-dial-rag:0.32.0
    env_file: .env
    environment:
      DIAL_URL: "http://core:8080"
      DIAL_RAG__INDEX_STORAGE__USE_DIAL_FILE_STORAGE: "true"
      DIAL_RAG__REQUEST__INDEXING__DESCRIPTION_INDEX__LLM__DEPLOYMENT_NAME: "gpt-4o-mini"
      DIAL_RAG__REQUEST__QA_CHAIN__CHAT_CHAIN__LLM__DEPLOYMENT_NAME: "gpt-4o"
```

## Running the advanced setup

```bash
cd dial-docker-compose-advanced
cp .env.example .env   # Edit with your secrets
docker compose -f docker-compose-base.yml up
```

Or with RAG:

```bash
docker compose -f docker-compose-full.yml up
```
