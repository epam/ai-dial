# Admin Panel

The Admin Panel consists of two services: a backend API and a frontend web UI. Both authenticate against Keycloak and manage DIAL Core's `config.json`.

## Admin Backend

From [`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
admin-backend:
  image: epam/ai-dial-admin-backend:0.13.0
  platform: linux/amd64
  user: root
  environment:
    ENABLE_CORE_CONFIG_VERSION_AUTO_DETECT: "true"
    CONFIG_EXPORT_OUTPUTFILE_PATH: /app/config/config.json
    CONFIG_REST_SECURITY_MODE: "oidc"
    CORE_CLIENT_URL: "http://core:8080"
    DEBUG_OPTS: "-agentlib:jdwp=transport=dt_socket,server=y,suspend=n,address=*:5006"
    DISABLE_SWAGGER_AUTHORIZATION: true
    DIAL_ADMIN_CLIENT_ID: "dial-admin-console"
    ENABLE_CONFIG_AUTO_IMPORT_ON_BOOTSTRAP: "true"
    H2_DATASOURCE_ENCRYPTED_FILE_KEY: ${H2_DATASOURCE_ENCRYPTED_FILE_KEY}
    H2_DATASOURCE_MASTER_KEY: ${H2_DATASOURCE_MASTER_KEY}
    H2_DATASOURCE_PASSWORD: ${H2_DATASOURCE_PASSWORD}
    H2_FILE: ./data/testdb
    METRICS_ENABLED: "false"
    'providers.keycloak.allowed-roles': 'ConfigAdmin,admin'
    'providers.keycloak.audiences': 'dial-admin-console'
    'providers.keycloak.issuer': "http://${COMMON_HOST}:${KC_PORT}/realms/dial"
    'providers.keycloak.jwk-set-uri': "http://keycloak:${KC_PORT}/realms/dial/protocol/openid-connect/certs"
    'providers.keycloak.role-claims': roles
  volumes:
    - ./admin/data:/app/data
    - ./core:/app/config
  depends_on:
    - core
  restart: always
```

### Environment variables

#### Configuration export

| Variable | Description |
|----------|-------------|
| `ENABLE_CORE_CONFIG_VERSION_AUTO_DETECT` | Automatically detect the Core config version. |
| `CONFIG_EXPORT_OUTPUTFILE_PATH` | Path where the backend writes the exported `config.json`. Shares the volume with Core. |
| `ENABLE_CONFIG_AUTO_IMPORT_ON_BOOTSTRAP` | Import existing `config.json` into the Admin database on first startup. |

#### Security (OIDC)

| Variable | Description |
|----------|-------------|
| `CONFIG_REST_SECURITY_MODE` | Authentication mode. Set to `oidc` for Keycloak integration. |
| `DIAL_ADMIN_CLIENT_ID` | Keycloak client ID for the Admin Panel. Must match the client in `realm-config.json`. |
| `providers.keycloak.allowed-roles` | Comma-separated roles allowed to access the Admin API. |
| `providers.keycloak.audiences` | Expected `aud` claim in JWT tokens. |
| `providers.keycloak.issuer` | Keycloak issuer URL for token validation. Uses `${COMMON_HOST}` for external access. |
| `providers.keycloak.jwk-set-uri` | JWKS endpoint URL for JWT signature verification. Uses the internal `keycloak` hostname. |
| `providers.keycloak.role-claims` | JWT claim path containing user roles. |

:::note
The `issuer` uses `${COMMON_HOST}` (external hostname), while `jwk-set-uri` uses the internal Docker network hostname `keycloak`. This is because issuer validation checks the `iss` claim in tokens issued to the browser (which sees the external hostname), while JWKS fetching happens server-to-server.
:::

#### Core connection

| Variable | Description |
|----------|-------------|
| `CORE_CLIENT_URL` | DIAL Core internal URL. |

#### Database (H2)

| Variable | Description |
|----------|-------------|
| `H2_DATASOURCE_ENCRYPTED_FILE_KEY` | H2 database file encryption key. Injected from `.env`. |
| `H2_DATASOURCE_MASTER_KEY` | H2 database master key. Injected from `.env`. |
| `H2_DATASOURCE_PASSWORD` | H2 database password. Injected from `.env`. |
| `H2_FILE` | H2 database file path. |

#### Other

| Variable | Description |
|----------|-------------|
| `DEBUG_OPTS` | JVM debug options. Enables remote debugging on port 5006. |
| `DISABLE_SWAGGER_AUTHORIZATION` | Disable Swagger UI authorization for development convenience. |
| `METRICS_ENABLED` | Enable Prometheus metrics endpoint. |

### Volumes

| Mount | Purpose |
|-------|---------|
| `./admin/data:/app/data` | H2 database storage |
| `./core:/app/config` | Shared `config.json` directory with Core |

## Admin Frontend

```yaml
admin-frontend:
  image: epam/ai-dial-admin-frontend:0.13.0
  platform: linux/amd64
  environment:
    DIAL_ADMIN_API_URL: "http://admin-backend:8080/"
    NEXTAUTH_URL: "http://${COMMON_HOST}:${DIAL_ADMIN_FRONTEND_HOST_PORT}"
    NEXTAUTH_SECRET: "${NEXTAUTH_SECRET}"
    NEXTAUTH_COOKIE_PREFIX: "admin"
    AUTH_KEYCLOAK_HOST: "http://keycloak:${KC_PORT}/realms/dial/"
    AUTH_KEYCLOAK_CLIENT_ID: "dial-admin-console"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
    ADMIN_ROLE_NAMES: 'ConfigAdmin,admin'
  ports:
    - "${DIAL_ADMIN_FRONTEND_HOST_PORT}:3000"
  depends_on:
    - keycloak
    - admin-backend
  restart: always
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `DIAL_ADMIN_API_URL` | Admin Backend internal URL. |
| `NEXTAUTH_URL` | Public URL of the Admin Frontend. Used by NextAuth.js for callback URLs. |
| `NEXTAUTH_SECRET` | NextAuth.js session encryption key. Shared with Chat. Injected from `.env`. |
| `NEXTAUTH_COOKIE_PREFIX` | Cookie name prefix. Set to `admin` to avoid collisions with Chat's NextAuth cookies. |
| `AUTH_KEYCLOAK_HOST` | Keycloak realm URL for OIDC discovery. |
| `AUTH_KEYCLOAK_CLIENT_ID` | Keycloak client ID. Must match `DIAL_ADMIN_CLIENT_ID` on the backend. |
| `AUTH_KEYCLOAK_SECRET` | Keycloak client secret. Injected from `.env`. |
| `ADMIN_ROLE_NAMES` | Comma-separated roles required to access the Admin UI. Must match `providers.keycloak.allowed-roles` on the backend. |
