# Adapters & RAG

## OpenAI Adapter

From [`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
adapter-openai:
  image: epam/ai-dial-adapter-openai:0.36.1
  platform: linux/amd64
  environment:
    DALLE3_DEPLOYMENTS: "dall-e-3"
    DIAL_USE_FILE_STORAGE: "true"
    DIAL_URL: "http://core:8080"
    GPT4O_DEPLOYMENTS: "gpt-4o"
    GPT4O_MINI_DEPLOYMENTS: "gpt-4o-mini"
    WEB_CONCURRENCY: "3"
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `DALLE3_DEPLOYMENTS` | Comma-separated Azure OpenAI deployment names for DALL-E 3. |
| `DIAL_USE_FILE_STORAGE` | When `true`, the adapter stores generated images in DIAL Core's file storage instead of returning base64. |
| `DIAL_URL` | DIAL Core internal URL. Used for file storage API calls. |
| `GPT4O_DEPLOYMENTS` | Comma-separated Azure OpenAI deployment names for GPT-4o. |
| `GPT4O_MINI_DEPLOYMENTS` | Comma-separated Azure OpenAI deployment names for GPT-4o mini. |
| `WEB_CONCURRENCY` | Number of Uvicorn worker processes. |

:::note
The deployment names (`GPT4O_DEPLOYMENTS`, etc.) must match the Azure OpenAI resource deployment names. Model-to-upstream routing is configured in [`config.json`](../../core/config-json/models).
:::

See [Adapter configuration](../../adapter-configuration) for the full adapter environment variable reference.

## DIAL RAG

[`docker-compose-full.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-full.yml) extends the base setup by including `docker-compose-base.yml` and adding the RAG service:

```yaml
include:
  - path: ./docker-compose-base.yml

services:
  dial-rag:
    image: ghcr.io/epam/ai-dial-rag:0.32.0
    env_file: .env
    environment:
      DIAL_URL: "http://core:8080"
      DIAL_RAG__INDEX_STORAGE__USE_DIAL_FILE_STORAGE: "true"
      DIAL_RAG__REQUEST__INDEXING__DESCRIPTION_INDEX__LLM__DEPLOYMENT_NAME: "gpt-4o-mini"
      DIAL_RAG__REQUEST__QA_CHAIN__CHAT_CHAIN__LLM__DEPLOYMENT_NAME: "gpt-4o"
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `DIAL_URL` | DIAL Core internal URL. |
| `DIAL_RAG__INDEX_STORAGE__USE_DIAL_FILE_STORAGE` | Store document indexes in DIAL Core's file storage. |
| `DIAL_RAG__REQUEST__INDEXING__DESCRIPTION_INDEX__LLM__DEPLOYMENT_NAME` | Model deployment used for generating document descriptions during indexing. |
| `DIAL_RAG__REQUEST__QA_CHAIN__CHAT_CHAIN__LLM__DEPLOYMENT_NAME` | Model deployment used for answering questions over indexed documents. |

:::note
RAG requires working model deployments (e.g., `gpt-4o`, `gpt-4o-mini`) configured in `config.json` with valid API keys. Without them, indexing and QA requests fail.
:::
