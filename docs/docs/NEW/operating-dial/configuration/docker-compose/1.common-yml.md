# common.yml

[`dial-docker-compose/common.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose/common.yml) defines the shared service definitions used by all quick start setups. Each setup-specific `docker-compose.yml` includes this file via the `include` directive.

## Full file

```yaml
services:
  themes:
    image: epam/ai-dial-chat-themes:0.9.1
    ports:
      - "3001:8080"

  chat:
    ports:
      - "3000:3000"
    image: epam/ai-dial-chat:0.26.0
    depends_on:
      - themes
      - core
    environment:
      NEXTAUTH_SECRET: "secret"
      THEMES_CONFIG_HOST: "http://themes:8080"
      DIAL_API_HOST: "http://core:8080"
      DIAL_API_KEY: "dial_api_key"
      ENABLED_FEATURES: "conversations-section,prompts-section,top-settings,..."
      KEEP_ALIVE_TIMEOUT: ${CHAT_KEEP_ALIVE_TIMEOUT:-20000}

  redis:
    image: redis:7.2.4-alpine3.19
    restart: always
    ports:
      - "6379:6379"
    command: >
      redis-server
      --maxmemory 2000mb
      --maxmemory-policy volatile-lfu
      --save ""
      --appendonly no
      --loglevel warning
    mem_limit: 2200M

  core:
    user: ${UID:-root}
    ports:
      - "8080:8080"
    image: epam/ai-dial-core:0.25.1
    environment:
      'AIDIAL_SETTINGS': '/opt/settings/settings.json'
      'JAVA_OPTS': '-Dgflog.config=/opt/settings/gflog.xml'
      'LOG_DIR': '/app/log'
      'STORAGE_DIR': '/app/data'
      'aidial.config.files': '["/opt/config/config.json"]'
      'aidial.storage.overrides': '{ "jclouds.filesystem.basedir": "data" }'
      'aidial.redis.singleServerConfig.address': 'redis://redis:6379'
    depends_on:
      - redis
    volumes:
      - ./settings:/opt/settings
      - ${DIAL_DIR:-.}/core:/opt/config
      - ${DIAL_DIR:-.}/core-logs/:/app/log
      - ${DIAL_DIR:-.}/core-data/:/app/data
```

## Services

### `themes`

Static file server for Chat theme CSS, images, and configuration. Served on port 8080 inside the container. See [`epam/ai-dial-chat-themes`](https://github.com/epam/ai-dial-chat-themes).

### `chat`

DIAL Chat web application. Connects to Core for API access and Themes for UI assets. Uses API key authentication in quick start mode (no Keycloak). See [DIAL Chat configuration](../chat-configuration) for all environment variables.

### `redis`

In-memory cache for DIAL Core. Configured as a volatile cache (no persistence). See [Redis settings](../core/settings-json/redis) for DIAL Core connection options.

### `core`

DIAL Core API server. Mounts `settings.json`, `gflog.xml`, and `config.json` from the host. See [DIAL Core configuration](../core/index) for all configuration files.

Key volume mounts:

| Mount | Purpose |
|-------|---------|
| `./settings:/opt/settings` | `settings.json` and `gflog.xml` |
| `./core:/opt/config` | `config.json` |
| `./core-logs:/app/log` | Log output directory |
| `./core-data:/app/data` | Filesystem blob storage |
