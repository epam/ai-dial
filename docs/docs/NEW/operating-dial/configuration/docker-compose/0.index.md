# Docker Compose configuration

DIAL provides ready-to-use Docker Compose setups for local development and evaluation. The compose files and their supporting `.env` files are maintained in the [AI DIAL repository](https://github.com/epam/ai-dial).

## File inventory

| Directory | Files | Purpose |
|-----------|-------|---------|
| [`dial-docker-compose/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose) | `common.yml` + per-setup `docker-compose.yml` | Quick start setups (echo app, models, Ollama, vLLM) |
| [`dial-docker-compose-advanced/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced) | `docker-compose-base.yml` + `docker-compose-full.yml` | Production-like setup (Keycloak, Admin Panel, RAG) |

## Compose file structure

Quick start setups use the `include` directive to share common service definitions:

```yaml
include:
  - path: ../common.yml
    env_file: ./.env
```

This means [`common.yml`](common-yml) is the foundation — it defines core services (Chat, Core, Redis, Themes). Each setup-specific file adds only the services and overrides needed for that scenario.

The advanced setup uses its own base file (`docker-compose-base.yml`) that includes all services needed for an authenticated deployment.

## .env file reference

Docker Compose setups use `.env` files for environment variable substitution. Variables defined in `.env` are available as `${VAR}` in compose files.

### Variable categories

| Category | Variables | Description |
|---|---|---|
| Ports | `DIAL_CHAT_HOST_PORT`, `DIAL_CORE_HOST_PORT`, `DIAL_ADMIN_FRONTEND_HOST_PORT`, `KC_PORT` | Host ports for each service. Allows running multiple DIAL instances on different ports. |
| Networking | `COMMON_HOST` | Hostname used in URLs for inter-service communication and Keycloak issuer configuration. |
| Auth secrets | `NEXTAUTH_SECRET`, `AUTH_KEYCLOAK_SECRET` | NextAuth session encryption key and Keycloak client secret. Both are required for authenticated deployments. |
| Encryption | `DIAL_CORE_ENCRYPTION_SECRET`, `DIAL_CORE_ENCRYPTION_KEY` | AES encryption parameters for DIAL Core blob storage prefix hashing. |
| Keycloak | `KC_ADMIN_PASSWORD` | Initial Keycloak admin password. |
| Database | `H2_DATASOURCE_ENCRYPTED_FILE_KEY`, `H2_DATASOURCE_MASTER_KEY`, `H2_DATASOURCE_PASSWORD` | Admin Backend H2 database encryption and access credentials. |

### Example .env file

From the [advanced Docker Compose setup](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced):

```ini
# Ports
DIAL_CHAT_HOST_PORT=3000
DIAL_CORE_HOST_PORT=8080
DIAL_ADMIN_FRONTEND_HOST_PORT=3100
KC_PORT=8180
COMMON_HOST=localhost

# Secrets
NEXTAUTH_SECRET=<generate-with-openssl-rand-base64-32>
AUTH_KEYCLOAK_SECRET=<keycloak-client-secret>
DIAL_CORE_ENCRYPTION_SECRET=<random-string>
DIAL_CORE_ENCRYPTION_KEY=<random-string>
KC_ADMIN_PASSWORD=<keycloak-admin-password>
H2_DATASOURCE_ENCRYPTED_FILE_KEY=<random-string>
H2_DATASOURCE_MASTER_KEY=<random-string>
H2_DATASOURCE_PASSWORD=<random-string>
```
