# Supported providers

DIAL ships three model adapters that translate provider-specific APIs into the Unified API. This page lists the providers each adapter supports, the model families available through each, and the authentication methods required.

## Adapter-provider mapping

| Provider | Adapter | Authentication |
|---|---|---|
| Azure OpenAI | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key or Azure AD token |
| OpenAI Platform | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key |
| Azure AI Foundry | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key or Azure AD token |
| Mistral | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key |
| Anthropic (direct) | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key |
| Databricks | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | API key |
| vLLM / Ollama | [OpenAI Adapter](https://github.com/epam/ai-dial-adapter-openai) | None (local) |
| AWS Bedrock | [Bedrock Adapter](https://github.com/epam/ai-dial-adapter-bedrock) | AWS IAM credentials |
| Google Vertex AI | [Vertex AI Adapter](https://github.com/epam/ai-dial-adapter-vertexai) | GCP service account or ADC |

## OpenAI-compatible models

Any model that exposes an OpenAI-compatible API can be accessed through the OpenAI Adapter. This includes self-hosted models served by vLLM, Ollama, TGI, and similar inference servers. Configure the adapter's endpoint to point to the model server and use the appropriate deployment environment variables.

For adapter environment variable reference, see [Adapter configuration](../../operating-dial/configuration/4.adapter-configuration.md).

## How to enable a model

Models are registered as deployments in DIAL Core through one of two methods:

- **DIAL Core config.json** — add a model entry with the adapter endpoint and upstream configuration. See the [config.json models reference](../../operating-dial/configuration/core/config-json/1.models.md).
- **DIAL Admin** — add models through the administration UI. See [Admin Panel: Models](/docs/tutorials/3.admin/entities-models.md).

## Adding an unsupported provider

Organizations that need to integrate a provider not covered by the official adapters can build a custom adapter using [DIAL SDK](https://github.com/epam/ai-dial-sdk). The [Tutorial: custom adapter](1.tutorial-custom-adapter.md) walks through the process from project setup to deployment.
