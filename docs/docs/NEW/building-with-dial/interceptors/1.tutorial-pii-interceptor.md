# Tutorial: PII-redacting interceptor

In this tutorial, you will build a generic interceptor that detects personally identifiable information (PII) in user messages, replaces it with placeholders before the request reaches the model, and restores the original values in the response. By the end, you will have a working interceptor running alongside DIAL Core and a model.

## Prerequisites

- Python 3.11 or later installed
- Docker and Docker Compose installed
- Basic familiarity with Python and DIAL concepts ([What are interceptors](index))

## What you will build

A chat completion interceptor that:

1. Scans every user message for names, organizations, and locations using a keyword-based detector.
2. Replaces detected entities with numbered placeholders (e.g., `John Smith` → `[PERSON_1]`).
3. Forwards the anonymized request to the model.
4. Replaces placeholders in the model's response with the original values.

The interceptor runs as a standalone service and is wired into DIAL Core as a local interceptor on a model deployment.

## Project structure

By the end of this tutorial, your project directory will look like this:

```
pii-interceptor/
├── app.py                  # Application entry point — registers the interceptor on DIALApp
├── interceptor.py          # Interceptor logic — PII detection and replacement
├── requirements.txt        # Python dependencies
├── Dockerfile              # Container image for the interceptor service
├── docker-compose.yml      # Runs DIAL Core + interceptor + DIAL Chat together
└── config.json             # DIAL Core configuration — declares and assigns the interceptor
```

## Step 2: Implement the interceptor

Create `pii-interceptor/interceptor.py`:

```python
import re
from typing_extensions import override
from aidial_interceptors_sdk.chat_completion.base import (
    ChatCompletionInterceptor,
)
from aidial_interceptors_sdk.utils.not_given import NotGiven


class PIIRedactorInterceptor(ChatCompletionInterceptor):
    replacements: dict[str, str] = {}
    reverse_replacements: dict[str, str] = {}
    counter: int = 0

    PII_PATTERNS: dict[str, str] = {
        "EMAIL": r"\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b",
        "PHONE": r"\b\d{3}[-.]?\d{3}[-.]?\d{4}\b",
        "SSN": r"\b\d{3}-\d{2}-\d{4}\b",
    }

    def _anonymize(self, text: str) -> str:
        for label, pattern in self.PII_PATTERNS.items():
            for match in re.finditer(pattern, text):
                original = match.group()
                if original not in self.replacements:
                    self.counter += 1
                    placeholder = f"[{label}_{self.counter}]"
                    self.replacements[original] = placeholder
                    self.reverse_replacements[placeholder] = original
                text = text.replace(original, self.replacements[original])
        return text

    def _deanonymize(self, text: str) -> str:
        for placeholder, original in self.reverse_replacements.items():
            text = text.replace(placeholder, original)
        return text

    @override
    async def on_request_message(self, path, message: dict) -> list[dict]:
        if content := message.get("content"):
            message["content"] = self._anonymize(content)
        return [message]

    @override
    async def on_response_message(
        self, path, message: dict | NotGiven | None
    ) -> dict | NotGiven | None:
        if isinstance(message, dict):
            if content := message.get("content"):
                message["content"] = self._deanonymize(content)
        return message
```

This interceptor uses regular expressions to detect emails, phone numbers, and Social Security numbers. Production interceptors typically use NLP libraries like spaCy — see the [spaCy anonymizer example](examples) for that approach.

**Verify:** `python -c "import interceptor"` runs without errors.

## Step 3: Create the application entry point

Create `pii-interceptor/app.py`. This file wires the interceptor class into a `DIALApp` instance so it can serve requests at the standard DIAL endpoint path:

```python
import os

from aidial_sdk import DIALApp
from aidial_sdk.telemetry.types import TelemetryConfig
from aidial_interceptors_sdk.chat_completion import (
    interceptor_to_chat_completion,
)
from aidial_interceptors_sdk.utils._http_client import get_http_client

from interceptor import PIIRedactorInterceptor

DIAL_URL = os.environ.get("DIAL_URL", "http://core:8080")


async def client_factory():
    return get_http_client()


app = DIALApp(
    dial_url=DIAL_URL,
    telemetry_config=TelemetryConfig(),
    add_healthcheck=True,
    propagate_auth_headers=True,
    allow_extra_request_fields=True,
)

app.add_chat_completion(
    "pii-redactor",
    interceptor_to_chat_completion(
        PIIRedactorInterceptor, DIAL_URL, client_factory
    ),
)
```

**Verify:** `python -c "from app import app; print(type(app))"` prints `<class 'aidial_sdk.app.DIALApp'>`.

## Step 4: Create the Dockerfile

Create `pii-interceptor/Dockerfile`:

```dockerfile
FROM python:3.11-slim

WORKDIR /app

COPY requirements.txt .
RUN pip install --no-cache-dir -r requirements.txt

COPY interceptor.py app.py ./

EXPOSE 5000

CMD ["uvicorn", "app:app", "--host", "0.0.0.0", "--port", "5000"]
```

## Step 5: Create the Docker Compose configuration

Create `pii-interceptor/docker-compose.yml`. This starts DIAL Core, the interceptor service, and DIAL Chat:

```yaml
version: "3.8"

services:
  core:
    image: epam/ai-dial-core:0.23.0
    platform: linux/amd64
    environment:
      JAVA_OPTS: "-Xmx256m"
      aidial.config.files: '["/opt/config/config.json"]'
    ports:
      - "8080:8080"
    volumes:
      - ./config.json:/opt/config/config.json

  pii-interceptor:
    build: .
    environment:
      DIAL_URL: "http://core:8080"
      LOG_LEVEL: "DEBUG"
    depends_on:
      - core

  chat:
    image: epam/ai-dial-chat:0.22.0
    platform: linux/amd64
    environment:
      NEXT_PUBLIC_DIAL_API_HOST: "http://core:8080"
    ports:
      - "3000:3000"
    depends_on:
      - core
```

:::note
This example uses a minimal setup. Replace the model configuration with your own adapter (e.g., OpenAI, Bedrock, or Ollama) depending on your environment.
:::

## Step 6: Configure DIAL Core

Create `pii-interceptor/config.json`. This file declares the interceptor and assigns it to a model deployment. Docker Compose mounts it into the DIAL Core container at `/opt/config/config.json`:

```json
{
  "routes": {
    "default": {
      "paths": ["/v1/(.*)"]
    }
  },
  "interceptors": {
    "pii-redactor": {
      "endpoint": "http://pii-interceptor:5000/openai/deployments/pii-redactor/chat/completions",
      "displayName": "PII Redaction",
      "description": "Anonymizes PII in requests and responses"
    }
  },
  "models": {
    "chat-gpt-4o-mini": {
      "type": "chat",
      "displayName": "GPT-4o Mini (PII-protected)",
      "endpoint": "<YOUR_MODEL_ADAPTER_ENDPOINT>",
      "interceptors": ["pii-redactor"]
    }
  }
}
```

Replace `<YOUR_MODEL_ADAPTER_ENDPOINT>` with the endpoint of your model adapter.

## Step 7: Build and run

```bash
docker compose up --build
```

**Verify:** All three services start without errors. The interceptor service logs `Uvicorn running on http://0.0.0.0:5000`.

## Step 8: Test the interceptor

Open DIAL Chat at `http://localhost:3000` and select the "GPT-4o Mini (PII-protected)" model.

Send a message containing PII:

```
My email is john.doe@example.com and my phone is 555-123-4567.
Can you confirm my contact details?
```

Check the interceptor service logs. You should see the anonymized request where the email and phone number are replaced with placeholders like `[EMAIL_1]` and `[PHONE_1]`.

The model receives the anonymized version and responds without seeing the real PII. The interceptor restores the original values in the response before it reaches you.
