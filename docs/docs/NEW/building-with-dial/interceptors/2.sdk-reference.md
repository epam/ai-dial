# Interceptors SDK reference

The DIAL Interceptors SDK is a Python framework for creating interceptors that transform chat completion and embedding requests flowing through DIAL Core. This page documents the public API: base classes, callback methods, wiring functions, and the client used to call upstream deployments.

Source repository: [github.com/epam/ai-dial-interceptors-sdk](https://github.com/epam/ai-dial-interceptors-sdk)

## Installation

```bash
pip install aidial-interceptors-sdk
```

**Python version:** 3.11 or later (< 3.13).

**Dependencies:** The Interceptors SDK depends on `aidial-sdk` (DIAL SDK), `fastapi`, `httpx`, and `openai`.

## Repository structure

The SDK source at [github.com/epam/ai-dial-interceptors-sdk](https://github.com/epam/ai-dial-interceptors-sdk) is organized as follows:

```
aidial_interceptors_sdk/
├── chat_completion/
│   ├── base.py                  # ChatCompletionInterceptor, ChatCompletionNoOpInterceptor
│   ├── adapter.py               # interceptor_to_chat_completion() wiring function
│   ├── request_handler.py       # RequestHandler — request-level callbacks
│   ├── request_message_handler.py  # RequestMessageHandler — message-level request callbacks
│   ├── response_handler.py      # ResponseHandler — response-level callbacks
│   ├── response_message_handler.py # ResponseMessageHandler — message-level response callbacks (code-generated)
│   ├── element_path.py          # ElementPath context object
│   ├── index_mapper.py          # Stage index mapping utilities
│   └── helpers.py               # Traversal helpers for nested dicts/lists
├── embeddings/
│   ├── base.py                  # EmbeddingsInterceptor, EmbeddingsNoOpInterceptor
│   └── adapter.py               # interceptor_to_embeddings() wiring function
├── dial_client.py               # DialClient — HTTP client for upstream calls and file storage
├── error.py                     # EarlyStreamExit exception
├── utils/
│   ├── not_given.py             # NotGiven sentinel type
│   ├── storage.py               # FileStorage client
│   ├── streaming.py             # Stream annotation and mapping utilities
│   └── ...                      # Internal helpers
└── examples/                    # Example interceptor implementations (see Examples page)
    ├── app.py                   # Entry point for the examples service
    ├── app_factory.py           # create_app() factory using DIALApp
    ├── registry.py              # Registry mapping deployment names to interceptor classes
    └── chat_completion/         # Chat completion interceptor examples
        ├── pirate.py            # Simplest pre-interceptor
        ├── blacklisted_words.py # Generic interceptor (request + response)
        ├── pii_anonymiser/      # PII anonymization with spaCy
        ├── google_dlp_anonymiser/ # PII anonymization with Google DLP
        └── ...
```

## Chat completion interceptors

### `ChatCompletionInterceptor`

Defined in [`aidial_interceptors_sdk/chat_completion/base.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/base.py).

The base class for chat completion interceptors. Inherits from `RequestHandler` ([`request_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/request_handler.py)) and `ResponseHandler` ([`response_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/response_handler.py)), both Pydantic `BaseModel` subclasses.

**Instance attributes (injected by the adapter):**

| Attribute | Type | Description |
|---|---|---|
| `dial_client` | `DialClient` | HTTP client for calling upstream deployments |
| `request` | `Request` | DIAL SDK request object (from `aidial_sdk.chat_completion`) |
| `response` | `Response` | DIAL SDK response object |
| `configuration` | `BaseModel \| None` | Parsed interceptor configuration, if a schema is defined |

**Property:**

| Property | Type | Description |
|---|---|---|
| `upstream_deployment_id` | `str \| None` | The DIAL deployment this interceptor is assigned to (from `X-DIAL-DEPLOYMENT-ID` header). Available since DIAL Core 0.43.0. |

#### Request callbacks

Override these methods to modify incoming requests. Callbacks execute in bottom-up order: message-level first, then list-level, then full request. Message-level callbacks are defined in [`request_message_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/request_message_handler.py); request-level callbacks are in [`request_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/request_handler.py).

| Method | Signature | Default behavior |
|---|---|---|
| `on_request_attachment` | `(path: ElementPath, attachment: dict) -> list[dict] \| dict` | Returns attachment unchanged |
| `on_request_attachments` | `(path: ElementPath, attachments: list[dict] \| NotGiven \| None) -> list[dict] \| NotGiven \| None` | Returns list unchanged |
| `on_request_state` | `(path: ElementPath, state: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns state unchanged |
| `on_request_custom_content` | `(path: ElementPath, custom_content: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns custom content unchanged |
| `on_request_stage` | `(path: ElementPath, stage: dict) -> list[dict] \| dict` | Returns stage unchanged |
| `on_request_stages` | `(path: ElementPath, stages: list[dict] \| NotGiven \| None) -> list[dict] \| NotGiven \| None` | Returns list unchanged |
| `on_request_message` | `(path: ElementPath, message: dict) -> list[dict]` | Returns `[message]` |
| `on_request_messages` | `(messages: list[dict]) -> list[dict]` | Returns list unchanged |
| `on_request` | `(request: dict) -> dict` | Returns request unchanged |

Methods returning `list[dict] | dict` allow adding or removing elements. Return a list to replace one element with multiple; return an empty list to remove it.

#### Response callbacks

Override these methods to modify outgoing responses. Response callbacks process streaming chunks — they are called for each chunk in the response stream. Message-level callbacks are defined in [`response_message_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/response_message_handler.py) (code-generated from the request handler); response-level callbacks are in [`response_handler.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/response_handler.py).

| Method | Signature | Default behavior |
|---|---|---|
| `on_response_attachment` | `(path: ElementPath, attachment: dict) -> list[dict] \| dict` | Returns attachment unchanged |
| `on_response_attachments` | `(path: ElementPath, attachments: list[dict] \| NotGiven \| None) -> list[dict] \| NotGiven \| None` | Returns list unchanged |
| `on_response_state` | `(path: ElementPath, state: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns state unchanged |
| `on_response_custom_content` | `(path: ElementPath, custom_content: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns custom content unchanged |
| `on_response_stage` | `(path: ElementPath, stage: dict) -> list[dict] \| dict` | Returns stage unchanged |
| `on_response_stages` | `(path: ElementPath, stages: list[dict] \| NotGiven \| None) -> list[dict] \| NotGiven \| None` | Returns list unchanged |
| `on_response_message` | `(path: ElementPath, message: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns message unchanged |
| `on_response_finish_reason` | `(path: ElementPath, finish_reason: str \| NotGiven \| None) -> str \| NotGiven \| None` | Returns unchanged |
| `on_response_choice` | `(path: ElementPath, choice: dict) -> list[dict] \| dict` | Returns choice unchanged |
| `on_response_choices` | `(choices: list[dict] \| NotGiven \| None) -> list[dict] \| NotGiven \| None` | Returns list unchanged |
| `on_response_usage` | `(usage: dict \| NotGiven \| None) -> dict \| NotGiven \| None` | Returns usage unchanged |
| `on_stream_chunk` | `(chunk: dict) -> None` | Sends chunk to response |

#### Stream lifecycle callbacks

| Method | Signature | Description |
|---|---|---|
| `on_stream_start` | `() -> None` | Called before processing the upstream response stream. Use to inject stages or initialize state. |
| `on_stream_end` | `() -> None` | Called after all chunks are processed. Use to flush buffers or close stages. |
| `on_stream_error` | `(error: AnnotatedException) -> None` | Called when the upstream returns an error. Default: raises the error. |
| `call_upstreams` | `(request: RequestDict, call_upstream: Callable) -> AsyncIterator[AnnotatedValue]` | Controls how the upstream is called. Default: single call. Override for fan-out patterns (e.g., the replicator example). |

#### Utility methods

| Method | Signature | Description |
|---|---|---|
| `reserve_stage_index` | `(choice_idx: int) -> int` | Reserves a stage index for injecting custom stages into the response. |
| `send_chunk` | `(chunk: BaseChunk \| dict) -> None` | Sends a chunk directly to the response stream. |
| `get_configuration` | `(cls: type[T]) -> T` | Returns the typed interceptor configuration. Raises `ValueError` if no configuration is set. |
| `configuration_schema` | `classmethod() -> type[BaseModel] \| None` | Returns the Pydantic model class for the interceptor's configuration schema. Return `None` (default) to disable the `/configuration` endpoint. |

### `ChatCompletionNoOpInterceptor`

A subclass of `ChatCompletionInterceptor` that passes requests and responses through unchanged. Useful for testing the interceptor chain without side effects.

## Embeddings interceptors

### `EmbeddingsInterceptor`

Defined in [`aidial_interceptors_sdk/embeddings/base.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/embeddings/base.py).

The base class for embeddings interceptors. An abstract `BaseModel` with a simpler API than chat completion interceptors.

**Instance attributes (injected by the adapter):**

| Attribute | Type | Description |
|---|---|---|
| `dial_client` | `DialClient` | HTTP client for calling upstream deployments |

#### Methods

| Method | Signature | Default behavior |
|---|---|---|
| `modify_input` | `(input: str) -> str` | Returns input unchanged |
| `modify_embedding` | `(embedding: str \| list[float]) -> str \| list[float]` | Returns embedding unchanged |
| `modify_request` | `(request: dict) -> dict` | Calls `modify_input` on each input string in the request |
| `modify_response` | `(response: dict) -> dict` | Calls `modify_embedding` on each embedding in the response |

For simple transformations, override `modify_input` and/or `modify_embedding`. For full control, override `modify_request` and/or `modify_response`.

### `EmbeddingsNoOpInterceptor`

A subclass of `EmbeddingsInterceptor` that passes requests and responses through unchanged.

## Wiring functions

These functions convert interceptor classes into DIAL SDK implementations that can be registered on a `DIALApp`.

### `interceptor_to_chat_completion`

Defined in [`aidial_interceptors_sdk/chat_completion/adapter.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/adapter.py).

```python
interceptor_to_chat_completion(
    cls: type[ChatCompletionInterceptor],
    dial_url: str,
    client_factory: HTTPClientFactory,
) -> ChatCompletion
```

Wraps a `ChatCompletionInterceptor` subclass into a `ChatCompletion` instance (from `aidial_sdk`). The returned object handles request parsing, upstream calling, stream processing, and configuration endpoint exposure.

### `interceptor_to_embeddings`

Defined in [`aidial_interceptors_sdk/embeddings/adapter.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/embeddings/adapter.py).

```python
interceptor_to_embeddings(
    cls: type[EmbeddingsInterceptor],
    dial_url: str,
    client_factory: HTTPClientFactory,
) -> Embeddings
```

Wraps an `EmbeddingsInterceptor` subclass into an `Embeddings` instance (from `aidial_sdk`).

### `HTTPClientFactory`

Type alias: `Callable[[], Awaitable[httpx.AsyncClient]]`

An async factory function that returns an `httpx.AsyncClient`. The factory is called once per request to create the HTTP client used for upstream communication.

## `DialClient`

Defined in [`aidial_interceptors_sdk/dial_client.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/dial_client.py).

The HTTP client provided to interceptors for calling the upstream deployment and accessing DIAL file storage.

| Attribute | Type | Description |
|---|---|---|
| `dial_url` | `str` | DIAL Core server URL |
| `client` | `AsyncAzureOpenAI` | OpenAI-compatible client configured to call the upstream through DIAL Core |
| `storage` | `FileStorage` | Client for reading and writing files on DIAL file storage |

The `client` attribute uses the `openai` library's `AsyncAzureOpenAI` class, configured with the reserved deployment name `interceptor`. DIAL Core resolves this to the next interceptor in the chain or the target deployment.

## `ElementPath`

Defined in [`aidial_interceptors_sdk/chat_completion/element_path.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/chat_completion/element_path.py).

A context object passed to request and response callbacks that identifies the current position in the message tree.

| Property | Type | Description |
|---|---|---|
| `message_idx` | `int \| None` | Index of the current message in the messages array |
| `choice_idx` | `int \| None` | Index of the current choice in the response |
| `stage_idx` | `int \| None` | Index of the current stage |
| `attachment_idx` | `int \| None` | Index of the current attachment |
| `choice_ctx` | `ChoiceContext \| None` | Choice-level context including stage index mapper |

## App wiring example

A minimal interceptor service with one chat completion and one embeddings interceptor. For a full working example, see [`aidial_interceptors_sdk/examples/app_factory.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/app_factory.py) and [`aidial_interceptors_sdk/examples/app.py`](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/app.py) in the SDK repository.

```python
from aidial_sdk import DIALApp
from aidial_sdk.telemetry.types import TelemetryConfig
from aidial_interceptors_sdk.chat_completion import (
    interceptor_to_chat_completion,
)
from aidial_interceptors_sdk.embeddings.adapter import (
    interceptor_to_embeddings,
)
from aidial_interceptors_sdk.utils._http_client import get_http_client

DIAL_URL = "http://dial-core:8080"


async def client_factory():
    return get_http_client()


app = DIALApp(
    dial_url=DIAL_URL,
    telemetry_config=TelemetryConfig(),
    add_healthcheck=True,
    propagate_auth_headers=True,
    allow_extra_request_fields=True,
)

app.add_chat_completion(
    "my-interceptor",
    interceptor_to_chat_completion(MyChatInterceptor, DIAL_URL, client_factory),
)

app.add_embeddings(
    "my-emb-interceptor",
    interceptor_to_embeddings(MyEmbInterceptor, DIAL_URL, client_factory),
)
```

Run with uvicorn:

```bash
uvicorn app:app --host 0.0.0.0 --port 5000
```

This exposes endpoints at:
- `http://localhost:5000/openai/deployments/my-interceptor/chat/completions`
- `http://localhost:5000/openai/deployments/my-emb-interceptor/embeddings`
