# Interceptor examples

The [Interceptors SDK repository](https://github.com/epam/ai-dial-interceptors-sdk) includes a catalog of example interceptors covering common use cases. These examples are provided for reference and demonstration — they are not production-ready without further hardening.

All examples are in the `aidial_interceptors_sdk/examples/` directory and can be run as a single DIAL service.

## Running the examples

Install the SDK with the examples extra and start the server:

```bash
pip install uvicorn python-dotenv "aidial-interceptors-sdk[examples]"
echo "DIAL_URL=<YOUR_DIAL_CORE_URL>" > .env
uvicorn "aidial_interceptors_sdk.examples.app:app" --host "0.0.0.0" --port 5000 --env-file ./.env
```

Each interceptor is exposed at its own endpoint, for example:
- `http://localhost:5000/openai/deployments/spacy-anonymizer/chat/completions`
- `http://localhost:5000/openai/deployments/normalize-vector/embeddings`

## Chat completion interceptors

### PII anonymization

#### spaCy anonymizer

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `spacy-anonymizer` (also `pii-anonymizer`) |
| Source | [pii_anonymiser/](https://github.com/epam/ai-dial-interceptors-sdk/tree/development/aidial_interceptors_sdk/examples/chat_completion/pii_anonymiser) |
| Dependencies | `spacy`, `en_core_web_sm` model |

Anonymizes PII in the request using [spaCy](https://spacy.io/) named entity recognition, calls the upstream, and restores the original values in the response. Detected entity types are configurable via the `SPACY_ANONYMIZER_LABELS_TO_REDACT` environment variable (default: `PERSON,ORG,GPE,PRODUCT`).

The interceptor creates two response stages: "Anonymized request" (shows what the model received) and "Original response" (shows the model's raw output before deanonymization).

#### Google DLP anonymizer

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `google-dlp-anonymizer` |
| Source | [google_dlp_anonymiser/](https://github.com/epam/ai-dial-interceptors-sdk/tree/development/aidial_interceptors_sdk/examples/chat_completion/google_dlp_anonymiser) |
| Dependencies | `google-cloud-dlp` |

Anonymizes PII using the [Google DLP API](https://cloud.google.com/sensitive-data-protection/docs/reference/rest/v2/projects.content/deidentify). Entity types can be configured per deployment via the interceptor configuration endpoint or globally via the `GOOGLE_DLP_ANONYMIZER_INFO_TYPES_TO_DE_IDENTIFY` environment variable.

Requires `GCP_PROJECT_ID` and the [DLP User](https://cloud.google.com/sensitive-data-protection/docs/iam-roles#dlp.user) IAM role.

### Content filtering

#### Blacklisted words

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `reject-blacklisted-words` |
| Source | [blacklisted_words.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/blacklisted_words.py) |

Rejects requests and responses containing words from a configurable blocklist. Demonstrates both request-side and response-side validation using `on_request_message` and `on_response_message`.

#### Reject external links

| Property | Value |
|---|---|
| Category | Pre |
| Deployment name | `reject-external-links` |
| Source | [reject_external_links.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/reject_external_links.py) |

Rejects any URL in DIAL attachments that does not point to DIAL Core storage. Prevents data exfiltration through attachment URLs.

### Prompt modification

#### Reply as pirate

| Property | Value |
|---|---|
| Category | Pre |
| Deployment name | `reply-as-pirate` |
| Source | [pirate.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/pirate.py) |

Injects "Reply as a pirate" into the system prompt. The simplest possible interceptor — demonstrates `on_request_messages` in about 10 lines of code.

### Observability

#### Statistics reporter

| Property | Value |
|---|---|
| Category | Post |
| Deployment name | `statistics-reporter` |
| Source | [statistics_reporter.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/statistics_reporter.py) |

Collects streaming statistics (tokens per second, finish reason, completion tokens) and reports them in a new response stage when the stream finishes.

#### Langfuse integration

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `langfuse` |
| Source | [langfuse/](https://github.com/epam/ai-dial-interceptors-sdk/tree/development/aidial_interceptors_sdk/examples/chat_completion/langfuse) |

### Media processing

#### Image watermark

| Property | Value |
|---|---|
| Category | Post |
| Deployment name | `image-watermark` |

### Performance

#### Cache

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `cache` |
| Source | [cache.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/cache.py) |

Caches incoming chat completion requests. Not production-ready — provided as a demonstration of the caching pattern.

#### Replicator

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `replicator:{n}` |
| Source | [replicator.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/replicator.py) |

Calls the upstream N times and combines the responses. Demonstrates the `call_upstreams` override for fan-out patterns and path parameter usage (`{n:int}`).

#### Whitespace accumulator

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `whitespace-accumulator` |
| Source | [whitespace_accumulator.py](https://github.com/epam/ai-dial-interceptors-sdk/blob/development/aidial_interceptors_sdk/examples/chat_completion/whitespace_accumulator.py) |

Buffers whitespace-only stream chunks and flushes them when non-whitespace content arrives or a timeout is reached. Reduces streaming noise. Works only for the first response choice.

### No-op

| Property | Value |
|---|---|
| Category | Generic |
| Deployment name | `no-op` |
