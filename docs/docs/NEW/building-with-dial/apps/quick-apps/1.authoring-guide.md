# Author a Quick App 2.0

This guide walks through creating a Quick App 2.0 using the UI wizard in DIAL Chat. You will set a model, write a system prompt, add tools, and test the app before publishing.

## Prerequisites

- Access to DIAL Chat with an authenticated user account.
- At least one language model deployed in your DIAL instance.
- (Optional) A Tool Set registered in DIAL Core if you want to add MCP tools.

---

## Step 1: Open the app wizard

In DIAL Chat, click the **+** button or the app selector, then select **Quick App 2.0** from the dropdown.

![DIAL Chat '+' button dropdown open, listing 'Quick App 2.0' and 'Quick app' as separate options](./img/add-app-dropdown.png)

The wizard opens at the **General info** step.

---

## Step 2: General info

Fill in the app's identity fields:

| Field | Required | Description |
|---|---|---|
| **Name** | Yes | The app's display name. |
| **Version** | Yes | Semantic version string (e.g., `1.0.0`). |
| **Icon** | No | Upload a PNG or SVG icon. |
| **Description** | No | A short description shown to users in the app list. |
| **Topics** | No | Category tags that help users find the app in the Marketplace. |

![Quick App 2.0 wizard — General info step with Name, Version, Description, and Topics fields filled in](./img/authoring-general-info.png)

Click **Next** to proceed.

---

## Step 3: App settings

The second wizard step configures the model, instructions, and tools.

![Quick App 2.0 wizard — App settings step showing Model selector, Instructions textarea, Temperature slider, and Agents & Tool Sets panel](./img/authoring-app-settings.png)

### Model

Select the language model for the orchestrator. The model processes user messages and decides when to invoke tools. Choose a model capable of function calling.

### Instructions

Write the system prompt that guides the model's behavior. The instructions field is a full-featured text area — use plain text or Markdown-formatted content.

**Tips for effective instructions:**

- State the app's purpose clearly in the first sentence.
- Describe what the model should and should not do.
- If tools are connected, tell the model when and how to use them.
- Use `{{variable}}` syntax for values that may change across deployments.

![Quick App 2.0 wizard — Instructions field with a multi-line system prompt typed in](./img/authoring-instructions.png)

### Temperature

The temperature slider controls response randomness:

- `0.0–0.3` — More deterministic. Good for factual, structured tasks.
- `0.4–0.7` — Balanced. Good for most conversational use cases.
- `0.8–1.0` — More creative. Good for writing, brainstorming.

### Attachment types and count

Optionally restrict which file types users can attach, and set a maximum number of attachments per message. Leave these blank to allow all types with no limit.

### Agents & Tool Sets

The **Agents & Tool Sets** panel lists all tools connected to this app. Click **+ Add** to attach a Tool Set or DIAL agent.

See [Add tools and agents to a Quick App 2.0](./working-with-tools-and-agents) for a complete walkthrough.

### Document relative URLs

To ground the model's responses in a document stored in DIAL, enter the file's relative URL here. This enables RAG (Retrieval Augmented Generation) using the document as a knowledge source.

---

## Step 4: Preview and test

While in the wizard, use the **Preview** panel on the right to send test messages and verify the app behaves as expected.

![Quick App 2.0 wizard — Preview panel on the right with a test message sent and a model response returned](./img/authoring-preview.png)

Test the main workflow and at least one edge case before saving:

- Does the model follow the instructions?
- Do tools get invoked when expected?
- Is the response quality acceptable at the chosen temperature?

---

## Step 5: Configure starters (optional)

Starters are suggested prompts shown to users at the start of a conversation. Click **Add starter** to add up to several suggested messages.

Good starters:

- Represent the most common user tasks.
- Are short enough to display fully in the UI.
- Are phrased as questions or requests, not commands.

---

## Step 6: Save the app

Click **Create** (or **Save** when editing). The app is saved to your personal workspace and is private by default.

---

## Create via API

You can also create Quick App 2.0 instances programmatically using the DIAL Core API.

**Endpoint:** `POST /v1/applications/{appId}`

**Example request body:**

```json
{
  "displayName": "My Research Assistant",
  "description": "Searches the web and summarizes findings.",
  "applicationTypeSchemaId": "https://mydial.epam.com/custom_application_schemas/quickapps2",
  "applicationProperties": {
    "orchestrator": {
      "deployment": {
        "name": "gpt-4o-mini",
        "parameters": { "temperature": 0.5 }
      },
      "system_prompt": {
        "type": "custom",
        "variables": {},
        "content": "You are a research assistant. When users ask questions requiring current information, use available tools to search before answering."
      }
    },
    "contexts": [],
    "tool_sets": [],
    "starters": [
      "Find the latest news about AI regulation",
      "Search for recent papers on transformer architectures"
    ]
  }
}
```

Replace `{appId}` with a unique identifier for your app. See the [DIAL Core API reference](https://dialx.ai/dial_api#tag/Applications/operation/saveCustomApplication) for authentication and full request details.

## Edit an existing Quick App 2.0

To edit an app you own:

1. Open the app in DIAL Chat.
2. Click the **Settings** or **Edit** icon.
3. The wizard reopens with the current configuration pre-filled.
4. Make changes and click **Save**.

Co-editing is supported if you grant WRITE access to another user — changes from either user are saved in real time.
