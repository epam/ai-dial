# Share and manage Tool Set permissions

A Tool Set you create in DIAL Chat is private by default — only you can see and use it. This guide explains how to share it with specific users, publish it to the Marketplace, and manage credentials for published toolsets.

---

## Visibility model

| State | Who can use it |
|---|---|
| **Private** | Only the toolset creator. |
| **Shared** | Specific users or groups you grant access to. |
| **Published** | All users on the platform (subject to administrator approval and `userRoles` in `config.json`). |

---

## Share with specific users

To share a toolset with a colleague without publishing it:

1. Open the toolset card in **My Workspace → Toolsets**.
2. In the actions menu, select **Share**.
3. Enter the user's email address or DIAL username.
4. Grant **READ** or **WRITE** access:
   - **READ** — the user can add the toolset to their workspace and use it in Quick Apps.
   - **WRITE** — the user can also edit the toolset.
5. Click **Share**.

The recipient sees the toolset in their workspace after accepting the share invitation.

---

## Publish to Marketplace

Publishing makes the toolset available on the DIAL Marketplace for the broader user audience. Published toolsets require administrator approval before they go live.

### Request publication

1. Open the toolset card in **My Workspace → Toolsets**.
2. In the actions menu, click **Publish**.
3. In the **Publish to** field, select the target destination (a public folder or Marketplace category).
4. Review the toolset details — name, description, and version are shown to Marketplace users.
5. Click **Submit**.

![My Workspace — toolset publish dialog with 'Publish to' destination selector and Submit button](../img/toolset-publish.png)

The request enters a pending state. An administrator reviews and approves it. After approval, the toolset appears on the Marketplace.

---

## Publish with organization credentials

If the toolset requires authentication and you want all users to be able to use it without logging in individually, publish the toolset together with credentials:

1. Log in to the toolset first (so your credentials are attached to it).
2. In the toolset card actions menu, click **Publish**.
3. In the publication request form, enable the **Publish with credentials** option.
4. Complete the publish request.

![My Workspace — toolset publish dialog with 'Publish with credentials' checkbox enabled](../img/toolset-credentials.png)

After approval, the toolset card shows **ORG CREDS** for all users. Any user can use the toolset without authenticating separately.

:::note
Publishing with credentials shares your personal login tokens with all users who have access to the toolset. Only do this for service accounts or credentials that are intended for shared use.
:::

---

## Unpublish a toolset

Unpublishing removes the toolset from the Marketplace. Existing Quick Apps that reference the toolset will start returning errors.

1. Find the toolset on the Marketplace (filter: **Sources = Public**).
2. In the toolset's actions menu, select **Unpublish**.
3. Fill in the unpublish request form.
4. The toolset is withdrawn after the request is approved by an administrator.

The toolset owner, administrators, and users with WRITE access can submit an unpublish request.

---

## Remove a public toolset from your workspace

If you have added a public toolset to your workspace but no longer need it:

1. Find the toolset in **My Workspace → Toolsets**.
2. Click the bookmark icon on the toolset card to toggle it off.

This removes the toolset from your workspace only — it remains available on the Marketplace for others.

---

## Delete a toolset

To permanently delete a toolset you own:

1. Open the toolset card in **My Workspace → Toolsets**.
2. In the actions menu, select **Delete**.

:::warning
Deletion is permanent. Any Quick App that references this toolset will return errors. The deleted toolset is highlighted in red in the Quick App 2.0 editor.
:::

You can only delete toolsets you own.

---

## Administrator: manage organization credentials

Administrators can log in and out of public toolsets using organization credentials, which makes the toolset available to all users in the organization without individual authentication.

1. Navigate to **Marketplace → Toolsets**.
2. Click the toolset that requires authentication.
3. On the toolset info card, click **Manage creds**.
4. Choose one of:
   - **Entire organization credentials** — log in or out using organization credentials. This affects all users.
   - **My credentials** — log in or out with your personal credentials only.

![DIAL Marketplace — admin toolset info card showing 'Manage creds' panel with 'Entire organization credentials' and 'My credentials' login options](../img/toolset-admin-creds.png)

After logging in with organization credentials, the toolset shows **ORG CREDS** for all users.
