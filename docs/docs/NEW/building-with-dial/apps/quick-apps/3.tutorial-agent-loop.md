# Tutorial: Build a Quick App with an agent loop

In this tutorial you will build a **Research Assistant** Quick App 2.0 that combines three tool types in a single agent loop:

1. A **REST API tool** — fetches current weather data (Open-Meteo, free, no API key needed)
2. A **DIAL deployment tool** — uses a second model to summarize findings
3. An **MCP tool set** — connects to an external search or data service

By the end, you will have a working app that demonstrates how the orchestrator model calls multiple tools within a single conversation turn, with each tool call visible as a collapsible stage in DIAL Chat.

## What you will build

A Research Assistant that:
- Accepts a question from the user
- Calls the weather API if the question involves weather or location
- Uses the summarizer model to condense long results into bullet points
- Returns a final composed answer

![Completed Research Assistant in DIAL Chat — conversation with get_weather and summarize tool call stages both expanded, showing tool arguments and results](./img/tutorial-result.png)

---

## Prerequisites

- Access to DIAL Chat with an authenticated user account.
- At least one language model deployed in your DIAL instance that supports function calling (e.g., GPT-4o, Claude 3.5 Haiku).
- A second model deployment for summarization (can be a smaller, faster model).
- (Optional) A DIAL-managed Tool Set registered in DIAL Core for the MCP step.

---

## Step 1: Create the Quick App 2.0

In DIAL Chat, click **+** or the app selector and choose **Quick App 2.0**.

The wizard opens at **General info**.

Fill in the following:

| Field | Value |
|---|---|
| **Name** | Research Assistant |
| **Version** | 1.0.0 |
| **Description** | Answers research questions using live data and a summarization agent. |
| **Topics** | Research |

![Quick App 2.0 wizard — General info step with Name='Research Assistant', Version='1.0.0', and Description filled in](./img/tutorial-step1.png)

Click **Next**.

---

## Step 2: Configure the model and instructions

On the **App settings** step:

1. **Model** — Select a function-calling capable model (e.g., `gpt-4o-mini`).
2. **Temperature** — Set to `0.3` for consistent, factual responses.
3. **Instructions** — Paste the system prompt below.

### System prompt (copy-pasteable)

```
You are a Research Assistant. Your job is to answer user questions accurately using the tools available to you.

Guidelines:
- For questions involving current weather or climate data for a location, use the get_weather tool.
- For any response that is more than 200 words, use the summarize tool to condense it before responding to the user.
- If a tool call fails, tell the user clearly and provide the best answer you can from existing knowledge.
- Always cite the source of data when you use a tool result.
- Be concise. Prefer bullet points over paragraphs.
```

## Step 3: Add the weather REST API tool

Click **+ Add** in the **Agents & Tool Sets** panel.

In the tool set editor, paste the following JSON configuration for the Open-Meteo weather API (free, no authentication required):

```json
{
  "type": "rest-api",
  "name": "weather",
  "authorization": null,
  "tools": [
    {
      "type": "restapi-tool",
      "rest_api_method_info": {
        "method_url": "https://api.open-meteo.com/v1/forecast",
        "method_type": "get"
      },
      "open_ai_tool": {
        "type": "function",
        "function": {
          "name": "get_weather",
          "description": "Get the current temperature for a geographic location specified by latitude and longitude.",
          "parameters": {
            "type": "object",
            "properties": {
              "latitude": {
                "type": "number",
                "description": "Location latitude.",
                "parameter_info": { "type": "query", "key": "latitude" }
              },
              "longitude": {
                "type": "number",
                "description": "Location longitude.",
                "parameter_info": { "type": "query", "key": "longitude" }
              },
              "current": {
                "type": null,
                "const": "temperature_2m",
                "parameter_info": { "type": "query", "key": "current" }
              }
            },
            "required": ["latitude", "longitude", "current"]
          }
        }
      },
      "display": {
        "stage": {
          "name": "Fetching weather for ({latitude}, {longitude})",
          "show": true
        }
      }
    }
  ]
}
```

The `current` parameter uses `"type": null` with a `"const"` value — the model never sees this parameter; it is always sent as `temperature_2m`. The `display.stage.name` uses `{latitude}` and `{longitude}` placeholders so the actual coordinates appear in the DIAL Chat stage header.

After saving, the weather tool set appears in the Agents & Tool Sets list.

![Quick App 2.0 wizard — Agents & Tool Sets panel showing the weather REST API tool set listed after being added](./img/tutorial-tool-config.png)

## Step 4: Add the summarizer deployment tool

Click **+ Add** again. Paste the following JSON for a DIAL deployment tool that calls a second model:

```json
{
  "type": "dial-deployment",
  "name": "summarization",
  "tools": [
    {
      "type": "deployment-tool",
      "open_ai_tool": {
        "type": "function",
        "function": {
          "name": "summarize",
          "description": "Summarize a long piece of text into 3–5 concise bullet points.",
          "parameters": {
            "type": "object",
            "properties": {
              "text": {
                "type": "string",
                "description": "The text to summarize."
              }
            },
            "required": ["text"]
          }
        }
      },
      "deployment": {
        "name": "gpt-4o-mini",
        "parameters": { "temperature": 0.1 }
      },
      "system_prompt": {
        "type": "custom",
        "variables": {},
        "content": "You are a summarization expert. Produce exactly 3–5 bullet points. Each bullet must be a complete sentence. Omit filler and padding."
      },
      "content_propagation": {
        "propagate_history": false
      },
      "fallback_configuration": {
        "strategies": [
          {
            "type": "continue",
            "instructions": "The summarization tool is unavailable. Provide a brief summary yourself in 3–5 bullet points."
          }
        ],
        "display_error_in_stage": true
      },
      "display": {
        "stage": {
          "name": "Summarizing",
          "show": true
        }
      }
    }
  ]
}
```

Replace `"gpt-4o-mini"` with any model deployment ID available in your DIAL instance.

Key configuration choices:
- `"propagate_history": false` — the summarizer receives only the text argument, not the full conversation. This keeps the sub-call focused and fast.
- The `fallback_configuration` uses `continue` so the orchestrator gracefully handles summarizer failures.

## Step 5: (Optional) Add a DIAL-managed MCP tool

If your DIAL instance has a registered Tool Set (for example, a web search or database MCP server), add it using its DIAL ID:

```json
{
  "type": "dial-mcp",
  "dial_id": "your-registered-toolset-id",
  "transport": "HTTP",
  "allowed_tools": ["search"]
}
```

Replace `"your-registered-toolset-id"` with the actual DIAL ID from **My Workspace → Toolsets**.

---

## Step 6: Test the agent loop

Use the **Preview** panel on the right side of the wizard to send a test message.

**Test prompt:** `What is the current temperature in Paris? Give me a summary.`

Expected behavior:
1. The orchestrator model calls `get_weather` with Paris coordinates.
2. The weather result is returned.
3. The orchestrator calls `summarize` with the raw API response.
4. The summarizer returns 3–5 bullet points.
5. The orchestrator composes a final response citing the source.

In DIAL Chat, each tool call appears as a collapsible stage. Expand them to see the exact arguments and results.

![Research Assistant in DIAL Chat — get_weather and summarize tool call stages expanded, showing input arguments and API results for each](./img/tutorial-tool-stages.png)

---

## Step 7: Add starters and save

Click through to the **Starters** step. Add two or three example prompts:

- `What is the weather in Tokyo right now?`
- `Summarize the key findings from my last search.`
- `What is the climate difference between London and Madrid?`

Starters are shown to users at the start of a conversation and help them discover what the app can do.

Click **Create** to save the app. It is now in your personal workspace.

![Research Assistant app card visible in My Workspace after clicking Create](./img/tutorial-final.png)
