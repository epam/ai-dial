# Quick App 2.0 configuration reference

Complete reference for the Quick App 2.0 JSON schema (`$id: https://mydial.epam.com/custom_application_schemas/quickapps2`). Use this page when creating or editing Quick App 2.0 instances via the [DIAL Core API](https://dialx.ai/dial_api#tag/Applications/operation/saveCustomApplication) or `config.json`.

For a UI-based authoring walkthrough, see [Author a Quick App 2.0](../create-via-ui).

---

## Top-level structure

The `applicationProperties` object for a Quick App 2.0 has three required fields and one optional field:

| Field | Required | Type | Description |
|---|---|---|---|
| `orchestrator` | **Yes** | `OrchestratorConfig` | The language model and system prompt. |
| `contexts` | **Yes** | array | Static context items (files, inline text) injected into the model's context. Pass `[]` if unused. |
| `tool_sets` | **Yes** | array | Tools the model can call. Pass `[]` if unused. |
| `starters` | No | array of strings | Suggested conversation openers shown to users. |

**Minimal valid Quick App 2.0:**

```json
{
  "applicationTypeSchemaId": "https://mydial.epam.com/custom_application_schemas/quickapps2",
  "applicationProperties": {
    "orchestrator": {
      "deployment": { "name": "gpt-4o-mini" },
      "system_prompt": {
        "type": "custom",
        "variables": {},
        "content": "You are a helpful assistant."
      }
    },
    "contexts": [],
    "tool_sets": []
  }
}
```

## Orchestrator (`orchestrator`)

The orchestrator defines the model that runs the agent loop and the system prompt it uses.

### `OrchestratorConfig`

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `deployment` | **Yes** | `DialDeploymentConfig` | — | DIAL deployment to use as the orchestrating model. |
| `system_prompt` | **Yes** | `SystemPromptConfig` | — | System prompt configuration (see [System prompt types](#system-prompt-types) below). |
| `max_iterations` | No | integer | `15` | Maximum number of agent loop iterations (tool call cycles) before the orchestrator stops. |

### `DialDeploymentConfig`

| Field | Required | Type | Description |
|---|---|---|---|
| `name` | **Yes** | string | DIAL deployment ID (e.g., `"gpt-4o-mini"`, `"claude-3-5-sonnet"`). |
| `parameters` | No | `DialDeploymentParameters` | Model parameter overrides. |

### `DialDeploymentParameters`

All fields are optional and default to `null` (model uses its own defaults):

| Field | Type | Description |
|---|---|---|
| `temperature` | float | Sampling temperature (0.0–2.0). Higher values = more random. |
| `top_p` | integer | Top-p nucleus sampling. |
| `seed` | integer | Random seed for reproducibility. |
| `stop` | string or array of strings | Stop sequences (up to 4). |
| `n` | integer | Number of completions to generate per request. |
| `max_tokens` | integer | Maximum tokens in the generated response. |
| `presence_penalty` | float | Penalty for new tokens based on presence (-2.0 to 2.0). |
| `frequency_penalty` | float | Penalty for repeated tokens based on frequency (-2.0 to 2.0). |
| `custom_fields` | `CustomFieldsConfig` | Deployment-specific configuration (schema defined per deployment). |

## System prompt types

The `system_prompt` field is a discriminated union. Set `type` to select the variant.

### `custom` — inline text

```json
{
  "type": "custom",
  "variables": { "language": "English" },
  "content": "You are a helpful assistant. Respond in {{language}}."
}
```

| Field | Required | Description |
|---|---|---|
| `variables` | **Yes** | Key-value map of preset variable values. Use `{}` if no variables. |
| `content` | **Yes** | System prompt text. Use `{{variable_name}}` syntax for substitution. |

### `dial` — file reference

Loads a system prompt stored as a file in DIAL.

```json
{
  "type": "dial",
  "path": "appdata/my-app/prompts/assistant.txt",
  "variables": { "language": "English" }
}
```

| Field | Required | Description |
|---|---|---|
| `path` | **Yes** | Relative path to the prompt file in DIAL, including bucket prefix. |
| `variables` | No | Variable substitutions applied to the loaded prompt. |
| `content` | — | Read-only: the resolved prompt text (populated at runtime). |

### `predefined` — named template

References a named prompt template managed by the platform.

```json
{
  "type": "predefined",
  "template": "helpful-assistant"
}
```

| Field | Required | Description |
|---|---|---|
| `template` | **Yes** | Template name (without file extension). |
| `content` | — | Read-only: the resolved prompt text (populated at runtime). |

## Contexts (`contexts`)

Contexts are static items injected into the model's conversation context at the start of each request. They supplement the system prompt with additional grounding material.

### `FileContextConfig` (type: `"file"`)

```json
{
  "type": "file",
  "url": "files/bucket-id/my-document.pdf",
  "description": "Company policy handbook"
}
```

| Field | Required | Description |
|---|---|---|
| `url` | **Yes** | Relative file URL in DIAL (DIAL-encoded format). |
| `description` | No | Description of what the file contains. |

### `UserDefinedContextConfig` (type: `"user-defined"`)

```json
{
  "type": "user-defined",
  "content": "Today is {{date}}. The user's timezone is {{timezone}}."
}
```

| Field | Required | Description |
|---|---|---|
| `content` | **Yes** | Inline text injected as context. |

## Tool sets (`tool_sets`)

Each entry in `tool_sets` is a discriminated union — set `type` to select the variant. All tool set types share these common fields:

| Field | Required | Type | Default | Description |
|---|---|---|---|---|
| `type` | **Yes** | string | — | Tool set type discriminator (see below). |
| `name` | See per-type | string | — | Display name for the tool set. |
| `description` | No | string | `null` | Description shown to the model. |
| `enabled` | No | boolean | `true` | Whether the tool set is active. |

---

### `rest-api` — REST API tool set

Calls external REST endpoints. Authentication is configured at the tool set level and applies to all tools within it.

```json
{
  "type": "rest-api",
  "name": "weather-api",
  "authorization": {
    "type": "bearer",
    "token": "my-token"
  },
  "tools": [ ... ]
}
```

| Field | Required | Description |
|---|---|---|
| `name` | **Yes** | Tool set name. |
| `authorization` | No | Authentication config (see [Authorization types](#authorization-types)). |
| `tools` | **Yes** | Array of `RestApiTool` or `PredefinedTool`. |

#### `RestApiTool` (type: `"restapi-tool"`)

| Field | Required | Description |
|---|---|---|
| `open_ai_tool` | **Yes** | OpenAI tool function definition with REST-specific parameter info. |
| `rest_api_method_info` | **Yes** | HTTP method and URL. |
| `display` | No | `ToolDisplayConfig` — how the tool appears in chat stages. |
| `fallback_configuration` | No | `ToolFallbackConfig` — error handling strategy. |
| `attachment` | No | `AttachmentConfig` — attachment type configuration. |
| `enabled` | No | Default `true`. |

**`rest_api_method_info` fields:**

| Field | Required | Description |
|---|---|---|
| `method_url` | **Yes** | Full URL of the REST endpoint. |
| `method_type` | **Yes** | HTTP method: `"get"`, `"post"`, `"put"`, or `"delete"`. |

**Parameter mapping (`parameter_info`):**

Each parameter in `open_ai_tool.function.parameters.properties` can include a `parameter_info` field:

| Field | Required | Description |
|---|---|---|
| `type` | **Yes** | Where to send the parameter: `"query"`, `"url"`, `"body"`, or `"header"`. |
| `key` | **Yes** | The parameter key name in the actual HTTP request. |

Constant parameters (hidden from the model) use `"type": null` with a `"const"` value:

```json
"current": {
  "type": null,
  "const": "temperature_2m",
  "parameter_info": { "type": "query", "key": "current" }
}
```

### `dial-deployment` — DIAL deployment tool set

Wraps one or more DIAL applications or models as callable tools.

```json
{
  "type": "dial-deployment",
  "name": "my-agents",
  "tools": [
    { "type": "dial-deployment-simple", "deployment_id": "summarizer-app" },
    { ... }
  ]
}
```

| Field | Required | Description |
|---|---|---|
| `name` | **Yes** | Tool set name. |
| `tools` | **Yes** | Array of `DialDeploymentTool`, `DialDeploymentSimpleTool`, or `PredefinedTool`. |

#### `DialDeploymentSimpleTool` (type: `"dial-deployment-simple"`)

Minimal reference to a DIAL deployment. The orchestrator passes the conversation context to it without customization.

| Field | Required | Description |
|---|---|---|
| `deployment_id` | **Yes** | DIAL deployment ID to use as a tool. |
| `enabled` | No | Default `true`. |

#### `DialDeploymentTool` (type: `"deployment-tool"`)

Full-featured deployment tool with custom OpenAI function definition, system prompt, and content propagation.

| Field | Required | Description |
|---|---|---|
| `open_ai_tool` | **Yes** | OpenAI tool function definition (name, description, parameters). |
| `deployment` | **Yes** | `DialDeploymentConfig` — the deployment to call. |
| `system_prompt` | No | `CustomSystemPromptConfig` or `DialSystemPromptConfig` — overrides for this tool call. |
| `content_propagation` | No | `ContentPropagation` — controls history and header forwarding. |
| `display` | No | `ToolDisplayConfig`. |
| `fallback_configuration` | No | `ToolFallbackConfig`. |
| `attachment` | No | `AttachmentConfig`. |
| `enabled` | No | Default `true`. |

---

### `mcp` — external MCP server

Connects directly to an MCP server by URL.

```json
{
  "type": "mcp",
  "name": "search-tools",
  "mcp_server_info": {
    "url": "https://mcp.example.com/search",
    "protocol": "streamable_http",
    "authorization": { "type": "bearer", "token": "my-token" }
  },
  "allowed_tools": ["web_search", "news_search"]
}
```

| Field | Required | Description |
|---|---|---|
| `name` | **Yes** | Tool set name. |
| `mcp_server_info` | **Yes** | `MCPServerInfo` — server URL, protocol, and auth. |
| `allowed_tools` | No | Allowlist of tool names from the server. If omitted, all tools are available. |
| `attachment` | No | `AttachmentConfig`. |
| `fallback_configuration` | No | `ToolFallbackConfig`. |

#### `MCPServerInfo`

| Field | Required | Description |
|---|---|---|
| `url` | **Yes** | MCP server URL. |
| `protocol` | **Yes** | Transport protocol: `"sse"` or `"streamable_http"`. |
| `authorization` | No | One of `BearerAuthorization`, `MCPApiKeyAuthorization`, `ClientIdSecretAuthorization`, or `BasicAuthorization`. |

**MCP API key authorization** (`MCPApiKeyAuthorization`):

| Field | Required | Description |
|---|---|---|
| `type` | — | `"api_key"` |
| `key` | **Yes** | API key value. |
| `name` | **Yes** | Header or parameter name to send the key in. |

---

### `dial-mcp` — DIAL-managed MCP tool set

References a Tool Set registered in DIAL Core's `config.json` by its DIAL resource ID.

```json
{
  "type": "dial-mcp",
  "dial_id": "my-registered-toolset",
  "transport": "HTTP",
  "allowed_tools": ["search"]
}
```

| Field | Required | Default | Description |
|---|---|---|---|
| `dial_id` | **Yes** | — | DIAL resource ID of the registered Tool Set. |
| `name` | No | `"dial-mcp"` | Display name. |
| `transport` | No | `"HTTP"` | `"HTTP"` or `"SSE"`. |
| `allowed_tools` | No | `null` | Allowlist of tool names. |
| `attachment` | No | — | `AttachmentConfig`. |
| `fallback_configuration` | No | — | `ToolFallbackConfig`. |

---

### `internal` — internal tool set

Tool set containing `InternalTool` entries — custom function definitions handled internally by the Quick Apps backend.

| Field | Required | Description |
|---|---|---|
| `name` | **Yes** | Tool set name. |
| `tools` | **Yes** | Array of `InternalTool` or `PredefinedTool`. |

#### `InternalTool` (type: `"internal-tool"`)

| Field | Required | Description |
|---|---|---|
| `open_ai_tool` | **Yes** | OpenAI tool function definition. |
| `display` | No | `ToolDisplayConfig`. |
| `fallback_configuration` | No | `ToolFallbackConfig`. |
| `attachment` | No | `AttachmentConfig`. |
| `enabled` | No | Default `true`. |

---

### `predefined` — predefined tool set

References a named tool set template managed by the platform.

| Field | Required | Description |
|---|---|---|
| `template_name` | **Yes** | Template name (without extension). |

---

### `PredefinedTool` (type: `"predefined-tool"`)

Can appear in `rest-api`, `dial-deployment`, or `internal` tool sets. References a single named tool template.

| Field | Required | Description |
|---|---|---|
| `template_name` | **Yes** | Tool template file name (without extension), e.g., `"rag_tool"`. |
| `enabled` | No | Default `true`. |

## Authorization types

Used in `RestApiToolSet.authorization` and as base types for MCP authorization.

### `ApiKeyAuthorization` (type: `"api_key"`)

| Field | Required | Description |
|---|---|---|
| `key` | **Yes** | API key value. |
| `name` | **Yes** | Name of the header or parameter. |
| `location` | **Yes** | Where to send the key: `"header"`, `"query"`, or `"body"`. |

### `BasicAuthorization` (type: `"basic"`)

| Field | Required | Description |
|---|---|---|
| `username` | **Yes** | Username. |
| `password` | **Yes** | Password. |

### `BearerAuthorization` (type: `"bearer"`)

| Field | Required | Description |
|---|---|---|
| `token` | **Yes** | Bearer token value. |

### `ClientIdSecretAuthorization` (type: `"client_id_secret"`)

| Field | Required | Default | Description |
|---|---|---|---|
| `client_id` | **Yes** | — | OAuth client ID. |
| `client_secret` | **Yes** | — | OAuth client secret. |
| `token_url` | **Yes** | — | Token endpoint URL. |
| `scope` | No | `null` | OAuth scope. |
| `aud` | No | `null` | OAuth audience. |

## Fallback configuration

`ToolFallbackConfig` controls what happens when a tool call fails.

| Field | Description |
|---|---|
| `strategies` | Array of `StopStrategyModel` or `ContinueStrategyModel`. |
| `display_error_in_stage` | Whether to show the error message in the chat stage. Default `true`. |

### `StopStrategyModel` (type: `"stop"`)

Stops the agent loop on error.

| Field | Description |
|---|---|
| `trigger_on` | Optional `TriggerOn` condition. If omitted, applies to all errors. |

### `ContinueStrategyModel` (type: `"continue"`)

Continues the agent loop after an error, optionally with instructions for the model.

| Field | Description |
|---|---|
| `trigger_on` | Optional `TriggerOn` condition. If omitted, applies to all errors. |
| `instructions` | Optional instructions for the model on what to do next. |

### `TriggerOn`

Matches error messages to decide which strategy applies.

| Field | Required | Description |
|---|---|---|
| `type` | **Yes** | `"contains"` or `"equals"`. |
| `value` | **Yes** | String to match against the error message. |
| `case_sensitive` | No | Default `false`. |

---

## Display configuration

`ToolDisplayConfig` controls how a tool call appears in DIAL Chat's stage UI.

| Field | Description |
|---|---|
| `stage` | `ToolStageConfig` — stage-level display settings. |

### `ToolStageConfig`

| Field | Default | Description |
|---|---|---|
| `name` | `null` | Stage title. Use `{argument_name}` for dynamic values from tool arguments. |
| `body` | `null` | Stage body content. Supports `{argument_name}` placeholders. |
| `show` | `true` | Whether the stage is visible in the chat UI. |

---

## Content propagation

`ContentPropagation` controls how conversation content is forwarded when calling a DIAL deployment tool.

| Field | Default | Description |
|---|---|---|
| `propagate_history` | `false` | If `true`, prior messages are propagated as `[user, assistant]` pairs (converted from `[assistant, tool]` format). |
| `propagate_headers` | `null` | List of HTTP headers to forward to the deployment. If `null`, default headers are used. |

---

## Attachment configuration

`AttachmentConfig` controls which file attachment types a tool set accepts and propagates.

| Field | Description |
|---|---|
| `supported_types` | List of supported MIME types (e.g., `["image/png", "text/plain"]`). `null` = all types. |
| `propagate_types_to_choice` | List of MIME types to propagate from the tool stage response to the final choice. |

---

## Starters

```json
"starters": [
  "What is the weather in Paris?",
  "Search for recent AI news"
]
```

Array of strings shown as clickable suggestions at the start of a conversation. Optional, defaults to `null`.
