# Custom App examples

A curated list of runnable Custom App examples, sample projects, and video walkthroughs. Each entry includes a description of what it demonstrates and a link to the source code.

## Starter examples

### Echo app

The simplest possible Custom App — receives a message and echoes it back. Demonstrates the minimal DIAL SDK integration: `ChatCompletion` base class, `Request`/`Response` handling, and Docker Compose setup with DIAL Core and Chat.

**Source:** [https://github.com/epam/ai-dial/tree/main/dial-docker-compose/application/echo](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/application/echo)

**Covered in:** [Getting started with DIAL SDK](1.getting-started-sdk.md)

---

### Standalone app with custom API

A Custom App that exposes both the standard DIAL chat completions endpoint and a custom REST API for configuration. Demonstrates mounting a FastAPI app alongside `DIALApp`, managing application state, and serving custom endpoints.

**Source:** [https://github.com/epam/ai-dial/tree/main/dial-samples/standalone-app-chat-ui](https://github.com/epam/ai-dial/tree/main/dial-samples/standalone-app-chat-ui)

---

### App with application runner (schema-rich)

A schema-rich Custom App that reads its configuration from DIAL Core using `request.request_dial_application_properties()`. Demonstrates how to build apps that conform to an application type schema — enabling UI wizards and per-instance configuration.

**Source:** [https://github.com/epam/ai-dial/tree/main/dial-samples/app-runner-chat-ui](https://github.com/epam/ai-dial/tree/main/dial-samples/app-runner-chat-ui)

---

## Calling patterns (cookbook)

These Jupyter notebooks demonstrate how to call DIAL applications as a client using various SDKs and tools.

### Text-to-text applications

Calling a text application using `curl`, Python `requests`, the OpenAI Python SDK, and LangChain — including both streaming and non-streaming modes.

**Source:** [https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_text_to_text_applications.ipynb](https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_text_to_text_applications.ipynb)

---

### Text-to-image applications

Calling an image generation application through the Unified API. Demonstrates how DIAL returns generated images as attachments.

**Source:** [https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_text_to_image_applications.ipynb](https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_text_to_image_applications.ipynb)

---

### Image-to-text applications

Sending images as attachments to a multimodal application. Demonstrates the DIAL attachment format (base64 and DIAL file storage URLs).

**Source:** [https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_image_to_text_applications.ipynb](https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_image_to_text_applications.ipynb)

---

### Image generation with configuration

Calling GPT image generation with custom configuration parameters passed through the request.

**Source:** [https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_gpt_image_1_with_configuration.ipynb](https://github.com/epam/ai-dial/blob/main/dial-cookbook/examples/how_to_call_gpt_image_1_with_configuration.ipynb)

## SDK examples (GitHub)

The DIAL SDK repository contains additional examples demonstrating specific features:

|---------|---------------------|
| [echo](https://github.com/epam/ai-dial-sdk/tree/main/examples/echo) | Minimal app, attachment echo-back |
