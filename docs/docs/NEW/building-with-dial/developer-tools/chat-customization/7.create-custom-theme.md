# Create a custom theme

In this tutorial, you will set up the DIAL Chat Themes server with a custom color scheme, logo, and authentication screen colors. By the end, you will have a Docker Compose stack running DIAL Chat with your own branded theme.

## Prerequisites

- Docker and Docker Compose V2 installed
- A text editor
- Basic familiarity with DIAL Chat layout concepts (see [Theming and design](5.theming-and-design.md))

## What you will build

A self-contained Docker Compose stack with three services: DIAL Core, DIAL Chat, and a themes server. The themes server serves a `config.json` file that defines your custom color palette, logo images, and authentication page colors. DIAL Chat fetches this configuration at startup and applies it to the UI.

```text
DIAL Chat  ──THEMES_CONFIG_HOST──▶  Themes Server (/app/static/)
                                       ├── config.json
                                       └── images/
```

## Project structure

By the end of this tutorial, your project will look like this:

```text
custom-theme/
├── docker-compose.yml
├── core/
│   └── config.json
└── themes/
    └── static/
        ├── config.json
        └── images/
            └── logo.svg
```

## Step 1: Create the project directory

Create the directory structure:

```bash
mkdir -p custom-theme/core custom-theme/themes/static/images
cd custom-theme
```

**Verify:** Run `find . -type d` and confirm you see `./core`, `./themes/static`, and `./themes/static/images`.

## Step 2: Create the DIAL Core configuration

Create `custom-theme/core/config.json` with a minimal configuration that includes an echo application for testing:

```json
{
  "routes": {},
  "keys": {
    "dial_api_key": {
      "project": "test",
      "role": "admin"
    }
  },
  "applications": {
    "echo": {
      "endpoint": "http://echo:8080/openai/deployments/echo/chat/completions",
      "displayName": "Echo",
      "iconUrl": "message-square-lines-alt.svg"
    }
  },
  "roles": {
    "admin": {
      "limits": {
        "echo": {}
      }
    },
    "default": {
      "limits": {
        "echo": {}
      }
    }
  }
}
```

:::warning
Color token names must remain exactly as shown. Renaming or removing tokens breaks the mapping between design tokens and the DIAL Chat CSS.
:::

The color tokens are organized into four groups:

- **`bg-*`** — background colors for layers, status states, and accents
- **`text-*`** — text colors for primary content, status messages, and accents
- **`stroke-*`** — border and outline colors
- **`controls-*`** — interactive element colors (buttons, inputs, toggles)

The `topicColors` object defines colors for Marketplace topic badges. The `authColors` object styles the sign-in page.

## Step 4: Add a custom logo

Place your logo file at `custom-theme/themes/static/images/logo.svg`. For this tutorial, create a minimal SVG placeholder:

```bash
cat > themes/static/images/logo.svg << 'LOGOEOF'
<svg xmlns="http://www.w3.org/2000/svg" viewBox="0 0 100 24">
  <text x="0" y="18" font-family="Arial, sans-serif" font-size="18" fill="#58A6FF">My DIAL</text>
</svg>
LOGOEOF
```

The themes server resolves relative paths in `config.json` against its own base URL. The `"chat-logo-dark": "images/logo.svg"` entry in the configuration tells DIAL Chat to load the logo from `http://themes:8080/images/logo.svg`.

Logo requirements:
- Maximum width: 100 px (as rendered in the Chat header)
- Any image format (SVG, PNG, JPG)
- Maximum file size: 512 MB

## Step 5: Write the Docker Compose file

Create `custom-theme/docker-compose.yml`:

```yaml
services:
  themes:
    image: epam/ai-dial-chat-themes:0.10.0
    platform: linux/amd64
    volumes:
      - ./themes/static:/app/static
    ports:
      - "3001:8080"

  chat:
    image: epam/ai-dial-chat:0.41.7
    platform: linux/amd64
    environment:
      THEMES_CONFIG_HOST: "http://themes:8080"
      DIAL_API_HOST: "http://core:8080"
      DIAL_API_KEY: "dial_api_key"
      NEXTAUTH_SECRET: "secret"
      ENABLED_FEATURES: "conversations-section,prompts-section,top-settings,top-clear-conversation,top-chat-info,top-chat-model-settings,empty-chat-settings,header,footer,custom-logo,marketplace"
    ports:
      - "3000:3000"
    depends_on:
      - themes
      - core

  core:
    image: epam/ai-dial-core:0.39.2
    platform: linux/amd64
    environment:
      AIDIAL_SETTINGS: "/opt/settings/settings.json"
      aidial.config.files: '["/opt/config/config.json"]'
      aidial.storage.overrides: '{ "jclouds.filesystem.basedir": "data" }'
    ports:
      - "8080:8080"
    volumes:
      - ./core:/opt/config

  echo:
    image: epam/ai-dial-core:0.39.2
    platform: linux/amd64
    entrypoint: ["java", "-cp", "/opt/service/*", "com.epam.aidial.core.server.EchoApplication"]
    ports:
      - "8081:8080"
```

Key configuration points:

- The volume mount `./themes/static:/app/static` replaces the default theme files with your custom ones.
- `THEMES_CONFIG_HOST` tells DIAL Chat where to fetch `config.json`.
- `custom-logo` in `ENABLED_FEATURES` enables logo display in the header.
- `depends_on` ensures the themes server starts before DIAL Chat.

## Step 6: Start the services

```bash
docker compose up
```

Wait for all services to report as ready. You will see log lines like:

```text
themes-1  | ... listening on 0.0.0.0:8080
chat-1    | ... ready on http://0.0.0.0:3000
core-1    | ... HTTP server started on port 8080
```

**Verify:** Open `http://localhost:3001/config.json` in your browser. You should see your custom theme configuration.

## Step 7: Verify in DIAL Chat

1. Open `http://localhost:3000` in your browser.
2. Confirm your custom dark color scheme is applied — the background should use your `bg-layer-0` color (`#0D1117`).
3. Check that your custom logo appears in the Chat header.
4. Open the user settings (click the settings icon in the header). The theme picker should show "My Dark Theme."
