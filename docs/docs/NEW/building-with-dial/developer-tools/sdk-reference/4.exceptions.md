# Exceptions

The `aidial_sdk.exceptions` module defines error classes that produce Unified API-compatible error responses. When you raise one of these exceptions inside `chat_completion` or `embeddings`, the SDK catches it and returns a properly formatted JSON error to the caller.

```python
from aidial_sdk.exceptions import HTTPException
```

## Error response format

All exceptions produce this JSON structure:

```json
{
  "error": {
    "message": "Human-readable error description",
    "type": "error_type",
    "param": null,
    "code": "error_code",
    "display_message": null
  }
}
```

The `display_message` field, when set, is shown directly to end users in DIAL Chat. The `message` field is always logged but may contain technical details not suitable for display.

## HTTPException

Base class for all DIAL SDK exceptions.

```python
HTTPException(
    message: str,
    status_code: int = 500,
    type: str | None = "runtime_error",
    param: str | None = None,
    code: str | None = None,
    display_message: str | None = None,
    headers: dict[str, str] | None = None,
)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `message` | `str` | *required* | Error message (logged, included in response). |
| `status_code` | `int` | `500` | HTTP status code. |
| `type` | `str \| None` | `"runtime_error"` | Error type identifier. |
| `param` | `str \| None` | `None` | The parameter that caused the error. |
| `code` | `str \| None` | `None` | Error code. Defaults to the string representation of `status_code`. |
| `display_message` | `str \| None` | `None` | User-facing message shown in DIAL Chat. |
| `headers` | `dict[str, str] \| None` | `None` | Additional HTTP response headers. |

### Methods

| Method | Returns | Description |
|--------|---------|-------------|
| `json_error()` | `dict` | The error response body as a dictionary. |
| `to_fastapi_response()` | `JSONResponse` | A FastAPI JSON response with the error body and status code. |
| `to_fastapi_exception()` | `FastAPIException` | A FastAPI `HTTPException` (for use in FastAPI exception handlers). |

## Subclasses

### InvalidRequestError

HTTP 400. The request is malformed or contains invalid values.

```python
from aidial_sdk.exceptions import InvalidRequestError

raise InvalidRequestError("The 'format' parameter must be 'json' or 'text'.")
```

| Field | Value |
|-------|-------|
| `status_code` | `400` |
| `type` | `"invalid_request_error"` |

### RequestValidationError

HTTP 422. The request fails validation (for example, a required field is missing or a value is out of range).

```python
from aidial_sdk.exceptions import RequestValidationError

raise RequestValidationError("Temperature must be between 0 and 2.")
```

| Field | Value |
|-------|-------|
| `status_code` | `422` |
| `type` | `"invalid_request_error"` |

### ResourceNotFoundError

HTTP 404. A requested resource (endpoint, API version) does not exist.

```python
from aidial_sdk.exceptions import ResourceNotFoundError

raise ResourceNotFoundError("Model 'gpt-5' is not available.")
```

| Field | Value |
|-------|-------|
| `status_code` | `404` |
| `type` | `"runtime_error"` |

### DeploymentNotFoundError

HTTP 404. The specified deployment does not exist. Uses the `"DeploymentNotFound"` error code.

```python
from aidial_sdk.exceptions import DeploymentNotFoundError

raise DeploymentNotFoundError("Deployment 'my-app-v2' not found.")
```

| Field | Value |
|-------|-------|
| `status_code` | `404` |
| `code` | `"DeploymentNotFound"` |

### ContextLengthExceededError

HTTP 400. The prompt exceeds the model's maximum context length. The error message is auto-generated from the parameters.

```python
from aidial_sdk.exceptions import ContextLengthExceededError

raise ContextLengthExceededError(max_context_length=8192, prompt_tokens=9500)
```

Produces the message: *"This model's maximum context length is 8192 tokens. However, your messages resulted in 9500 tokens. Please reduce the length of the messages."*

| Field | Value |
|-------|-------|
| `status_code` | `400` |
| `type` | `"invalid_request_error"` |
| `code` | `"context_length_exceeded"` |
| `param` | `"messages"` |

### RuntimeServerError

HTTP 500. An unexpected error occurred during processing. This is also the exception the SDK wraps around unhandled exceptions in `chat_completion`.

```python
from aidial_sdk.exceptions import RuntimeServerError

raise RuntimeServerError("Failed to connect to the vector database.")
```

| Field | Value |
|-------|-------|
| `status_code` | `500` |
| `type` | `"runtime_error"` |

### InternalServerError

HTTP 500. An internal server error with `type` set to `"internal_server_error"` instead of `"runtime_error"`.

```python
from aidial_sdk.exceptions import InternalServerError

raise InternalServerError("Unexpected state: queue is empty.")
```

| Field | Value |
|-------|-------|
| `status_code` | `500` |
| `type` | `"internal_server_error"` |

### TruncatePromptSystemError

HTTP 400. The system messages alone exceed `max_prompt_tokens`. Used by `truncate_prompt` implementations. The error message is auto-generated and also set as `display_message`.

```python
from aidial_sdk.exceptions import TruncatePromptSystemError

raise TruncatePromptSystemError(max_prompt_tokens=4000, prompt_tokens=4500)
```

### TruncatePromptSystemAndLastUserError

HTTP 400. The system messages plus the last user message exceed `max_prompt_tokens`. The error message is auto-generated and also set as `display_message`.

```python
from aidial_sdk.exceptions import TruncatePromptSystemAndLastUserError

raise TruncatePromptSystemAndLastUserError(max_prompt_tokens=4000, prompt_tokens=5000)
```

## Unhandled exceptions

If your `chat_completion` or `embeddings` method raises an exception that is not an `HTTPException` subclass, the SDK catches it, logs the traceback, and returns a `RuntimeServerError` with the message `"Error during stream generation"`. The original exception details are not exposed to the caller.
