# Telemetry

DIAL SDK includes built-in OpenTelemetry instrumentation for traces, metrics, and logs. Enable it by passing a `TelemetryConfig` to `DIALApp` or by calling `app.configure_telemetry()`.

Telemetry requires the optional extras:

```bash
pip install "aidial-sdk[telemetry]"
```

## TelemetryConfig

```python
from aidial_sdk.telemetry.types import TelemetryConfig
```

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `service_name` | `str \| None` | `None` | OpenTelemetry service name. Sets the `service.name` resource attribute. |
| `tracing` | `TracingConfig \| None` | Auto-detected | Tracing configuration. Defaults to `TracingConfig()` if `OTEL_TRACES_EXPORTER` is set, otherwise `None`. |
| `logs` | `LogsConfig \| None` | Auto-detected | Log export configuration. Defaults to `LogsConfig()` if `OTEL_LOGS_EXPORTER` is set, otherwise `None`. |
| `metrics` | `MetricsConfig \| None` | Auto-detected | Metrics configuration. Defaults to `MetricsConfig()` if `OTEL_METRICS_EXPORTER` is set, otherwise `None`. |

## TracingConfig

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `otlp_export` | `bool` | `True` if `"otlp"` is in `OTEL_TRACES_EXPORTER` | Export spans via OTLP gRPC. |
| `logging` | `bool` | Value of `OTEL_PYTHON_LOG_CORRELATION` | When `True`, injects `trace_id` and `span_id` into console log messages. |

When tracing is enabled, the SDK instruments the following HTTP client libraries (if installed):

- `requests` (synchronous)
- `aiohttp` (async)
- `httpx` (sync and async)
- `urllib` (standard library)

FastAPI request handling is also instrumented.

## LogsConfig

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `otlp_export` | `bool` | `True` if `"otlp"` is in `OTEL_LOGS_EXPORTER` | Export log records via OTLP gRPC. |
| `level` | `int` | `logging.INFO` | Minimum log level to export. Uses standard Python `logging` levels. |

## MetricsConfig

| Field | Type | Default | Description |
|-------|------|---------|-------------|
| `otlp_export` | `bool` | `True` if `"otlp"` is in `OTEL_METRICS_EXPORTER` | Export metrics via OTLP gRPC. |
| `prometheus_export` | `bool` | `True` if `"prometheus"` is in `OTEL_METRICS_EXPORTER` | Expose a Prometheus scrape endpoint. |
| `port` | `int` | `OTEL_EXPORTER_PROMETHEUS_PORT` or `9464` | Port for the Prometheus HTTP server. |

When metrics are enabled, the SDK instruments system metrics (CPU, memory) via `SystemMetricsInstrumentor` and FastAPI request metrics.

## Environment variables

The SDK reads these standard OpenTelemetry environment variables to auto-configure telemetry. Set them before your application starts.

| Variable | Description | Example |
|----------|-------------|---------|
| `OTEL_TRACES_EXPORTER` | Comma-separated list of trace exporters. | `otlp` |
| `OTEL_LOGS_EXPORTER` | Comma-separated list of log exporters. | `otlp` |
| `OTEL_METRICS_EXPORTER` | Comma-separated list of metric exporters. | `otlp,prometheus` |
| `OTEL_EXPORTER_PROMETHEUS_PORT` | Port for the Prometheus scrape endpoint. | `9464` |
| `OTEL_PYTHON_LOG_CORRELATION` | Inject trace/span IDs into log messages. | `true` |
| `OTEL_EXPORTER_OTLP_ENDPOINT` | OTLP collector endpoint (standard OTEL SDK variable). | `http://collector:4317` |
| `OTEL_SERVICE_NAME` | Service name (alternative to `TelemetryConfig.service_name`). | `my-dial-app` |

## Usage examples

### Minimal: OTLP traces only

```bash
export OTEL_TRACES_EXPORTER=otlp
export OTEL_EXPORTER_OTLP_ENDPOINT=http://collector:4317
```

```python
from aidial_sdk import DIALApp
from aidial_sdk.telemetry.types import TelemetryConfig

app = DIALApp(telemetry_config=TelemetryConfig(service_name="my-app"))
```

### Full: traces, logs, metrics with Prometheus

```bash
export OTEL_TRACES_EXPORTER=otlp
export OTEL_LOGS_EXPORTER=otlp
export OTEL_METRICS_EXPORTER=otlp,prometheus
export OTEL_EXPORTER_OTLP_ENDPOINT=http://collector:4317
export OTEL_EXPORTER_PROMETHEUS_PORT=9464
export OTEL_PYTHON_LOG_CORRELATION=true
```

```python
from aidial_sdk import DIALApp
from aidial_sdk.telemetry.types import TelemetryConfig

app = DIALApp(
    telemetry_config=TelemetryConfig(service_name="my-app"),
    add_healthcheck=True,
)
```

With this configuration:
- Traces are exported to the OTLP collector and all outgoing HTTP calls are traced.
- Log records are exported to the OTLP collector and console logs include `trace_id`/`span_id`.
- Metrics are exported to both the OTLP collector and a Prometheus scrape endpoint on port 9464.

### Programmatic configuration (no environment variables)

```python
from aidial_sdk import DIALApp
from aidial_sdk.telemetry.types import (
    LogsConfig,
    MetricsConfig,
    TelemetryConfig,
    TracingConfig,
)

config = TelemetryConfig(
    service_name="my-app",
    tracing=TracingConfig(otlp_export=True, logging=True),
    logs=LogsConfig(otlp_export=True),
    metrics=MetricsConfig(
        otlp_export=True,
        prometheus_export=True,
        port=9090,
    ),
)

app = DIALApp(telemetry_config=config)
```
