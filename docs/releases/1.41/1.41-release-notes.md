# Release Notes

The purpose of this document is to provide a quick summary of all the biggest new features added in this version and provide some additional description, video, or tutorials for major features.

## Brief Summary

The highlights of this release include significant improvements to **Quick Apps 2.0** including Agent Skills support, **chat UI enhancements** for custom applications and attachments handling, **publication rules** logic, and support for **several new models**.

## Major Enhancements

**Agent Skills Specification Support**: This feature is buried under the hood, but it significantly improves predictability of file-related processes with external agents in Quick Apps 2.0. Since Quick Apps are responsible for orchestrating between several external tools with various protocls (MCP servers, REST APIs, etc.) which accept file-related parameters in different formats, agents are forced to guess what kind of file they are without much context or information. By implementing the Agent Skills specification with Markdown instruction files, DIAL now supports far more detailed instructions for agents on how to handle different types of files. This should significantly improve support and scalability of complicated business workflows when working with multiple filetypes. Manual configuration of complicated Quick Apps workflows can be done using the new **JSON Editor**, allowing power users even more flexibility.
  
**UI Enhancements in Conversations (Attachment Handling)**: While many clients use DIAL as a one-stop shop for all their GenAI needs, others want to integrate DIAL agents it into their own websites. DIAL supports this with its chat overlay capabilities. In this release, we've added improvements to makee the look & feel of DIAL applications far more flexible with features like borderless custom visualizers that are maximized by default, improved image attachment handlers, and improvements for custom themes. This allows developers to create applications that fit seamlessly into external UIs.

**New Models and Adapter Enhancements**: DIAL remains up to date with the latest models. From **Vertex**, we added support for `gemini-3-pro` and `gemini-3.1-pro-preview`. In our **OpenAI** adapter, we have added support for `Sora 2` and `gpt-4o-audio-preview` with audio completions. And in **Bedrock** adapter, we have added support for `Claude Sonnet 4.6` and `Claude Opus 4.6`, as well as support for structured output in models. With DIAL, you never have to worry about the modlels you work with becoming obsolete.

## Additional Notes

For full technical release notes with all bug fixes and additional features, please consult the [upgrade guide](upgrade-to-1.41.md) with all the tags for each component, as well as the DIAL documentation.
* **Publications:** Implemented publication rules framework, including support for nested user claims in publication requests, and ability to unpublish specific versions of an application
* **QuickApps 2.0**: Add ability to use `response_format` in QuickApps 2.0 to force format of responses
* **Admin**: Now uses the same File Manager as the DIAL Chat
* **Application Schema**: Allows application type runner to manage application schemas, reducing number of required config files
