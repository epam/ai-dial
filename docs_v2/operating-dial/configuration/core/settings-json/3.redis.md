# Redis

DIAL Core requires Redis for in-memory caching of resources, sessions, and rate-limit counters. Redis is configured on two levels: the DIAL Core connection settings in `settings.json` and the Redis server itself.

## Connection settings

Either single server or cluster configuration must be provided in `settings.json`.

### `redis.singleServerConfig.address`

Redis single server address.

```json
{
  "redis": {
    "singleServerConfig": {
      "address": "redis://redis:6379"
    }
  }
}
```

### `redis.clusterServersConfig.nodeAddresses`

Array of Redis cluster node addresses.

```json
{
  "redis": {
    "clusterServersConfig": {
      "nodeAddresses": [
        "redis://host1:6379",
        "redis://host2:6379"
      ]
    }
  }
}
```

### Cloud-managed Redis

For cloud-managed Redis with IAM authentication:

#### `redis.provider.name`

Cloud provider: `aws-elasti-cache`, `gcp-memory-store`, or `azure-redis-cache`.

#### `redis.provider.userId`

IAM-enabled user ID (AWS ElastiCache).

#### `redis.provider.accountName`

Service account resource name in `projects/-/serviceAccounts/{ACCOUNT}` format (GCP).

#### `redis.provider.region`

Geo region (AWS ElastiCache).

#### `redis.provider.clusterName`

Redis cluster name (AWS ElastiCache).

#### `redis.provider.serverless`

Whether the cache is serverless (AWS ElastiCache).

The Redis server itself is configured via command-line arguments in the Docker Compose file. See the [Quick Start](../../docker-compose/quick-start#redis-server-configuration) and [Advanced setup](../../docker-compose/advanced/core-services#redis) Docker Compose pages for server configuration details.
