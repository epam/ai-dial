# Core, Redis & Themes

This page documents the Core, Redis, and Themes service definitions in the advanced [`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml).

## Core

```yaml
core:
  image: epam/ai-dial-core:0.39.2
  platform: linux/amd64
  environment:
    'JAVA_OPTS': '-Dgflog.config=/opt/settings/gflog.xml'
    'proxy.config.files': '["/app/config/config.json"]'
    'aidial.identityProviders.keycloak.jwksUrl': 'http://keycloak:${KC_PORT}/realms/dial/protocol/openid-connect/certs'
    'aidial.identityProviders.keycloak.rolePath': 'roles'
    'aidial.access.admin.rules': '[{"source": "roles", "function": "EQUAL", "targets": ["admin"]}]'
    'aidial.identityProviders.keycloak.loggingKey': 'email'
    'aidial.identityProviders.keycloak.loggingSalt': 'loggingSalt'
    'aidial.encryption.secret': '${DIAL_CORE_ENCRYPTION_SECRET}'
    'aidial.encryption.key': '${DIAL_CORE_ENCRYPTION_KEY}'
    'aidial.redis.singleServerConfig.address': 'redis://redis:6379'
    'aidial.storage.provider': 'filesystem'
    'aidial.storage.prefix': 'core'
    'aidial.storage.bucket': 'dial'
    'aidial.storage.createBucket': 'true'
    'aidial.storage.overrides': '{"jclouds.filesystem.basedir": "data"}'
    'aidial.applications.includeCustomApps': 'true'
  ports:
    - "${DIAL_CORE_HOST_PORT}:8080"
  volumes:
    - ./core:/app/config
    - ./core-logs:/app/log
    - ./core-data:/app/data
    - ./settings:/opt/settings
  depends_on:
    - redis
  restart: always
```

### Environment variables

All `aidial.*` variables override their corresponding keys in [`settings.json`](../../core/settings-json/index). See [Configuration precedence](../../core/settings-json/index#configuration-precedence) for override rules.

| Variable | settings.json key | Description | Reference |
|----------|------------------|-------------|-----------|
| `JAVA_OPTS` | — | JVM options. Points to the [gflog.xml](../../core/gflog-xml) logging config. | — |
| `proxy.config.files` | `config.files` | Path to the dynamic [`config.json`](../../core/config-json/index). | [Config Reload](../../core/settings-json/config-reload) |
| `aidial.identityProviders.keycloak.jwksUrl` | `identityProviders.keycloak.jwksUrl` | JWKS endpoint for JWT validation. | [Identity Providers](../../core/settings-json/identity-providers) |
| `aidial.identityProviders.keycloak.rolePath` | `identityProviders.keycloak.rolePath` | JWT claim path for user roles. | [Identity Providers](../../core/settings-json/identity-providers) |
| `aidial.access.admin.rules` | `access.admin.rules` | Rules matching JWT claims for admin access. | [Access Control](../../core/settings-json/access) |
| `aidial.identityProviders.keycloak.loggingKey` | `identityProviders.keycloak.loggingKey` | JWT claim used for user identification in logs. | [Identity Providers](../../core/settings-json/identity-providers) |
| `aidial.identityProviders.keycloak.loggingSalt` | `identityProviders.keycloak.loggingSalt` | Salt for hashing user info in logs. | [Identity Providers](../../core/settings-json/identity-providers) |
| `aidial.encryption.secret` | `encryption.secret` | AES encryption secret. Injected from `.env`. | [Encryption](../../core/settings-json/encryption) |
| `aidial.encryption.key` | `encryption.key` | AES encryption key. Injected from `.env`. | [Encryption](../../core/settings-json/encryption) |
| `aidial.redis.singleServerConfig.address` | `redis.singleServerConfig.address` | Redis connection address. | [Redis](../../core/settings-json/redis) |
| `aidial.storage.provider` | `storage.provider` | Blob storage provider. | [Storage](../../core/settings-json/storage) |
| `aidial.storage.prefix` | `storage.prefix` | Base prefix for stored resources. | [Storage](../../core/settings-json/storage) |
| `aidial.storage.bucket` | `storage.bucket` | Storage bucket name. | [Storage](../../core/settings-json/storage) |
| `aidial.storage.createBucket` | `storage.createBucket` | Create the bucket on startup if missing. | [Storage](../../core/settings-json/storage) |
| `aidial.storage.overrides` | `storage.overrides` | Storage property overrides (jclouds). | [Storage](../../core/settings-json/storage) |
| `aidial.applications.includeCustomApps` | `applications.includeCustomApps` | Include custom apps in listing endpoint. | [Access Control](../../core/settings-json/access) |

### Volume mounts

| Mount | Purpose |
|-------|---------|
| `./core:/app/config` | `config.json` |
| `./core-logs:/app/log` | Log output directory |
| `./core-data:/app/data` | Filesystem blob storage |
| `./settings:/opt/settings` | `settings.json` and `gflog.xml` |

### Differences from quick start

| Aspect | Quick Start | Advanced |
|--------|------------|---------|
| Configuration method | `AIDIAL_SETTINGS` env var pointing to `settings.json` | `aidial.*` env var overrides (no `AIDIAL_SETTINGS`) |
| Config file path | `aidial.config.files` | `proxy.config.files` |
| Identity providers | `disableJwtVerification: true` | Keycloak JWKS validation |
| Encryption | Not configured | `encryption.secret` and `encryption.key` from `.env` |
| Custom apps | Not enabled | `includeCustomApps: true` |

## Redis

```yaml
redis:
  image: redis:7.2.4-alpine3.19
  restart: always
  ports:
    - "6600:6379"
  command: >
    redis-server
    --maxmemory 2000mb
    --maxmemory-policy volatile-lfu
    --save ""
    --appendonly no
    --loglevel warning
  mem_limit: 2200M
```

The Redis configuration is identical to the [quick start](../quick-start#redis-server-configuration) except the host port maps to `6600` instead of `6379` to avoid conflicts when running both setups simultaneously.

See [Redis connection settings](../../core/settings-json/redis) for DIAL Core's `redis.*` configuration in `settings.json`.

## Themes

```yaml
themes:
  image: epam/ai-dial-chat-themes:0.10.0
  platform: linux/amd64
  ports:
    - "3101:8080"
```
