# Authentication

The advanced setup uses Keycloak as the identity provider. DIAL Chat authenticates users via OIDC, and DIAL Core validates JWT tokens against Keycloak's JWKS endpoint.

## Keycloak

From [`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml):

```yaml
keycloak:
  image: keycloak/keycloak:26.3
  user: root
  command: ["start-dev", "--import-realm", "--http-port=${KC_PORT}", "--hostname-backchannel-dynamic=true"]
  environment:
    KC_BOOTSTRAP_ADMIN_USERNAME: admin
    KC_BOOTSTRAP_ADMIN_PASSWORD: ${KC_ADMIN_PASSWORD}
    KC_HOSTNAME: http://localhost:${KC_PORT}
    KC_HOSTNAME_STRICT: false
    KC_HOSTNAME_STRICT_HTTPS: false
    KC_HTTP_ENABLED: true
    KC_PROXY: edge
    KC_INITIALIZE_EMPTY: 'true'
    KC_FEATURE_CACHE_STACK: 'true'
    QUARKUS_HTTP_LIMITS_MAX_HEADER_SIZE: 64k
  ports:
    - "${KC_PORT}:${KC_PORT}"
  volumes:
    - ./keycloak/data:/opt/keycloak/data
    - ./keycloak/import/realm-config.json:/opt/keycloak/data/import/realm-config.json
  restart: always
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `KC_BOOTSTRAP_ADMIN_USERNAME` | Initial admin username. Hardcoded to `admin`. |
| `KC_BOOTSTRAP_ADMIN_PASSWORD` | Initial admin password. Injected from `.env` via `${KC_ADMIN_PASSWORD}`. |
| `KC_HOSTNAME` | Public URL of the Keycloak server. |
| `KC_HOSTNAME_STRICT` | When `false`, Keycloak accepts requests on any hostname. |
| `KC_HOSTNAME_STRICT_HTTPS` | When `false`, Keycloak does not require HTTPS for the hostname. |
| `KC_HTTP_ENABLED` | Enable plain HTTP. Required when running behind a reverse proxy or in development. |
| `KC_PROXY` | Proxy mode. `edge` indicates an external reverse proxy terminates TLS. |
| `KC_INITIALIZE_EMPTY` | Initialize an empty database on first startup. |
| `KC_FEATURE_CACHE_STACK` | Enable the cache stack feature for improved session handling. |
| `QUARKUS_HTTP_LIMITS_MAX_HEADER_SIZE` | Maximum HTTP header size. Set to `64k` to accommodate large JWT tokens. |

:::warning
The `start-dev` command runs Keycloak in development mode. For production deployments, use `start` with a properly configured database and TLS.
:::

### Realm import

Keycloak imports the realm configuration at startup from `./keycloak/import/realm-config.json`. This file defines the `dial` realm, clients, roles, and default users. See [realm-config.json](../../realm-config-json) for the realm file structure.

### Volumes

| Mount | Purpose |
|-------|---------|
| `./keycloak/data` | Keycloak data directory (embedded H2 database in dev mode) |
| `./keycloak/import/realm-config.json` | Realm configuration for automatic import |

## Chat (authenticated)

In the advanced setup, Chat uses Keycloak OIDC instead of API key authentication:

```yaml
chat:
  image: epam/ai-dial-chat:0.41.7
  platform: linux/amd64
  environment:
    AUTH_KEYCLOAK_CLIENT_ID: "dial-chat"
    AUTH_KEYCLOAK_DIAL_ROLES_FIELD: "resource_access.dial-chat.roles"
    AUTH_KEYCLOAK_HOST: "http://keycloak:${KC_PORT}/realms/dial/"
    AUTH_KEYCLOAK_SECRET: "${AUTH_KEYCLOAK_SECRET}"
    DIAL_API_HOST: "http://core:8080"
    ENABLED_FEATURES: "conversations-section,prompts-section,top-settings,..."
    KEEP_ALIVE_TIMEOUT: ${CHAT_KEEP_ALIVE_TIMEOUT:-20000}
    NEXTAUTH_SECRET: "${NEXTAUTH_SECRET}"
    THEMES_CONFIG_HOST: "http://themes:8080"
    NEXTAUTH_URL: "http://localhost:${DIAL_CHAT_HOST_PORT}"
    NEXTAUTH_URL_INTERNAL: "http://chat:3000"
  ports:
    - "${DIAL_CHAT_HOST_PORT}:3000"
  depends_on:
    - themes
    - core
    - keycloak
  restart: always
```

### Environment variables

| Variable | Description |
|----------|-------------|
| `AUTH_KEYCLOAK_CLIENT_ID` | Keycloak client ID for DIAL Chat. Must match the client configured in `realm-config.json`. |
| `AUTH_KEYCLOAK_DIAL_ROLES_FIELD` | Path to the roles claim in the Keycloak token. Uses `resource_access.dial-chat.roles` for client-scoped roles. |
| `AUTH_KEYCLOAK_HOST` | Keycloak realm URL for OIDC discovery. |
| `AUTH_KEYCLOAK_SECRET` | Keycloak client secret. Injected from `.env`. |
| `DIAL_API_HOST` | DIAL Core internal URL. |
| `ENABLED_FEATURES` | Comma-separated list of enabled Chat UI features. |
| `KEEP_ALIVE_TIMEOUT` | HTTP keep-alive timeout in milliseconds. |
| `NEXTAUTH_SECRET` | NextAuth.js session encryption key. Injected from `.env`. |
| `THEMES_CONFIG_HOST` | Themes service internal URL. |
| `NEXTAUTH_URL` | Public URL of DIAL Chat. Used by NextAuth.js for callback URLs. |
| `NEXTAUTH_URL_INTERNAL` | Internal Chat URL for server-side NextAuth.js calls. |

### Comparison with quick start Chat

| Aspect | Quick Start | Advanced |
|--------|------------|---------|
| Authentication | `DIAL_API_KEY` (static key) | Keycloak OIDC (`AUTH_KEYCLOAK_*`) |
| Session encryption | Hardcoded `"secret"` | `${NEXTAUTH_SECRET}` from `.env` |
| Dependencies | Core, Themes | Core, Themes, Keycloak |
| Port | `3000` (fixed) | `${DIAL_CHAT_HOST_PORT}` (configurable) |

See [DIAL Chat configuration](../../chat-configuration) for all Chat environment variables.
