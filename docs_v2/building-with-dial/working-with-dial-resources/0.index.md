# Working with DIAL resources

This section is for developers who manage DIAL resources programmatically through the [Unified API](https://dialx.ai/dial_api). It covers storing files, publishing resources to a shared space, sharing them with other users or deployments, subscribing to change events, and the authorization mechanisms that govern access. You should already be familiar with calling the Unified API and authenticating with an API key or JWT.

In DIAL, a **resource** is a file, conversation, prompt, application, or Tool Set. Every resource is created in the private space of its owner — a user or a deployment — and is accessible only to that owner until it is explicitly shared or published.

## In this section

- [File management](1.file-management.md) — upload, list, download, and delete files through the File Storage Management API. The most common first task for a new DIAL developer.
- [Publications API](2.publications-api.md) — publish private resources to the shared Public space and manage access rules.
- [Sharing API](3.sharing-api.md) — grant other users or deployments access to a resource without making it public.
- [Notifications](4.notifications.md) — receive workflow and resource-change events through the Notification and Subscribe APIs.
- [Per-request keys](5.per-request-keys.md) — reference for the short-lived keys DIAL Core issues for each request to manage file access, cost attribution, and tracing.
- [Auth matrix for apps](6.auth-matrix-for-apps.md) — reference for the consent model that secures access across application dependency chains.
