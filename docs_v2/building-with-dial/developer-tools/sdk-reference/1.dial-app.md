# DIALApp

`DIALApp` is the entry point for every DIAL SDK application. It extends [FastAPI](https://fastapi.tiangolo.com/) and registers your application logic under the Unified API route structure that DIAL Core expects.

```python
from aidial_sdk import DIALApp
```

## Constructor

```python
DIALApp(
    dial_url: str | None = None,
    propagate_auth_headers: bool = False,
    telemetry_config: TelemetryConfig | None = None,
    add_healthcheck: bool = False,
    **kwargs,
)
```

| Parameter | Type | Default | Description |
|-----------|------|---------|-------------|
| `dial_url` | `str \| None` | `None` | Base URL of the DIAL Core instance. Required when `propagate_auth_headers` is `True`. |
| `propagate_auth_headers` | `bool` | `False` | When `True`, the SDK captures the `api-key` header from incoming requests and automatically injects it into outgoing HTTP calls (via `aiohttp`, `httpx`, and `requests`) that target `dial_url`. This enables per-request key propagation when your app calls upstream models through DIAL Core. |
| `telemetry_config` | `TelemetryConfig \| None` | `None` | Configures OpenTelemetry tracing, metrics, and log export. Requires the `telemetry` extras (`pip install "aidial-sdk[telemetry]"`). See [Telemetry](5.telemetry.md). |
| `add_healthcheck` | `bool` | `False` | When `True`, adds a `GET /health` endpoint that returns `{"status": "ok"}`. Access logs for this path are suppressed. |
| `**kwargs` | | | Passed through to the `FastAPI` constructor. Use for `title`, `version`, `docs_url`, and other FastAPI settings. |

## Methods

### add_chat_completion

```python
app.add_chat_completion(
    deployment_name: str,
    impl: ChatCompletion,
    *,
    heartbeat_interval: float | None = None,
) -> DIALApp
```

Registers a [ChatCompletion](chat-completion/0.index.md) implementation under the given deployment name. This creates the following routes:

| Route | Method | Description |
|-------|--------|-------------|
| `/openai/deployments/{name}/chat/completions` | POST | Chat completion (streaming and non-streaming) |
| `/openai/deployments/{name}/rate` | POST | Response rating |
| `/openai/deployments/{name}/tokenize` | POST | Token counting (only if `impl.tokenize` is overridden) |
| `/openai/deployments/{name}/truncate_prompt` | POST | Prompt truncation (only if `impl.truncate_prompt` is overridden) |
| `/openai/deployments/{name}/configuration` | GET | Configuration endpoint (only if `impl.configuration` is overridden) |

The `heartbeat_interval` parameter (in seconds) controls how often a keep-alive comment is sent during streaming responses. When set, the SDK sends `: heartbeat\n\n` comments to prevent proxy timeouts on long-running completions.

Returns `self` for chaining.

### add_embeddings

```python
app.add_embeddings(
    deployment_name: str,
    impl: Embeddings,
) -> DIALApp
```

Registers an [Embeddings](3.embeddings.md) implementation. Creates a single route:

| Route | Method | Description |
|-------|--------|-------------|
| `/openai/deployments/{name}/embeddings` | POST | Embeddings generation |

Returns `self` for chaining.

### configure_telemetry

```python
app.configure_telemetry(config: TelemetryConfig)
```

Initializes OpenTelemetry instrumentation. Called automatically when `telemetry_config` is passed to the constructor. See [Telemetry](5.telemetry.md) for details.

Raises `ValueError` if the `telemetry` extras are not installed.

## Usage example

```python
import uvicorn
from aidial_sdk import DIALApp
from aidial_sdk.chat_completion import ChatCompletion, Request, Response


class MyApp(ChatCompletion):
    async def chat_completion(
        self, request: Request, response: Response
    ) -> None:
        with response.create_single_choice() as choice:
            choice.append_content("Hello from MyApp!")


app = DIALApp(
    dial_url="http://core:8080",
    propagate_auth_headers=True,
    add_healthcheck=True,
)
app.add_chat_completion("my-app", MyApp())

if __name__ == "__main__":
    uvicorn.run(app, port=5000)
```

## Auth header propagation

When `propagate_auth_headers=True`, the SDK intercepts outgoing HTTP requests made by the following libraries and injects the `api-key` header from the original incoming request:

- **aiohttp** — via `TraceConfig` on `ClientSession`
- **httpx** — via monkey-patching `Client.build_request` and `AsyncClient.build_request`
- **requests** — via monkey-patching `Session.send`

Only requests whose URL starts with `dial_url` receive the injected header. This allows your app to call upstream DIAL Core deployments (models, other apps) using the caller's credentials without manually extracting and forwarding the key.

## Extending with FastAPI features

Because `DIALApp` extends `FastAPI`, you can use any FastAPI feature: middleware, dependency injection, custom routes, exception handlers, and lifespan events. Custom routes are served alongside the Unified API routes.
