# Use Cases

This section answers one question: what can you do with DIAL, and why is it worth doing on DIAL rather than another way? Each page describes a real-world scenario, explains the value, then shows how you would build it and where to start. It is for evaluators deciding whether DIAL fits a problem, and for builders looking for a concrete starting point.

Every scenario leads with the value and the reasoning, then drops into a "How you'd build it" section with the DIAL components involved and links into the rest of the documentation. You can read the first half to decide, and the second half to act.

## Scenarios

- [Org-wide AI assistant](1.org-wide-ai-assistant.md) — give every employee safe, governed access to many models through one interface.
- [Chat with your knowledge base](2.chat-with-your-knowledge-base.md) — ground answers in your own documents with retrieval-augmented generation.
- [No-code AI tools for business teams](3.no-code-ai-tools.md) — let non-developers compose models, tools, and agents without writing code.
- [One API for every model provider](4.one-api-every-provider.md) — call any model through a single interface and swap providers without rewriting apps.
- [Centralized cost and usage governance](5.cost-and-usage-governance.md) — see, limit, and attribute AI spend across teams and applications.
- [Custom production AI agents](6.custom-production-agents.md) — build, deploy, and operate bespoke agents with full control over logic and dependencies.
- [Embed AI in existing tools](7.embed-ai-in-existing-tools.md) — bring DIAL into Microsoft Teams, Excel, automation pipelines, and your own web apps.
- [Measure and improve AI quality](8.measure-and-improve-quality.md) — make model and prompt changes against metrics instead of intuition.
- [Compliance and PII-safe AI](9.compliance-and-pii-safe-ai.md) — enforce redaction, audit, and access boundaries on every request.
