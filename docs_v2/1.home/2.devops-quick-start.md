# DevOps quick start

In this tutorial, you will stand up DIAL with Docker Compose and call it directly through the [Unified API](../2.understand-dial/2.architecture/4.unified-api-overview.md) using `curl`. By the end, you will have sent an authenticated chat completion request to DIAL Core and received a response. This path is for platform engineers who operate DIAL as a service rather than through the UI. No cloud account or model API key is required.

## Prerequisites

- Docker Engine with Docker Compose v2.20.0 or higher. See the [Docker documentation](https://docs.docker.com/desktop/).
- A command-line HTTP client (`curl` is used here).
- Free TCP ports `8080` (DIAL Core) and `3000` (DIAL Chat) on your machine.

## What you'll build

A local DIAL Core instance exposing the Echo application as a deployment, which you will call over HTTP. The same request shape works against any model or application in a production deployment — only the host, key, and deployment name change.

## Step 1: Get the Docker Compose files

Download the [`dial-docker-compose/application`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose/application/) folder from the DIAL repository.

**Verify:** the folder contains `docker-compose.yml` and a `core/config.json` file.

## Step 2: Launch DIAL

From the folder that contains `docker-compose.yml`, start the stack:

```bash
docker compose up
```

**Verify:** the `core` service logs show DIAL Core listening on port `8080`.

## Step 3: Inspect the configuration

DIAL Core registers deployments, API keys, and roles in `core/config.json`. This setup defines one application (`echo`), one API key, and a `default` role with access to it:

```json
{
  "applications": {
    "echo": {
      "displayName": "My Echo App",
      "endpoint": "http://echo:5000/openai/deployments/echo/chat/completions"
    }
  },
  "keys": {
    "dial_api_key": {
      "project": "TEST-PROJECT",
      "role": "default"
    }
  },
  "roles": {
    "default": {
      "limits": { "echo": {} }
    }
  }
}
```

The key `dial_api_key` is a development placeholder. In production, keys and roles come from [dynamic settings](../2.understand-dial/4.security-and-governance/2.access-control-reference.md), not a static file.

## Step 4: Call the Unified API

Send a chat completion request to the `echo` deployment. Authenticate with the `Api-Key` header:

```bash
curl http://localhost:8080/openai/deployments/echo/chat/completions \
  -H "Api-Key: dial_api_key" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "role": "user", "content": "Hello, DIAL" }
    ]
  }'
```

**Verify:** the response is a JSON object whose assistant message contains `Hello, DIAL` — the Echo application returns your input unchanged.

## What you learned

- How to run DIAL Core with Docker Compose
- How DIAL Core registers deployments, keys, and roles in `config.json`
- How to send an authenticated request to the Unified API with `curl`
