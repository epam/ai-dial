# DIAL Stack

This page lists the technologies, storage, and infrastructure that DIAL components rely on. DIAL is designed to be lightweight: it favors community-driven, cost-effective solutions to reduce complexity, avoid licensing risk, and keep the total cost of ownership low. The only required component is [DIAL Core](./1.architecture-highlights.md#dial-core-required).

## Component technologies

| Component | Language and framework |
|---|---|
| [DIAL Core](https://github.com/epam/ai-dial-core) | Java 17, Vert.x |
| [DIAL Chat](https://github.com/epam/ai-dial-chat) | Next.js, TypeScript |
| Analytics Realtime, model adapters, interceptors, agent builders, code interpreter | Python 3.11, FastAPI |

Components written in Python may bring additional dependencies. See the [open-source component list](https://dialx.ai/open-source) for the repository of each component.

All technologies used in DIAL — including Java, Python, and JavaScript libraries — are continuously monitored for security vulnerabilities and licensing issues.

## Storage

DIAL does not require a centralized database. It requires two storage dependencies:

| Dependency | Purpose | Supported options |
|---|---|---|
| Blob storage | Persists conversations, prompts, applications, and user files | AWS S3, Google Cloud Storage, Azure Blob Storage, or a local file system (self-hosted) |
| Redis | In-memory cache with write-back (write-behind) caching | Standalone or cluster |

DIAL is cloud-agnostic: the blob storage backend is configurable per deployment.

## Deployment and observability

DIAL runs on the cloud, as a self-hosted solution, or on a personal laptop.

| Concern | Supported options |
|---|---|
| Containers | Docker, Podman, or other containerization tools |
| Cloud marketplaces | [AWS](https://aws.amazon.com/marketplace/pp/prodview-zx4u372lqvtfi), [Google Cloud](https://console.cloud.google.com/marketplace/product/epam-mp-dial/epam-ai-dial), [Microsoft Azure](https://azuremarketplace.microsoft.com/en-us/marketplace/apps/epam-2436412.epam_dial) |
| Orchestration | Kubernetes and [Helm](https://github.com/epam/ai-dial-helm) for production deployments |
| Observability | [OpenTelemetry](https://opentelemetry.io/) for metrics, traces, and logs |
| Secret management | Azure Key Vault, HashiCorp Vault, AWS Secrets Manager, GCP Cloud Key Management |

## Authentication

DIAL natively supports [OpenID Connect](https://openid.net/developers/how-connect-works/) and [OAuth 2.0](https://oauth.net/2/), and integrates with enterprise identity providers where you define user roles and attributes for a custom permissions model.
