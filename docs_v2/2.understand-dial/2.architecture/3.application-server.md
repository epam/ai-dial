# Application server

Beyond routing requests to models, DIAL acts as an application server: it hosts, deploys, and manages GenAI applications across their full lifecycle. This page explains what that means, the development approaches it supports, and the middleware it provides out of the box. It is for developers deciding how much of an application's lifecycle to hand to the platform. For hands-on builds, follow the tutorials linked under [Further reading](#further-reading).

## Why DIAL as an application server

Building a production GenAI application means more than calling a model. You need hosting, access control, rate limiting, load balancing across model deployments, observability, and a way to share and publish the result. DIAL provides this as middleware, so you focus on your application's logic instead of rebuilding the surrounding platform. Applications built on DIAL are compatible with the [Unified API](./4.unified-api-overview.md), which means they can be invoked by other applications as agents in multi-agent workflows.

The lifecycle has five stages: develop, host and deploy, test and evaluate, release, and operate.

## Develop

DIAL supports two development approaches.

**Develop from scratch.** Build an application yourself with the [DIAL SDK](https://github.com/epam/ai-dial-sdk) and the [Unified API](https://dialx.ai/dial_api). You can use any technology — LangChain, LlamaIndex, Semantic Kernel, vector databases, or anything else. Because the result conforms to the Unified API, other applications can call it as an [agent](../../3.building-with-dial/1.apps/0.index.md) in a custom workflow.

**Use predefined application types.** DIAL ships schema-based [application types](../../3.building-with-dial/1.apps/0.index.md) that simplify building common kinds of applications: [Quick Apps](../../3.building-with-dial/1.apps/2.quick-apps/0.index.md), [Code Apps](../../3.building-with-dial/1.apps/3.code-apps/0.index.md), and [Mind Map Studio](../../3.building-with-dial/1.apps/4.mind-map-studio/0.index.md). Schemas define an application's structure, including its UI and editor, so end users can create instances through no-code editors. Organizations can also define custom application types with tailored logic, UI, and wizards.

DIAL provides middleware out of the box to support powerful, multi-agent applications:

- **Language models** — connectivity to leading vendors, open-source models, fine-tuned models, and self-hosted models. See [supported providers](../../3.building-with-dial/3.adapters/2.supported-providers.md).
- **[Interceptors](../../3.building-with-dial/2.interceptors/0.index.md)** — custom logic on incoming and outgoing requests for PII obfuscation, guardrails, and safety checks.
- **Access control** — integration with identity providers to implement a custom role-based access control model. See [auth and access control](../../4.operating-dial/5.auth-and-access-control/0.index.md).
- **Rate and cost limits** — flexible usage and cost limits for JWTs and API keys. See [roles and rate limits](../../4.operating-dial/5.auth-and-access-control/3.roles-and-rate-limits.md).
- **Load balancer** — request distribution across model deployments, with provisioned-throughput support, to prevent bottlenecks and optimize cost.
- **Observability** — OpenTelemetry-based insight into performance and health.
- **Evaluation toolkit** — measurement of retrieval and generation quality for RAG-like applications. See [evaluations](../../3.building-with-dial/4.evaluations/0.index.md).

DIAL also supports real-time co-development: granting WRITE permission lets multiple users work on the same application simultaneously.

## Host and deploy

Applications of standard types — Quick Apps, Code Apps, and Mind Maps — are automatically hosted and deployed on DIAL infrastructure. The platform handles hosting, scaling, file storage, and management, so developers do not manage that work themselves.

DIAL also supports applications hosted and deployed outside its infrastructure. You enable these external applications through the Unified API, the DIAL Chat UI wizards, direct DIAL Core configuration, or [DIAL Admin](./1.architecture-highlights.md#dial-admin). For the practical steps, see [registering apps in DIAL Core](../../3.building-with-dial/1.apps/5.custom-apps/5.register-app.md).

## Test and evaluate

DIAL provides tools to verify that an application meets quality standards before release:

- **Evaluation toolkit** — a UI and [library](https://github.com/epam/ai-dial-rag-eval) to calculate metrics for information retrieval and generation in RAG-like applications.
- **Application logs** — access logs for Code Apps through the [Unified API](https://dialx.ai/dial_api#tag/Applications/operation/getApplicationLogs) and the DIAL Chat UI for debugging and monitoring.
- **Observability** — OpenTelemetry metrics and traces help identify bottlenecks and analyze health and performance, visualized in tools such as Grafana.
- **Usage analytics** — Analytics Realtime analyzes chat completion logs to extract insights for optimization, without storing sensitive data.

## Release

Before release, an application is accessible only to its creator. DIAL offers two release paths:

- **Sharing** — grant READ or WRITE access to specific users, with optional re-sharing permission. Access can be revoked at any time. See the [Sharing API](../../3.building-with-dial/7.working-with-dial-resources/3.sharing-api.md).
- **Publishing** — make an application available on the [Marketplace](../../3.building-with-dial/5.developer-tools/2.chat-customization/6.marketplace.md) to users with specific roles or to everyone. Publishing requires administrator approval, and access can be revoked after approval. See the [Publications API](../../3.building-with-dial/7.working-with-dial-resources/2.publications-api.md).

Published and shared applications are accessible through the Marketplace, DIAL Chat, and the Unified API.

## Operate

DIAL provides tools to manage applications throughout their lifecycle:

- **Access and management** — owners edit applications, revoke usage, collaborate, and manage settings through the DIAL Chat workspace or the Unified API.
- **Monitoring** — Analytics Realtime extracts insights from chat completion logs (usage patterns, conversation topics, sentiment), and administrators view real-time and historical metrics to enforce SLAs, optimize cost, and troubleshoot anomalies.
- **Evaluation** — the evaluation toolkit compares application outputs against ground truth and calculates metrics to fine-tune retrieval and generation behavior.
- **Auto-scaling** — DIAL automatically scales Code Apps, adjusting resources to real-time workload demand for performance and cost efficiency.
