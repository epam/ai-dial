# Unified API overview

The Unified API is the single, OpenAI-compatible interface that [DIAL Core](./1.architecture-highlights.md#dial-core-required) exposes for reaching every language model, embedding model, and application. This page explains the design principle behind it, the features it adds on top of the OpenAI protocol, and the other APIs DIAL Core exposes alongside it. It is for developers deciding how to integrate with DIAL. For endpoint-level detail, see the [DIAL Core API reference](https://dialx.ai/dial_api).

## One protocol for everything

DIAL Core provides a single Unified API, based on the OpenAI API, for accessing all language models, embedding models, and applications. The design principle is a unification layer that makes all models and applications interchangeable — delivering a consistent conversational experience and future-proofing the development of custom GenAI applications.

This matters because it eliminates the need for a different protocol per integration. A client written against the Unified API can switch between an OpenAI model, a Bedrock model, and a custom application without code changes, because [adapters](../../3.building-with-dial/3.adapters/0.index.md) translate each provider's native API into this one protocol. The same OpenAI compatibility lets existing tools and SDKs talk to DIAL with minimal changes.

## Chat completion features

The Unified API extends the OpenAI Chat Completions protocol with features for building richer applications:

- **Streaming** — real-time, token-level response delivery over server-sent events for low-latency conversations.
- **Token usage** — accurate token consumption reporting, including during streaming, for cost and quota management.
- **Seeds** — seed parameters for deterministic, reproducible outputs.
- **Tools** — a standardized way for models to call external APIs (formerly known as functions).
- **Multimodality** — support for non-textual content such as image-to-text, text-to-image, text-to-video, image-to-video, and file transfers.
- **Attachments** — applications can accept and produce file attachments.
- **Stages** — render the intermediate steps an agent took to produce a response.
- **Interactive controls** — buttons, dropdowns, checkboxes, and other controls in generated responses for richer interaction.
- **Custom renderers** — define custom visualizers for conversations using any visualization library.
- **State management** — maintain and pass application and model state across requests.
- **Configuration management** — pass application-specific configuration in a request.

## Other APIs

DIAL Core exposes additional APIs alongside chat completions:

- **[Embedding API](https://dialx.ai/dial_api#operation/sendEmbeddingsRequest)** — unified access to any embedding model, including asymmetric models, instruct models, and multi-modal embeddings.
- **[MCP Server API](https://dialx.ai/dial_api#tag/Toolsets)** — MCP-based communication over HTTP transport, used by [tool sets](../../3.building-with-dial/1.apps/2.quick-apps/1.quick-app-2/9.tool-sets/0.index.md).
- **[File Storage Management API](https://dialx.ai/dial_api#tag/Files)** — access and manage files in DIAL file storage. See [file management](../../3.building-with-dial/7.working-with-dial-resources/1.file-management.md).
- **Resource management APIs** — manage [Applications](https://dialx.ai/dial_api#tag/Applications), [Prompts](https://dialx.ai/dial_api#tag/Prompts), [Conversations](https://dialx.ai/dial_api#tag/Conversations), and [Toolsets](https://dialx.ai/dial_api#tag/Toolsets).
- **Collaboration APIs** — the [Publications API](https://dialx.ai/dial_api#tag/Publications) and [Sharing API](https://dialx.ai/dial_api#tag/Sharing).

## Authentication and access

Requests to the Unified API are authenticated with API keys or JWTs and are subject to role-based access control, usage limits, and cost limits enforced by DIAL Core. For per-request file access and cost attribution, DIAL Core issues short-lived [per-request keys](../../3.building-with-dial/7.working-with-dial-resources/5.per-request-keys.md). See [auth and access control](../../4.operating-dial/5.auth-and-access-control/0.index.md) for configuration.
