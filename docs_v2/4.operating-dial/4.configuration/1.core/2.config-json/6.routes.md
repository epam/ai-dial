# Routes configuration

Routes proxy requests to upstream services. Use routes to expose custom endpoints, integrate external services, or create pre-configured responses.

Routes are defined in `config.json` under `routes.<route_name>`.

## Route attributes

### `paths`

List of path patterns to match. Supports plain strings and regular expressions.

**Required**

```json
{
  "routes": {
    "custom-api": {
      "paths": ["/api/custom/.*"]
    }
  }
}
```

### `methods`

HTTP methods this route handles.

| Value | Description |
|-------|-------------|
| `GET` | Read requests |
| `POST` | Create/action requests |
| `PUT` | Update requests |
| `DELETE` | Delete requests |
| `HEAD` | Metadata requests |
| `PATCH` | Partial update requests |

```json
{
  "routes": {
    "custom-api": {
      "paths": ["/api/custom/.*"],
      "methods": ["GET", "POST"]
    }
  }
}
```

### `upstreams`

Upstream servers to proxy requests to. Same schema as [model upstreams](models#upstream-configuration), with additional WebSocket support (`ws://`, `wss://`).

```json
{
  "routes": {
    "backend-api": {
      "paths": ["/api/v1/.*"],
      "upstreams": [
        {
          "endpoint": "http://backend-service:8080"
        }
      ]
    }
  }
}
```

### `rewritePath`

Replace the upstream path with the original request path.

**Default:** `false`

```json
{
  "routes": {
    "passthrough": {
      "paths": ["/external/.*"],
      "rewritePath": true,
      "upstreams": [
        { "endpoint": "http://external-service:8080" }
      ]
    }
  }
}
```

### `userRoles`

Roles that can access this route. When omitted, all authenticated users have access.

```json
{
  "routes": {
    "admin-api": {
      "paths": ["/admin/.*"],
      "userRoles": ["admin"]
    }
  }
}
```

### `response`

Pre-configured response with `status` and `body`. Mutually exclusive with `upstreams`.

```json
{
  "routes": {
    "health": {
      "paths": ["/health"],
      "methods": ["GET"],
      "response": {
        "status": 200,
        "body": "{\"status\": \"ok\"}"
      }
    }
  }
}
```

### `maxRetryAttempts`

Maximum retries on upstream failure.

```json
{
  "routes": {
    "backend-proxy": {
      "paths": ["/api/backend/.*"],
      "upstreams": [
        { "endpoint": "http://backend:8080" }
      ],
      "maxRetryAttempts": 3
    }
  }
}
```

### `order`

Route priority. Lower values have higher priority. Routes are matched in order.

**Default:** `2147483647` (lowest priority)

```json
{
  "routes": {
    "specific": {
      "paths": ["/api/v1/users"],
      "order": 1
    },
    "general": {
      "paths": ["/api/.*"],
      "order": 100
    }
  }
}
```

:::note
Order cannot be negative.
:::

## Examples

### Static response

```json
{
  "routes": {
    "version": {
      "paths": ["/version"],
      "methods": ["GET"],
      "response": {
        "status": 200,
        "body": "{\"version\": \"1.0.0\"}"
      }
    }
  }
}
```

### Proxy to backend service

```json
{
  "routes": {
    "backend-api": {
      "paths": ["/api/v1/.*"],
      "methods": ["GET", "POST", "PUT", "DELETE"],
      "upstreams": [
        { "endpoint": "http://backend:8080" }
      ],
      "rewritePath": true
    }
  }
}
```

### WebSocket route

```json
{
  "routes": {
    "websocket": {
      "paths": ["/ws/.*"],
      "methods": ["GET"],
      "upstreams": [
        { "endpoint": "ws://realtime-service:8080" }
      ]
    }
  }
}
```

### Admin-only route

```json
{
  "routes": {
    "admin-api": {
      "paths": ["/admin/.*"],
      "methods": ["GET", "POST"],
      "userRoles": ["admin"],
      "upstreams": [
        { "endpoint": "http://admin-service:8080" }
      ]
    }
  }
}
```

### Load-balanced route

```json
{
  "routes": {
    "api": {
      "paths": ["/api/.*"],
      "methods": ["GET", "POST"],
      "upstreams": [
        { "endpoint": "http://api-1:8080", "weight": 2 },
        { "endpoint": "http://api-2:8080", "weight": 1 }
      ],
      "maxRetryAttempts": 2
    }
  }
}
```
