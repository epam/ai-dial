# Docker Compose configuration

DIAL ships with ready-to-use Docker Compose setups maintained in the [AI DIAL repository](https://github.com/epam/ai-dial). Two scenarios are available:

| | Quick Start | Advanced |
|---|---|---|
| **Use case** | Local development, evaluation, demos | Production-like deployment with authentication |
| **Directory** | [`dial-docker-compose/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose) | [`dial-docker-compose-advanced/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced) |
| **Core services** | Chat, Core, Redis, Themes | Chat, Core, Redis, Themes |
| **Authentication** | API key (no identity provider) | Keycloak with OIDC |
| **Admin Panel** | No | Yes (Backend + Frontend) |
| **Model adapters** | Per-setup (OpenAI, Ollama, vLLM) | OpenAI adapter included |
| **RAG** | No | Optional (`docker-compose-full.yml`) |

## Quick Start

Multiple quick start setups share a common base (`common.yml`) and add setup-specific services. Each setup is a self-contained directory under `dial-docker-compose/`.

See [Quick Start](quick-start) for all available configurations.

## Advanced

A single production-like setup with Keycloak, Admin Panel, and model adapters. Two compose files: a base with all core services and an extended variant that adds RAG.

See [Advanced setup](advanced/index) for the full service breakdown.

## Environment variables

Both scenarios use `.env` files for port assignments, secrets, and host configuration. See the [.env reference](env-reference) for all variables.
