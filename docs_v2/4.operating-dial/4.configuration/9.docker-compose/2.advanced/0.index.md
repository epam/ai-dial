# Advanced setup

The advanced Docker Compose setup in [`dial-docker-compose-advanced/`](https://github.com/epam/ai-dial/tree/main/dial-docker-compose-advanced) provides a production-like deployment with Keycloak authentication, an Admin Panel, and multiple model adapters.

## File structure

| File | Purpose |
|------|---------|
| [`docker-compose-base.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-base.yml) | All core services: Chat, Core, Redis, Themes, Keycloak, Admin Backend, Admin Frontend, OpenAI Adapter |
| [`docker-compose-full.yml`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/docker-compose-full.yml) | Includes `docker-compose-base.yml` and adds DIAL RAG |
| [`.env`](https://github.com/epam/ai-dial/blob/main/dial-docker-compose-advanced/.env.example) | Port assignments, secrets, and host configuration |

## Services

`docker-compose-base.yml` defines 8 services:

| Service | Image | Description | Details |
|---------|-------|-------------|---------|
| `core` | `epam/ai-dial-core:0.39.2` | API server with identity providers and encryption | [Core, Redis & Themes](core-services) |
| `redis` | `redis:7.2.4-alpine3.19` | In-memory cache | [Core, Redis & Themes](core-services) |
| `themes` | `epam/ai-dial-chat-themes:0.10.0` | Static theme server | [Core, Redis & Themes](core-services) |
| `chat` | `epam/ai-dial-chat:0.41.7` | Web UI with Keycloak auth | [Authentication](auth) |
| `keycloak` | `keycloak/keycloak:26.3` | Identity provider | [Authentication](auth) |
| `adapter-openai` | `epam/ai-dial-adapter-openai:0.36.1` | OpenAI/Azure adapter | [Adapters & RAG](adapters-and-rag) |
| `admin-backend` | `epam/ai-dial-admin-backend:0.13.0` | Admin API | [Admin Panel](admin) |
| `admin-frontend` | `epam/ai-dial-admin-frontend:0.13.0` | Admin web UI | [Admin Panel](admin) |

`docker-compose-full.yml` extends the base with one additional service:

| Service | Image | Description | Details |
|---------|-------|-------------|---------|
| `dial-rag` | `ghcr.io/epam/ai-dial-rag:0.32.0` | Document indexing and retrieval | [Adapters & RAG](adapters-and-rag) |

## Configuration approach

Unlike the [quick start setups](../quick-start), which load `settings.json` via the `AIDIAL_SETTINGS` environment variable, the advanced Core service uses `aidial.*` environment variable overrides directly. Each `aidial.*` variable maps to a key in `settings.json`. See [settings.json — Configuration precedence](../../core/settings-json/index#configuration-precedence) for override rules.

## Running the setup

Base setup (without RAG):

```bash
cd dial-docker-compose-advanced
cp .env.example .env   # Edit with your secrets
docker compose -f docker-compose-base.yml up
```

Full setup (with RAG):

```bash
docker compose -f docker-compose-full.yml up
```

| Service | URL |
|---------|-----|
| DIAL Chat | `http://localhost:${DIAL_CHAT_HOST_PORT}` (default: 3000) |
| DIAL Core API | `http://localhost:${DIAL_CORE_HOST_PORT}` (default: 8080) |
| Admin Panel | `http://localhost:${DIAL_ADMIN_FRONTEND_HOST_PORT}` (default: 3100) |
| Keycloak Admin | `http://localhost:${KC_PORT}` (default: 8180) |
