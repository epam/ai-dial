# Enable publications in DIAL Chat

Turn on the publication feature so DIAL Chat users can publish conversations and prompts for review and organization-wide sharing. This guide is for DevOps engineers who configure DIAL deployments. It assumes you already run DIAL Core and DIAL Chat, and that you administer an identity provider (IdP).

Publications let users submit content through an approval flow before it becomes available to others. Enabling the feature requires three coordinated changes: user roles in your IdP, access rules in DIAL Core, and feature flags in DIAL Chat.

## Prerequisites

- A running DIAL Core and DIAL Chat deployment.
- Administrative access to your identity provider.
- Familiarity with [authentication and access control](../5.auth-and-access-control/0.index.md) in DIAL.

## Step 1: Define the user hierarchy in your IdP

Create the user hierarchy and claims in your identity provider, then connect DIAL to it. The claims you assign — such as `roles` — drive the access rules that DIAL Core evaluates in the next step.

For the list of supported providers and connection steps, see [SSO / IdP setup](../5.auth-and-access-control/4.sso-idp/0.index.md).

## Step 2: Configure access rules in DIAL Core

DIAL Core maps IdP claims to permissions through access rules in `config.json`. Define rules that match your organization's access policies.

The following default rule grants admin access to any user whose `roles` claim contains the value `admin`:

```json
{
  "access": {
    "admin": {
      "rules": [
        {
          "function": "CONTAIN",
          "source": "roles",
          "targets": ["admin"]
        }
      ]
    }
  }
}
```

For the full set of rule functions and role attributes, see [Roles configuration](./1.core/2.config-json/4.roles.md).

## Step 3: Enable the feature flags in DIAL Chat

DIAL Chat exposes publications through the `ENABLED_FEATURES` environment variable. Add `conversations-publishing` and `prompts-publishing` to the comma-separated list:

```yaml
chat:
  environment:
    ENABLED_FEATURES: "conversations-publishing,prompts-publishing"
```

Keep any features your deployment already enables in the same list — `ENABLED_FEATURES` replaces the full set, it does not append. For the complete list of flags and other DIAL Chat settings, see [DIAL Chat configuration](./2.chat-configuration.md).

## Result

After restarting DIAL Chat, users see publishing controls for conversations and prompts. Submitted items enter the approval flow before they become available to other users.
