# Install the Quick Apps feature

Enable Quick Apps in your DIAL deployment by deploying the Quick Apps executor service and wiring it into DIAL Core and DIAL Chat. This guide is for DevOps engineers running DIAL on Kubernetes. Once installed, users and developers create Quick Apps through the UI, the API, or `config.json` — see [What are Quick Apps](../../3.building-with-dial/1.apps/2.quick-apps/0.index.md) for the authoring side.

## Goal

Deploy the Quick Apps executor and configure DIAL Core and DIAL Chat so Quick Apps become available platform-wide.

## Prerequisites

- A DIAL deployment on Kubernetes with Helm access (see [Helm chart overview](index))
- Permission to deploy Helm charts and edit DIAL Core and DIAL Chat configuration in the cluster
- [DIAL Core](https://github.com/epam/ai-dial-core) version 0.28.0 or higher
- [DIAL Chat](https://github.com/epam/ai-dial-chat) version 0.29.0 or higher

## Step 1: Deploy the Quick Apps executor

The executor is a container deployed with the generic `dial/dial-extension` chart. Create a `values.yaml`:

```yaml
image:
  pullPolicy: Always
  registry: <REGISTRY>
  repository: <QUICKAPPS_IMAGE_REPOSITORY>
  tag: <QUICKAPPS_IMAGE_TAG>
  pullSecrets:
    - <PULL_SECRET_NAME>

fullnameOverride: quickapps

ingress:
  enabled: false

serviceAccount:
  create: true

env:
  DIAL_URL: "http://dial-core.dial.svc.cluster.local"
  API_VERSION: "2025-01-01-preview"
  TEMPERATURE_FALLBACK: "true"
  SYSTEM_PROMPT_FALLBACK: "true"
```

The environment variables control executor behavior:

- `DIAL_URL` — the in-cluster URL of DIAL Core.
- `API_VERSION` — the unified protocol version used to call models and apps. Omit to use the default `2025-01-01-preview`.
- `TEMPERATURE_FALLBACK` — when `true`, the executor omits `temperature` for models that do not support it.
- `SYSTEM_PROMPT_FALLBACK` — when `true`, the executor appends instructions to each user message for models without system-prompt support.
- `RAG_DEPLOYMENT_NAME` — optional; overrides the default RAG deployment (`dial-rag`).

Deploy the executor:

```bash
helm upgrade --install dial-quickapps dial/dial-extension \
  -f ./values.yaml \
  --version <CHART_VERSION> \
  -n dial
```

## Step 2: Register the Quick Apps application type in DIAL Core

Add the Quick Apps application type schema to DIAL Core's configuration, and set its endpoints to the executor service deployed in Step 1:

```json
{
  "applicationTypeSchemas": [
    {
      "$schema": "https://dial.epam.com/application_type_schemas/schema#",
      "$id": "https://mydial.epam.com/custom_application_schemas/quickapps",
      "dial:applicationTypeDisplayName": "QuickApp",
      "dial:applicationTypeCompletionEndpoint": "http://quickapps.dial.svc.cluster.local/openai/deployments/quick_apps/chat/completions",
      "dial:applicationTypeConfigurationEndpoint": "http://quickapps.dial.svc.cluster.local/openai/deployments/quick_apps/configuration",
      "dial:appendApplicationPropertiesHeader": true
    }
  ]
}
```

`dial:appendApplicationPropertiesHeader` controls how the executor receives a Quick App's configuration:

- `true` — DIAL Core appends the configuration to the chat completion request headers.
- `false` — the executor fetches the configuration from DIAL Core in a separate request. Prefer this when configurations are too large for a request header.

:::note
The full application type schema defines the configurable properties of a Quick App. Maintain it in your DIAL Core configuration. For the property model, see the [Quick App configuration reference](../../3.building-with-dial/1.apps/2.quick-apps/2.quick-app-original/2.reference.md).
:::

## Step 3: Configure DIAL Chat

Set these environment variables in the DIAL Chat values so the UI exposes Quick Apps:

```yaml
chat:
  env:
    # Quick Apps completions host; may serve public and private resources
    QUICK_APPS_HOST: "http://quickapps.dial.svc.cluster.local"
    # applicationTypeSchemaId for the Quick App application type
    QUICK_APPS_SCHEMA_ID: "https://mydial.epam.com/custom_application_schemas/quickapps"
    # Default model used by Quick Apps
    QUICK_APPS_MODEL: "gpt-4o-2024-05-13"
```

Add `quick-apps` to the enabled features:

```yaml
chat:
  env:
    ENABLED_FEATURES: "...,quick-apps"
```

## Step 4: Enable custom apps in DIAL Core

Set the following environment variable for DIAL Core so it serves custom application types, including Quick Apps:

```yaml
core:
  env:
    aidial.applications.includeCustomApps: "true"
```

Apply the updated Helm values to DIAL Core and DIAL Chat to roll out the changes.

## Result

The Quick Apps executor runs in the cluster, DIAL Core recognizes the Quick App application type, and DIAL Chat shows the Quick Apps creation flow. Users can now build Quick Apps.
