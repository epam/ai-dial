# Building with DIAL

This section is for developers who build applications, extensions, and integrations on the DIAL platform. Whether you are creating a no-code Quick App or writing a production-grade Custom App with the DIAL SDK, start here.

## Apps

The DIAL app ecosystem — Custom Apps, Quick Apps, Code Apps, and Mind Map Studio. [Learn](1.apps/1.when-to-use-which.md) what each app type does, when to use it, and how it fits into the platform.

[Go to DIAL Apps](1.apps/0.index.md)

## Extension points

Interceptors that transform requests and responses in flight, and adapters that connect external AI providers to the Unified API.

- [Interceptors](2.interceptors/0.index.md) — middleware for PII redaction, content filtering, audit logging, and compliance enforcement
- [Adapters](3.adapters/0.index.md) — translate external AI provider APIs into the Unified API

## Quality and testing

Evaluate and improve the quality of your DIAL applications with the RAG evaluation toolkit.

- [Evaluations](4.evaluations/0.index.md) — measure retrieval and generation quality of RAG applications

## Developer tools

SDK reference, Chat customization, data visualization, and theming.

- [SDK reference](5.developer-tools/1.sdk-reference/0.index.md) — Python framework for building Custom Apps and model adapters that plug into the DIAL platform
- [Chat customization](5.developer-tools/2.chat-customization/0.index.md) — developer-facing extension points and customization options that let you control what Chat renders, how it looks, and how it integrates with your applications

## Integrations

Connect DIAL with external systems — chatbots, productivity add-ins, workflow automation, and orchestration patterns.

- [Chatbot integrations](6.integrations/1.chatbot-integrations/0.index.md) — embed DIAL in Microsoft Teams and Microsoft Copilot
- [Productivity add-ins](6.integrations/2.productivity-add-ins/0.index.md) — bring DIAL's models and applications into the desktop and productivity tools your users already work in
- [Workflow automation](6.integrations/3.workflow-automation/0.index.md) — let you chain applications, services, and APIs into pipelines that run without manual steps
