# Tutorial: build a RAG app

In this tutorial, you will build a Retrieval-Augmented Generation (RAG) Custom App that answers questions about uploaded text files. The app receives a document as an attachment, splits it into chunks, stores them in a local vector index, and uses an LLM to generate grounded answers. You will learn how to handle file attachments, call upstream models, and use streaming responses. Basic familiarity with the [Getting started tutorial](1.getting-started-sdk.md) and the [translator app tutorial](2.tutorial-translator-app.md) is assumed.

## Prerequisites

- Completed the [Getting started with DIAL SDK](1.getting-started-sdk.md) tutorial
- Completed the [Tutorial: translator app](2.tutorial-translator-app.md) (or understand per-request key propagation)
- Docker and Docker Compose (V2) installed
- At least 4 GB of free RAM (for Ollama)

## What you will build

A RAG application that:

1. Accepts a text file attachment from the user.
2. Splits the file content into chunks and creates an in-memory vector index.
3. On each question, retrieves relevant chunks and sends them as context to an LLM.
4. Streams the LLM's grounded answer back to the user.

```
User (with file) → DIAL Core → RAG App → [chunk + embed + retrieve] → DIAL Core → LLM → RAG App → DIAL Core → User
```

## Step 3: Implement the RAG app

Create `rag-app/app.py`:

```python
import os
from typing import Optional

import json
import httpx
import uvicorn

from aidial_sdk import DIALApp
from aidial_sdk.chat_completion import ChatCompletion, Request, Response

DIAL_URL = os.environ.get("DIAL_URL", "http://core:8080")
MODEL_DEPLOYMENT = os.environ.get("MODEL_DEPLOYMENT", "chat-model")
OLLAMA_MODEL = os.environ.get("OLLAMA_MODEL", "llama3.2:1b")
CHUNK_SIZE = int(os.environ.get("CHUNK_SIZE", "200"))
TOP_K = int(os.environ.get("TOP_K", "3"))


def split_into_chunks(text: str, chunk_size: int) -> list[str]:
    words = text.split()
    chunks = []
    for i in range(0, len(words), chunk_size):
        chunk = " ".join(words[i : i + chunk_size])
        if chunk.strip():
            chunks.append(chunk)
    return chunks


def simple_similarity(query: str, chunk: str) -> float:
    query_words = set(query.lower().split())
    chunk_words = set(chunk.lower().split())
    if not query_words:
        return 0.0
    return len(query_words & chunk_words) / len(query_words)


def retrieve(query: str, chunks: list[str], top_k: int) -> list[str]:
    scored = [(chunk, simple_similarity(query, chunk)) for chunk in chunks]
    scored.sort(key=lambda x: x[1], reverse=True)
    return [chunk for chunk, score in scored[:top_k] if score > 0]


class RagApplication(ChatCompletion):
    def __init__(self):
        self.document_chunks: dict[str, list[str]] = {}

    async def chat_completion(
        self, request: Request, response: Response
    ) -> None:
        last_message = request.messages[-1]
        user_text = last_message.content or ""

        attachment_content = await self._extract_attachment(request)

        if attachment_content:
            chunks = split_into_chunks(attachment_content, CHUNK_SIZE)
            self.document_chunks[request.jwt or "default"] = chunks
            with response.create_single_choice() as choice:
                choice.append_content(
                    f"Document indexed. Split into {len(chunks)} chunks. "
                    f"Ask me a question about it."
                )
            return

        chunks = self.document_chunks.get(request.jwt or "default", [])
        if not chunks:
            with response.create_single_choice() as choice:
                choice.append_content(
                    "No document uploaded yet. Please attach a text file "
                    "and send it to index its content."
                )
            return

        relevant_chunks = retrieve(user_text, chunks, TOP_K)
        context = "\n\n---\n\n".join(relevant_chunks)

        answer = await self._ask_model(context, user_text)
        with response.create_single_choice() as choice:
            choice.append_content(answer)


    async def _ask_model(
        self, context: str, user_text: str
    ) -> str:
        """Call a language model through DIAL Core."""
        url = (
            f"{DIAL_URL}/openai/deployments/"
            f"{MODEL_DEPLOYMENT}/chat/completions"
        )
        
        headers = {
            "Content-Type": "application/json",
            }

        payload = {
            "model": OLLAMA_MODEL,
            "messages": [
                {
                    "role": "system",
                    "content": (
                        "You are a helpful assistant. Answer the user's "
                        "question based ONLY on the following context. If the "
                        "answer is not in the context, say you don't know.\n\n"
                        f"Context:\n{context}"
                    ),
                },
                {"role": "user", "content": user_text},
            ],
        }

        async with httpx.AsyncClient() as client:
            resp = await client.post(
                url, headers=headers, json=payload, timeout=120.0
            )
            resp.raise_for_status()
            data = resp.json()
            return data["choices"][0]["message"]["content"]

    async def _extract_attachment(self, request: Request) -> Optional[str]:
        last_message = request.messages[-1]
        if not last_message.custom_content:
            return None
        attachments = last_message.custom_content.attachments
        if not attachments:
            return None

        attachment = attachments[0]
        if attachment.data:
            return attachment.data
        if attachment.url:
            return await self._download_file(request, attachment.url)
        return None

    async def _download_file(
        self, request: Request, url: str
    ) -> Optional[str]:
        import httpx

        if not url.startswith(("http://", "https://")):
            url = f"{DIAL_URL}/v1{url}" if url.startswith("/") else f"{DIAL_URL}/v1/{url}"

        headers = {}
        if request.jwt:
            headers["api-key"] = request.jwt
        async with httpx.AsyncClient() as client:
            resp = await client.get(url, headers=headers)
            if resp.status_code == 200:
                return resp.text
        return None


app = DIALApp(dial_url=DIAL_URL, propagate_auth_headers=True)
app.add_chat_completion("rag-app", RagApplication())
```

To run locally with uvicorn:
```bash
uvicorn app:app --host 0.0.0.0 --port 5000
```

Key concepts in this code:

- **File attachments** arrive in `custom_content.attachments` on the last message. Each attachment has either inline `data` or a `url` pointing to DIAL file storage.
- **Streaming responses** use the OpenAI SDK's `stream=True` parameter. Each chunk's content is appended to the response via `choice.append_content()`.
- **Per-request key propagation** (`propagate_auth_headers=True`) ensures the upstream LLM call is authorized automatically.
- **Simple keyword-based retrieval** is used here to keep the tutorial focused on the DIAL integration pattern. In production, you would use vector embeddings (e.g., FAISS, Chroma) for semantic retrieval.

## Step 6: Configure DIAL Core

Create `core/config.json`:

```json
{
  "models": {
    "chat-model": {
      "type": "chat",
      "displayName": "Local LLM (Ollama)",
      "endpoint": "http://ollama:11434/v1/chat/completions"
    }
  },
  "applications": {
    "rag-app": {
      "displayName": "RAG Document Q&A",
      "description": "Upload a text file, then ask questions about its content",
      "endpoint": "http://rag-app:5000/openai/deployments/rag-app/chat/completions",
      "inputAttachmentTypes": ["text/plain"],
      "maxInputAttachments": 1
    }
  },
  "keys": {
    "dial_api_key": {
      "project": "TEST-PROJECT",
      "role": "default"
    }
  },
  "roles": {
    "default": {
      "limits": {
        "chat-model": {},
        "rag-app": {}
      }
    }
  }
}
```

The `inputAttachmentTypes` field tells DIAL Chat to show the file upload button for this application and restrict uploads to plain text files. `maxInputAttachments` limits it to one file per message.

## What you learned

- How to handle file attachments in a Custom App using `custom_content.attachments`.
- How to stream responses from an upstream LLM back to the user via `choice.append_content()`.
- How to configure `inputAttachmentTypes` and `maxInputAttachments` in DIAL Core to enable file uploads for your app.
- The RAG pattern: ingest → chunk → retrieve → generate, all within a single DIAL Custom App.
- How per-request key propagation secures app-to-model communication automatically.

## Production considerations

This tutorial uses keyword matching for retrieval to keep the focus on DIAL integration. For production RAG applications:

- Use vector embeddings (via an embedding model deployed in DIAL or an external service) for semantic retrieval.
- Use a persistent vector store (FAISS, Chroma, Weaviate, Qdrant) instead of in-memory storage.
- Implement document chunking strategies appropriate for your content type.
- Consider using [DIAL RAG](https://github.com/epam/ai-dial-rag) — EPAM's production RAG service built on the same patterns shown here.
